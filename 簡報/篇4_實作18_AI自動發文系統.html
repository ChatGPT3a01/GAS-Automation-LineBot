🔍 程式碼解析 1 — Gemini API 串接（callGemini）

這段的目的：呼叫 Google 的 Gemini AI，送出提示詞（Prompt），取回 AI 生成的文字回覆。

function callGemini(prompt) {
  var url = 'https://generativelanguage.googleapis.com/v1beta/models/'
    + GEMINI_MODEL + ':generateContent?key=' + GEMINI_API_KEY;

  var payload = {
    contents: [{
      parts: [{
        text: prompt
      }]
    }],
    generationConfig: {
      temperature: 0.8,
      maxOutputTokens: 1024
    }
  };

  var options = {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  };

  try {
    var response = UrlFetchApp.fetch(url, options);
    var code = response.getResponseCode();

    if (code !== 200) {
      Logger.log('Gemini API 呼叫失敗，狀態碼：' + code);
      Logger.log('錯誤內容：' + response.getContentText());
      return null;
    }

    var json = JSON.parse(response.getContentText());

    if (json.candidates && json.candidates[0] &&
        json.candidates[0].content && json.candidates[0].content.parts) {
      var text = json.candidates[0].content.parts[0].text;
      Logger.log('Gemini 回覆：' + text.substring(0, 100) + '...');
      return text;
    }

    Logger.log('Gemini 回覆格式異常：' + JSON.stringify(json));
    return null;

  } catch (error) {
    Logger.log('Gemini API 錯誤：' + error.message);
    return null;
  }
}

逐行解釋：

  • url — Gemini API 的端點網址，格式：models/{模型名}:generateContent?key={你的Key}
  • contents → parts → text — 把我們的提示詞放進 Gemini 要求的 JSON 格式中
  • temperature: 0.8 — 控制 AI 創意度（0=保守，1=天馬行空），0.8 是「有創意但不失控」
  • maxOutputTokens: 1024 — 限制回覆長度，約 500~800 個中文字，足夠一篇貼文
  • UrlFetchApp.fetch(url, options) — GAS 內建的 HTTP 請求工具
  • json.candidates[0].content.parts[0].text — Gemini API 固定的回覆結構路徑
  • try-catch — 捕獲網路錯誤，避免程式崩潰

🔍 程式碼解析 3 — Prompt 工程（buildPrompt）

這段的目的：設計一段精心撰寫的提示詞，讓 AI 每次都產出格式一致、品質穩定的社群貼文。

function buildPrompt(topic) {
  var prompt = '你是一位專業的社群內容創作者。\n\n';
  prompt += '請幫我撰寫一篇關於「' + topic + '」的社群貼文。\n\n';
  prompt += '要求：\n';
  prompt += '1. 標題：吸引人、簡潔有力（15字以內）\n';
  prompt += '2. 內文：200~300字，口語化、易讀\n';
  prompt += '3. 包含 2~3 個實用的重點或建議\n';
  prompt += '4. 結尾加上 3~5 個相關的 hashtag\n';
  prompt += '5. 適當使用 emoji 讓文章更生動\n\n';
  prompt += '請用以下格式回覆：\n';
  prompt += '【標題】你的標題\n';
  prompt += '【內文】你的內文\n';
  prompt += '【標籤】#tag1 #tag2 #tag3';

  return prompt;
}

Prompt 設計三大原則

🎭

1. 角色設定

「你是一位專業的社群內容創作者」讓 AI 進入對應角色，產出更專業

📏

2. 明確要求

字數（200~300字）、格式（口語化）、風格（含 emoji）— 越明確越不會亂跑

📐

3. 結構化輸出

【標題】【內文】【標籤】固定格式，方便後續程式用正則表達式解析

💡 為什麼要用【標題】【內文】【標籤】格式？
因為 AI 的回覆是一整段文字，我們需要用固定的「標記」把不同部分分開。有了這些標記，下一個函式 parseResponse() 才能用正則表達式精準提取每個欄位，分別存入試算表的不同欄位中。

🔍 程式碼解析 4 — 回覆解析（parseResponse）

這段的目的：用正則表達式從 AI 回覆中精準提取標題、內文、標籤三個部分。

function parseResponse(response) {
  var result = {
    title: '',
    content: '',
    tags: ''
  };

  // 提取標題
  var titleMatch = response.match(/【標題】(.+?)(\n|【)/);
  if (titleMatch) {
    result.title = titleMatch[1].trim();
  }

  // 提取內文
  var contentMatch = response.match(/【內文】([\s\S]+?)【標籤】/);
  if (contentMatch) {
    result.content = contentMatch[1].trim();
  }

  // 提取標籤
  var tagsMatch = response.match(/【標籤】(.+)/);
  if (tagsMatch) {
    result.tags = tagsMatch[1].trim();
  }

  // 如果格式解析失敗，用整篇文字作為內文
  if (!result.title && !result.content) {
    result.title = '每日分享';
    result.content = response;
    result.tags = '';
  }

  return result;
}

正則表達式逐一拆解

正則表達式拆解說明提取結果
/【標題】(.+?)(\n|【)/ (.+?) = 盡量少的任意字元；到換行 \n 或下一個 【 為止 標題文字（例如「AI 改變生活」）
/【內文】([\s\S]+?)【標籤】/ [\s\S]+? = 任意字元（含換行）盡量少；到 【標籤】 為止 內文全文（可能跨多行）
/【標籤】(.+)/ (.+) = 【標籤】之後的所有文字到行尾 hashtag 列表（例如「#AI #科技」）

💡 什麼是 .match()？

JavaScript 的字串方法，用正則表達式搜尋匹配的內容。回傳陣列：[0] 是完整匹配、[1] 是第一個括號群組。

✅ 容錯設計

即使 AI 沒按照格式回覆，程式也不會崩潰 — 會用預設標題「每日分享」搭配完整回覆文字，確保系統正常運作。

🔍 程式碼解析 5 — 試算表記錄（initRecordSheet + saveToSheet）

這段的目的：自動建立「AI 發文記錄」工作表，並把每篇 AI 文章的日期、主題、標題、內文、標籤、狀態記錄下來。

📁 initRecordSheet() — 初始化記錄表

function initRecordSheet() {
  var ss = SpreadsheetApp.openById(SPREADSHEET_ID);
  var sheet = ss.getSheetByName(SHEET_NAME);

  if (!sheet) {
    sheet = ss.insertSheet(SHEET_NAME);
    var headers = ['日期', '主題', '標題', '內文', '標籤', '狀態'];
    sheet.getRange(1, 1, 1, headers.length).setValues([headers]);
    sheet.getRange(1, 1, 1, headers.length)
      .setFontWeight('bold')
      .setBackground('#1b5e20')
      .setFontColor('white');
    sheet.setFrozenRows(1);
  }

  return sheet;
}
  • SpreadsheetApp.openById() — 用 ID 開啟試算表
  • getSheetByName(SHEET_NAME) — 找名為「AI 發文記錄」的工作表
  • if (!sheet) — 如果不存在就自動建立，設定 6 欄標題、深綠背景、白色粗體、凍結首列

📁 saveToSheet(topic, article) — 記錄到試算表

function saveToSheet(topic, article) {
  var sheet = initRecordSheet();
  var now = Utilities.formatDate(new Date(), 'Asia/Taipei', 'yyyy/MM/dd HH:mm');

  sheet.appendRow([
    now,
    topic,
    article.title,
    article.content,
    article.tags,
    '已推播'
  ]);

  Logger.log('文章已記錄到試算表');
}
  • Utilities.formatDate() — 將日期格式化為台灣時間的 yyyy/MM/dd HH:mm
  • sheet.appendRow() — 在最後一行新增一列資料，依序填入 6 個欄位
💡 試算表預覽效果：第一次執行後，試算表會自動出現「AI 發文記錄」工作表，標題列為深綠色白字，每篇文章佔一列。

🔍 程式碼解析 6 — 主函式（generateAndPost）

這段的目的：串起前面所有函式，完成「選主題 → 建 Prompt → 問 AI → 解析 → 記錄 → 推播」的完整流程。

function generateAndPost(topic) {
  Logger.log('===== 開始 AI 自動發文 =====');

  // 如果沒指定主題，隨機選一個
  if (!topic) {
    topic = TOPICS[Math.floor(Math.random() * TOPICS.length)];
  }
  Logger.log('主題：' + topic);

  // 步驟 1：建立 Prompt
  var prompt = buildPrompt(topic);
  Logger.log('Prompt 已建立');

  // 步驟 2：呼叫 Gemini API
  var aiResponse = callGemini(prompt);

  if (!aiResponse) {
    pushText(LINE_USER_ID, '⚠️ AI 自動發文失敗：無法取得 AI 回覆，請檢查 API Key。');
    return;
  }

  // 步驟 3：解析回覆
  var article = parseResponse(aiResponse);
  Logger.log('標題：' + article.title);

  // 步驟 4：記錄到試算表
  saveToSheet(topic, article);

  // 步驟 5：推播到 Line
  var message = '🤖 AI 每日好文\n';
  message += '━━━━━━━━━━━━━━━\n\n';
  message += '📌 ' + article.title + '\n\n';
  message += article.content + '\n\n';
  if (article.tags) {
    message += article.tags + '\n\n';
  }
  message += '━━━━━━━━━━━━━━━\n';
  message += '📝 主題：' + topic + '\n';
  message += '💡 由 Gemini AI + GAS 自動產生';

  // Line 單則訊息上限 5000 字
  if (message.length > 4900) {
    message = message.substring(0, 4900) + '\n\n...（已截斷）';
  }

  pushText(LINE_USER_ID, message);

  Logger.log('===== AI 自動發文完成 =====');
}

📱 Line 推播失敗

狀態碼 401Token 不正確 → 重新複製 Channel Access Token
狀態碼 400User ID 格式錯 → 確認是大寫 U 開頭
完全沒反應確認 Line Bot 已加為好友（用手機掃 QR Code）

⏰ 排程沒有觸發

  • 新建立的排程最快隔天才會觸發
