🤖 Line Bot 是什麼？

Line Bot 是一個「自動回應的 LINE 帳號」，可以主動推播訊息，也可以被動回覆使用者。

運作原理

👤 使用者手機
⇄
☁️ LINE 平台
⇄
⚙️ GAS 程式

📤 推播（Push）

由程式主動發訊息給使用者

  • GAS 呼叫 LINE API
  • LINE 平台轉送到手機
  • 使用者不需要先傳訊息
  • 適合：通知、提醒、報表

💬 回覆（Reply）

使用者傳訊息後，程式自動回覆

  • 使用者傳訊息到 LINE Bot
  • LINE 轉送到 GAS（Webhook）
  • GAS 處理後回覆
  • 適合：問答、查詢、互動
本實作重點：先學「推播」（Push），下一個實作再學「回覆」（Reply + Webhook）。

🔑 步驟一：建立 LINE Developers 帳號

到 LINE Developers 網站註冊開發者帳號。

1 開啟瀏覽器
前往 https://developers.line.biz/
2 登入 LINE 帳號
點右上角「Log in」→ 選擇「以 LINE 帳號登入」→ 輸入 Email 和密碼（或掃 QR Code）
3 填寫開發者資料
第一次登入會要求填寫：姓名、Email → 勾選同意條款 → 點「Create my account」
4 進入 Console 頁面
看到 LINE Developers Console 的主畫面 → 恭喜，開發者帳號建立完成！
注意：如果你已經有 LINE Developers 帳號，直接登入即可，不需要重新建立。

📦 步驟二：建立 Provider 和 Channel

Provider 是「開發者/公司」，Channel 是「一個 Bot 應用」。

1 建立 Provider
在 Console 首頁 → 點「Create」（或「Create a new provider」）→ 輸入名稱（例如：GAS教學）→ 點「Create」
2 建立 Messaging API Channel
進入 Provider 頁面 → 點「Create a Messaging API channel」
3 填寫 Channel 資料
Channel typeMessaging API（自動帶入）
Provider選擇剛建立的 Provider
Channel name輸入 Bot 名稱（例如：我的第一個Bot）
Channel description輸入描述（例如：GAS 自動化教學用）
Category選擇分類（例如：Education）
Subcategory選擇子分類（例如：School）
4 同意條款並建立
勾選兩個同意條款 → 點「Create」→ 確認彈窗 → Channel 建立完成！

🎫 步驟三：取得 Channel Access Token

Token 就是「通行證」，你的程式需要它才能呼叫 LINE API。

1 進入 Channel 設定頁
點擊剛建立的 Channel 名稱 → 進入 Channel 設定頁面
2 切換到 Messaging API 分頁
頁面上方有多個分頁 → 點「Messaging API」分頁
3 發行 Token
滑到最下方 → 找到「Channel access token」區塊 → 點「Issue」按鈕 → 產生一串很長的英數字
4 複製 Token
點 Token 旁邊的「複製」按鈕 → 先貼到記事本備用
重要：Token 非常長（約 170 個字元），複製時請確認有複製完整，不要缺字！這是最容易出錯的地方。
安全提醒：Token 等同密碼，不要公開分享或上傳到 GitHub！

👤 步驟四：取得 User ID

User ID 是你的 LINE 帳號的唯一識別碼，推播訊息需要知道「傳給誰」。

1 切換到 Basic settings 分頁
在 Channel 設定頁面 → 點上方「Basic settings」分頁
2 找到 Your user ID
滑到最下方 → 找到「Your user ID」欄位 → 以 U 開頭的一串字（約 33 個字元）
3 複製 User ID
點複製按鈕 → 貼到記事本備用
小知識：User ID 格式為 U + 32 個十六進位字元，例如 U1234567890abcdef1234567890abcdef。 這個 ID 是你自己的，推播時就是傳訊息給你自己的 LINE。

目前你應該有兩個值

名稱長度用途
Channel Access Token約 170 字元程式的「通行證」，證明有權呼叫 API
Your user ID33 字元（U 開頭）指定推播的「收件人」

⚙️ 步驟五：設定程式碼

把 Token 和 User ID 填入程式碼的設定區。

// ===== 設定區（請填入你自己的值） =====

var LINE_USER_ID = '你的 User ID 貼在這裡';
var LINE_GROUP_ID = '你的 Group ID 貼在這裡（不需要群組通知就留空白）';

1 建立新的 GAS 專案
到 Google 試算表 → 擴充功能 → Apps Script → 清空預設程式碼
2 貼上程式碼
把下載的 .gs 檔案內容全部貼上
3 填入你的 Token 和 User ID

常見錯誤：Token 前後不要有多餘空格，也不要不小心刪掉引號 ' '！

📤 步驟六：pushText() 推播文字

第一個推播函式 — 傳送一則文字訊息到你的 LINE。 

// ===== 推播文字訊息 =====
function pushText(userId, text) {
  var payload = {
    to: userId,
    messages: [{
      type: 'text',
      text: text
    }]
  };

  UrlFetchApp.fetch(LINE_PUSH_URL, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify(payload)
  });
}

// ===== 測試推播 =====
function testPush() {
  pushText(LINE_USER_ID, '🎉 恭喜！你的 Line Bot 推播成功了！');
  Logger.log('✅ 已推播文字訊息');
}

程式碼解說

  • payload — 要傳送的資料，包含「傳給誰」和「訊息內容」
  • to: userId — 指定收件人的 User ID
  • messages — 訊息陣列，可以放多則訊息
  • type: 'text' — 訊息類型是文字
  • UrlFetchApp.fetch() — GAS 發送 HTTP 請求到 LINE API
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 用 Token 做身分驗證

🖼️ 步驟七：pushImage() 推播圖片

除了文字，也可以推播圖片到 LINE！ 

// ===== 推播圖片訊息 =====
function pushImage(userId, imageUrl) {
  var payload = {
    to: userId,
    messages: [{
      type: 'image',
      originalContentUrl: imageUrl,   // 原始圖片網址
      previewImageUrl: imageUrl       // 預覽圖網址（可以跟原圖一樣）
    }]
  };

  UrlFetchApp.fetch(LINE_PUSH_URL, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify(payload)
  });
}

// ===== 測試推播圖片 =====
function testPushImage() {
  var url = 'https://developers.line.biz/assets/img/logo/line-developers-logo.png';
  pushImage(LINE_USER_ID, url);
  Logger.log('✅ 已推播圖片');
}
圖片網址要求：
1. 必須是 https:// 開頭（不支援 http）
2. 必須是公開可存取的網址
3. 支援格式：JPEG、PNG
4. 圖片大小上限：10 MB

📦 步驟八：pushLine() 多則訊息

一次推播多則訊息（最多 5 則），可以混合文字和圖片。 

// ===== 推播多則訊息（通用版） =====
function pushLine(userId, messages) {
  var payload = {
    to: userId,
    messages: messages    // 直接傳入訊息陣列
  };

  UrlFetchApp.fetch(LINE_PUSH_URL, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify(payload)
  });
}

// ===== 測試多則訊息 =====
function testMultiMessage() {
  var messages = [
    { type: 'text', text: '📊 今日報告' },
    { type: 'text', text: '✅ 系統運作正常\n📈 資料已更新 50 筆\n⏰ 下次更新：明天 09:00' },
    { type: 'text', text: '以上是自動推播，如有問題請回覆！' }
  ];
  pushLine(LINE_USER_ID, messages);
  Logger.log('✅ 已推播多則訊息');
}
注意：LINE 每次推播最多 5 則訊息。如果超過 5 則會出現錯誤。文字中可以用 \n 換行來節省訊息數量。

💬 步驟九：回覆函式（預覽）

回覆函式用於 Webhook 場景 — 使用者傳訊息時自動回覆。先了解結構，下一個實作會用到。 

// ===== 回覆文字訊息 =====
function replyText(replyToken, text) {
  var payload = {
    replyToken: replyToken,    // 用 replyToken 取代 userId
    messages: [{
      type: 'text',
      text: text
    }]
  };

  UrlFetchApp.fetch(LINE_REPLY_URL, {   // 注意：用 REPLY_URL
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify(payload)
  });
}

// ===== 回覆多則訊息 =====
function replyLine(replyToken, messages) {
  var payload = {
    replyToken: replyToken,
    messages: messages
  };

  UrlFetchApp.fetch(LINE_REPLY_URL, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify(payload)
  });
}

📤 推播（Push）

  • 用 userId 指定收件人
  • 用 LINE_PUSH_URL
  • 可以隨時主動發送
  • 免費方案每月有額度限制

💬 回覆（Reply）

  • 用 replyToken（一次性）
  • 用 LINE_REPLY_URL
  • 只能在收到訊息後回覆
  • 回覆不計入額度

❓ 常見問題排解

問題原因解決方法
❌ 401 Unauthorized Token 錯誤或過期 重新複製 Token，確認沒有多餘空格
❌ 400 Bad Request User ID 格式錯誤 確認 User ID 以 U 開頭且為 33 字元
❌ 沒收到訊息 沒加 Bot 好友 用手機 LINE 掃 QR Code 加好友
❌ 圖片傳送失敗 圖片網址不是 https 確認圖片 URL 以 https:// 開頭
❌ Exception: Request failed 網址拼錯或無法連線 確認 LINE_PUSH_URL 正確，網路正常
❌ 訊息超過 5 則 messages 陣列超過 5 個 每次推播最多 5 則，可分批發送
除錯技巧：如果推播失敗，在 UrlFetchApp.fetch() 前加上 Logger.log(JSON.stringify(payload))，可以在執行紀錄中看到實際傳送的資料。
