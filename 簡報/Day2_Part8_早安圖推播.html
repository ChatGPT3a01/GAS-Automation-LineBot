🔍 程式碼解析 1 — Line Bot 推播函式

這段的目的：提供 3 個推播工具函式，讓後面的程式碼可以輕鬆送出文字或圖片到 Line。

📁 pushText(to, text) — 推播文字訊息

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}
  • to — 要推播給誰（就是 LINE_USER_ID）
  • text — 要傳送的文字內容
  • 它其實是呼叫下面的 pushLine()，把文字包成 Line 要求的格式

📁 pushImage(to, imageUrl) — 推播圖片訊息

function pushImage(to, imageUrl) {
  pushLine(to, [{
    type: 'image',
    originalContentUrl: imageUrl,
    previewImageUrl: imageUrl
  }]);
}
  • imageUrl — 圖片的公開網址（必須是 https:// 開頭，且可公開存取）
  • originalContentUrl — 點開圖片時顯示的原圖
  • previewImageUrl — 聊天室中的縮圖（這裡跟原圖用同一個網址）

📁 pushLine(to, messages) — 推播底層函式

function pushLine(to, messages) {
  var url = 'https://api.line.me/v2/bot/message/push';
  var options = {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      to: to,
      messages: messages
    }),
    muteHttpExceptions: true
  };

  var response = UrlFetchApp.fetch(url, options);
  var code = response.getResponseCode();

  if (code !== 200) {
    Logger.log('推播失敗，狀態碼：' + code);
    Logger.log('錯誤內容：' + response.getContentText());
  } else {
    Logger.log('推播成功！');
  }
}
  • UrlFetchApp.fetch() — GAS 內建的 HTTP 請求工具，等同於瀏覽器發送網路請求
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 告訴 Line「我是哪一個 Bot」的身份驗證
  • muteHttpExceptions: true — 即使發生錯誤也不要中斷程式，讓我們自己處理錯誤
  • getResponseCode() — 取得回應的狀態碼，200 表示成功

🔍 程式碼解析 2 — DriveApp 操作

這段的目的：找到 Google Drive 上存放早安圖的資料夾，然後列出裡面的所有圖片檔案。

📁 getImagesFromFolder(folderId)

function getImagesFromFolder(folderId) {
  var folder = DriveApp.getFolderById(folderId);
  var files = folder.getFiles();
  var images = [];

  while (files.hasNext()) {
    var file = files.next();
    var mimeType = file.getMimeType();

    // 只取圖片檔案（jpg、png、gif、webp）
    if (mimeType.indexOf('image/') === 0) {
      images.push(file);
      Logger.log('找到圖片：' + file.getName() + '（' + mimeType + '）');
    }
  }

  Logger.log('共找到 ' + images.length + ' 張圖片');
  return images;
}

逐行解釋：

  • DriveApp.getFolderById(folderId) — GAS 內建工具，用資料夾 ID 找到對應的資料夾物件
  • folder.getFiles() — 取出資料夾中所有檔案（不論類型），回傳一個「迭代器」（FileIterator）
  • while (files.hasNext()) — 逐一檢查每個檔案，hasNext() 表示「還有下一個嗎？」
  • file.next() — 取得下一個檔案
  • file.getMimeType() — 取得檔案類型，圖片的開頭都是 image/（例如 image/jpeg、image/png）
  • mimeType.indexOf('image/') === 0 — 檢查 MIME Type 是否以 image/ 開頭（索引位置 0 就是開頭）
  • images.push(file) — 確認是圖片後，把它放進 images 陣列中
  • Logger.log() — 記錄資訊到執行記錄，方便除錯
💡 什麼是 MIME Type？
MIME Type 是檔案格式的標準標記。常見圖片類型：image/jpeg（JPG）、image/png（PNG）、image/gif（GIF）、image/webp（WebP）。用 indexOf('image/') === 0 可以一次過濾出所有圖片格式。

🎲 程式碼解析 3 — 隨機選取

這段的目的：從收集到的多張圖片中，隨機挑選一張出來。

📁 pickRandomImage(images)

function pickRandomImage(images) {
  var index = Math.floor(Math.random() * images.length);
  var selected = images[index];
  Logger.log('隨機選中：' + selected.getName());
  return selected;
}

逐行解釋：

  • Math.random() — 產生一個 0 ~ 0.999... 之間的隨機小數（例如 0.73）
  • Math.random() * images.length — 把隨機小數乘以圖片總數（例如 0.73 * 5 = 3.65）
  • Math.floor() — 向下取整數（例如 3.65 → 3），得到一個陣列索引
  • images[index] — 用索引從陣列中取出那張圖片

圖解運作原理（假設有 5 張圖片）

步驟運算結果
1Math.random()0.73（隨機）
20.73 * 53.65
3Math.floor(3.65)3
4images[3]第 4 張圖（索引從 0 起算）
💡 為什麼是第 4 張？
陣列索引從 0 開始：
images[0] = 第1張、images[1] = 第2張、images[2] = 第3張、images[3] = 第4張、images[4] = 第5張

🔗 程式碼解析 4 — 公開連結

這段的目的：把 Google Drive 上的圖片設為公開，並產生 Line Bot 可以讀取的圖片網址。

📁 getPublicImageUrl(file)

function getPublicImageUrl(file) {
  // 設定為「任何人皆可檢視」
  file.setSharing(DriveApp.Access.ANYONE_WITH_LINK, DriveApp.Permission.VIEW);

  // 取得檔案 ID
  var fileId = file.getId();

  // 組合直接存取的圖片網址
  // 這個網址格式可以讓 Line Bot 直接顯示圖片
  var imageUrl = 'https://lh3.googleusercontent.com/d/' + fileId;

  Logger.log('圖片公開網址：' + imageUrl);
  return imageUrl;
}

逐行解釋：

  • file.setSharing(Access, Permission) — 設定檔案的分享權限
  • DriveApp.Access.ANYONE_WITH_LINK — 「知道連結的人」都可以存取
  • DriveApp.Permission.VIEW — 只能「檢視」（不能編輯或刪除）
  • file.getId() — 取得檔案的唯一 ID
  • 'https://lh3.googleusercontent.com/d/' + fileId — 組合出 Google 圖片串流伺服器的網址
⚠️ 為什麼要設公開？
Line 伺服器需要透過 URL 下載圖片才能傳給使用者。如果圖片是私密的，Line 伺服器會拿到 403 權限錯誤，圖片就無法顯示。

Google Drive 圖片 URL 格式比較

URL 格式 用途 Line Bot 可用？
drive.google.com/file/d/{id}/view 網頁預覽（HTML 頁面） ✘ 不可用
drive.google.com/uc?id={id} 直接下載連結 △ 可能失效（會重新導向）
lh3.googleusercontent.com/d/{id} 直接圖片串流 ✔ 推薦使用

📋 程式碼解析 5 — 主函式 sendMorningImage()

這段的目的：這是排程會自動執行的主函式，把前面所有函式串起來，完成「選圖 → 公開 → 推播」的完整流程。

function sendMorningImage() {
  Logger.log('===== 開始發送早安圖 =====');

  // 步驟 1：取得資料夾中的所有圖片
  var images = getImagesFromFolder(MORNING_FOLDER_ID);

  if (images.length === 0) {
    Logger.log('資料夾中沒有圖片！');
    pushText(LINE_USER_ID, '⚠️ 早安圖資料夾中沒有圖片，請上傳一些圖片。');
    return;
  }

  // 步驟 2：隨機挑選一張
  var selectedImage = pickRandomImage(images);

  // 步驟 3：取得公開連結
  var imageUrl = getPublicImageUrl(selectedImage);

  // 步驟 4：推播圖片到 Line
  pushImage(LINE_USER_ID, imageUrl);

  // 步驟 5：附帶早安文字訊息
  var now = new Date();
  var dateStr = Utilities.formatDate(now, 'Asia/Taipei', 'yyyy/MM/dd (EEE)');
  var greetings = [
    '🌅 早安！新的一天，新的開始！',
    '☀️ 早安！今天也要元氣滿滿喔！',
    '🌸 早安！美好的一天從微笑開始！',
    '🌈 早安！願你今天一切順利！',
    '💪 早安！今天也要加油喔！'
  ];
  var greeting = greetings[Math.floor(Math.random() * greetings.length)];

  pushText(LINE_USER_ID, greeting + '\n\n📅 ' + dateStr + '\n📸 今日早安圖：' + selectedImage.getName());

  Logger.log('===== 早安圖發送完成 =====');
}

⏰ 排程進階技巧

除了用介面設定觸發條件，你還可以透過程式碼來更精確地控制排程。（進階選讀，可跳過）

用程式碼設定觸發條件

// 建立每日觸發條件（可以指定更精確的時間）
function createDailyTrigger() {
  // 先清除舊的觸發條件（避免重複）
  var triggers = ScriptApp.getProjectTriggers();
  for (var i = 0; i < triggers.length; i++) {
    if (triggers[i].getHandlerFunction() === 'sendMorningImage') {
      ScriptApp.deleteTrigger(triggers[i]);
    }
  }

  // 建立新的每日觸發條件
  ScriptApp.newTrigger('sendMorningImage')
    .timeBased()
    .atHour(7)          // 早上 7 點
    .everyDays(1)       // 每天
    .create();

  Logger.log('✅ 觸發條件已建立：每天早上 7 點');
}

其他排程選項

方法效果
.everyMinutes(30)每 30 分鐘
.everyHours(1)每小時
.everyDays(1)每天
.onWeekDay(ScriptApp.WeekDay.MONDAY)每週一

實用延伸

  • 週末不發送 — 在 sendMorningImage 開頭加星期判斷
  • 多時段發送 — 建立多個觸發條件
  • 節日特別圖 — 建立不同資料夾，依日期切換
💡 提示：執行 createDailyTrigger() 一次就好，它會自動清除舊的再建新的，不會重複觸發。

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？（第一步！）

  1. 在 Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
  2. 會出現執行歷史記錄列表，每次執行都會留下一筆記錄
  3. 找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
  4. 展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

圖片無法顯示

  • 確認圖片已設為公開（程式碼會自動處理，但手動檢查：右鍵圖片 → 共用 → 確認「知道連結的人」可檢視）
  • 在瀏覽器貼上 https://lh3.googleusercontent.com/d/你的檔案ID 測試能否看到圖片
  • 確認圖片格式為 JPG 或 PNG，大小在 1MB 以內

收不到 Line 推播

  • 先跑 testPush 函式（只推播文字），確認 Token 和 ID 是否正確
  • 確認 Line Bot 已加為好友（用手機掃 QR Code）
  • 檢查執行記錄是否有「推播失敗，狀態碼：401」→ Token 不正確
  • 免費帳號每月推播有次數上限（500 則）

「找不到資料夾」錯誤

  • 錯誤訊息通常是 Folder not found 或 Invalid argument: id
  • 回到 Google Drive 開啟「早安圖」資料夾，重新從網址列複製 ID
  • 確認複製的是 /folders/ 後面那段，不要包含斜線
  • 確認資料夾沒有被刪除或移到垃圾桶

排程沒有觸發

  • 回到「觸發條件」頁面（左側鬧鐘圖示），確認列表中有一筆記錄
  • 如果記錄旁邊有紅色驚嘆號，表示上次排程執行失敗 → 先手動跑 testSendMorningImage 確認程式碼沒有問題
  • 排程設定的時區以你的 Google 帳號設定為準（通常是台灣時間 GMT+8）
