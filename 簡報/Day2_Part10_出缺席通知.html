🔍 程式碼解析 1 — 試算表資料讀取

這段的目的：初始化試算表工作表、新增出缺席記錄。如果工作表不存在就自動建立。

📁 initSheet() — 初始化工作表

function initSheet() {
  var ss = SpreadsheetApp.openById(SPREADSHEET_ID);
  var sheet = ss.getSheetByName(SHEET_NAME);

  if (!sheet) {
    sheet = ss.insertSheet(SHEET_NAME);
    // 設定標題列
    var headers = ['姓名', '日期', '狀態', '備註'];
    sheet.getRange(1, 1, 1, headers.length).setValues([headers]);
    sheet.getRange(1, 1, 1, headers.length)
      .setFontWeight('bold')
      .setBackground('#1b5e20')
      .setFontColor('white');
    sheet.setFrozenRows(1);

    Logger.log('已建立工作表：' + SHEET_NAME);
  }

  return sheet;
}

逐行解釋：

  • SpreadsheetApp.openById(SPREADSHEET_ID) — 用試算表 ID 開啟試算表
  • ss.getSheetByName(SHEET_NAME) — 在試算表中找到名為「出缺席記錄」的工作表
  • if (!sheet) — 如果找不到這個工作表（第一次執行時不存在）
  • ss.insertSheet(SHEET_NAME) — 自動建立新工作表
  • sheet.getRange(1, 1, 1, headers.length).setValues([headers]) — 在第 1 列寫入標題（姓名、日期、狀態、備註）
  • .setFontWeight('bold').setBackground('#1b5e20').setFontColor('white') — 把標題列設為粗體、深綠底色、白色字
  • sheet.setFrozenRows(1) — 凍結第 1 列（捲動時標題列不會跑掉）

📁 addRecord(name, status, note) — 新增記錄

function addRecord(name, status, note) {
  var sheet = initSheet();
  var today = Utilities.formatDate(new Date(), 'Asia/Taipei', 'yyyy/MM/dd');
  sheet.appendRow([name, today, status, note || '']);
  Logger.log('已新增記錄：' + name + ' - ' + status);
}
  • initSheet() — 先確保工作表存在
  • Utilities.formatDate(new Date(), 'Asia/Taipei', 'yyyy/MM/dd') — 取得今天日期，格式為 2026/03/09
  • sheet.appendRow() — 在試算表最後一列新增一筆記錄
  • note || '' — 如果沒有備註，就填空字串

🔍 程式碼解析 2 — 出缺席統計

這段的目的：從所有記錄中篩選出今天的出缺席，並分類為出席、缺席、請假三組。

📁 getTodayAttendance() — 取得今日出缺席

function getTodayAttendance() {
  var sheet = initSheet();
  var data = sheet.getDataRange().getValues();
  var today = Utilities.formatDate(new Date(), 'Asia/Taipei', 'yyyy/MM/dd');

  var result = {
    present: [],  // 出席
    absent: [],   // 缺席
    leave: [],    // 請假
    total: 0
  };

  // 從第 2 列開始（跳過標題列）
  for (var i = 1; i < data.length; i++) {
    var name = data[i][0];
    var date = data[i][1];
    var status = data[i][2];
    var note = data[i][3];

    // 日期比對（處理日期格式）
    var recordDate;
    if (date instanceof Date) {
      recordDate = Utilities.formatDate(date, 'Asia/Taipei', 'yyyy/MM/dd');
    } else {
      recordDate = String(date);
    }

    if (recordDate === today) {
      result.total++;
      if (status === '出席') {
        result.present.push(name);
      } else if (status === '缺席') {
        result.absent.push({ name: name, note: note });
      } else if (status === '請假') {
        result.leave.push({ name: name, note: note });
      }
    }
  }

  return result;
}

逐行解釋：

  • sheet.getDataRange().getValues() — 一次讀取工作表中所有有資料的儲存格，回傳二維陣列
  • data[i][0] 是第 i 列的 A 欄（姓名），data[i][1] 是 B 欄（日期），data[i][2] 是 C 欄（狀態），data[i][3] 是 D 欄（備註）
  • if (date instanceof Date) — 試算表會自動把日期格式的文字轉成 Date 物件，需要先判斷再統一格式
  • result.present.push(name) — 出席的人只記錄姓名
  • result.absent.push({ name: name, note: note }) — 缺席的人記錄姓名和備註（因為通知需要顯示原因）

📁 getAbsenceCount() — 統計每人累計缺席次數

function getAbsenceCount() {
  var sheet = initSheet();
  var data = sheet.getDataRange().getValues();
  var counts = {};

  for (var i = 1; i < data.length; i++) {
    var name = data[i][0];
    var status = data[i][2];

    if (status === '缺席') {
      counts[name] = (counts[name] || 0) + 1;
    }
  }

  return counts;
}
  • var counts = {} — 建立空物件，用來存每個人的缺席次數
  • counts[name] = (counts[name] || 0) + 1 — 如果這個人之前沒有記錄就從 0 開始，然後 +1
  • 這個函式會掃描所有日期的缺席記錄，不只是今天

🔍 程式碼解析 3 — 推播訊息組合

這段的目的：把出缺席統計資料組合成一則格式化的通知訊息。如果全員出席就回傳 null（不推播）。

📁 buildAttendanceMessage() — 組合通知訊息

function buildAttendanceMessage() {
  var attendance = getTodayAttendance();
  var today = Utilities.formatDate(new Date(), 'Asia/Taipei', 'yyyy/MM/dd (EEE)');

  // 情況 1：今天沒有任何記錄 → 不推播
  if (attendance.total === 0) {
    Logger.log('今日沒有出缺席記錄');
    return null;
  }

  // 情況 2：全員出席（沒有缺席也沒有請假）→ 不推播
  if (attendance.absent.length === 0 && attendance.leave.length === 0) {
    Logger.log('今日全員出席，不需推播');
    return null;
  }

  var absenceCounts = getAbsenceCount();

  var message = '📋 出缺席通知\n';
  message += '📅 ' + today + '\n';
  message += '━━━━━━━━━━━━━━━\n';

  // 統計資訊
  var attendanceRate = Math.round(
    attendance.present.length / attendance.total * 100
  );
  message += '\n📊 今日統計：\n';
  message += '  ✅ 出席：' + attendance.present.length + ' 人\n';
  message += '  ❌ 缺席：' + attendance.absent.length + ' 人\n';
  message += '  📝 請假：' + attendance.leave.length + ' 人\n';
  message += '  📈 出席率：' + attendanceRate + '%\n';

  // 缺席名單
  if (attendance.absent.length > 0) {
    message += '\n❌ 缺席名單：\n';
    for (var i = 0; i < attendance.absent.length; i++) {
      var item = attendance.absent[i];
      var count = absenceCounts[item.name] || 1;
      message += '  • ' + item.name + '（累計 ' + count + ' 次）';
      if (item.note) message += '（' + item.note + '）';
      message += '\n';
    }
  }

  // 請假名單
  if (attendance.leave.length > 0) {
    message += '\n📝 請假名單：\n';
    for (var j = 0; j < attendance.leave.length; j++) {
      var leaveItem = attendance.leave[j];
      message += '  • ' + leaveItem.name;
      if (leaveItem.note) message += '（' + leaveItem.note + '）';
      message += '\n';
    }
  }

  message += '\n━━━━━━━━━━━━━━━';
  message += '\n💡 由 GAS 自動推播';

  return message;
}

關鍵邏輯說明：

  • 回傳 null 的兩種情況：今天完全沒有記錄，或全員出席且沒有請假。主函式收到 null 就不推播
  • Math.round(出席人數 / 總人數 * 100) — 計算出席率百分比並四捨五入
  • absenceCounts[item.name] || 1 — 查詢該學生的累計缺席次數，如果查不到就預設為 1 次
  • 'yyyy/MM/dd (EEE)' — 日期格式帶星期，例如 2026/03/09 (Mon)

🔍 程式碼解析 4 — 主函式流程

這段的目的：這是排程會自動執行的主函式，串起「組合訊息 → 判斷是否推播 → 送出」的完整流程。

📁 sendAttendanceReport() — 主函式

function sendAttendanceReport() {
  Logger.log('===== 開始檢查出缺席 =====');

  var message = buildAttendanceMessage();

  if (message) {
    // 有缺席或請假，推播通知
    pushText(PUSH_TARGET, message);
    Logger.log('已推播出缺席通知');
  } else {
    Logger.log('全員出席或無記錄，不推播');
  }

  Logger.log('===== 出缺席檢查完成 =====');
}

逐行解釋：

  • buildAttendanceMessage() — 組合訊息，如果全員出席會回傳 null
  • if (message) — 在 JavaScript 中，null 會被當作 false，非空字串會被當作 true
  • pushText(PUSH_TARGET, message) — 把訊息推播到 PUSH_TARGET（個人或群組）

📁 pushText() 和 pushLine() — Line Bot 推播函式

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}

function pushLine(to, messages) {
  var url = 'https://api.line.me/v2/bot/message/push';
  var options = {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      to: to,
      messages: messages
    }),
    muteHttpExceptions: true
  };

  var response = UrlFetchApp.fetch(url, options);
  var code = response.getResponseCode();

  if (code !== 200) {
    Logger.log('推播失敗，狀態碼：' + code);
    Logger.log('錯誤內容：' + response.getContentText());
  } else {
    Logger.log('推播成功！（目標：' + to.substring(0, 8) + '...）');
  }
}
  • pushText() 是方便函式，它呼叫 pushLine() 並把文字包成 Line 要求的格式
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 告訴 Line「我是哪一個 Bot」的身份驗證
  • to.substring(0, 8) + '...' — 日誌中只顯示 ID 前 8 碼，保護隱私
  • muteHttpExceptions: true — 即使發生錯誤也不要中斷程式，讓我們自己處理錯誤

❓ 常見問題排解（強化版）

🔍 怎麼找到錯誤訊息？（第一步！）

📍 位置：Apps Script 編輯器左側選單
👆 動作：找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
👀 結果：會出現執行歷史記錄列表，每次執行都會留下一筆記錄
👆 動作：找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
👀 結果：展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

「Spreadsheet not found」或「Invalid argument: id」

  • 試算表 ID 貼錯了
  • 回到試算表網址列，重新複製 /d/ 和 /edit 之間的那串
  • 確認前後沒有空格、沒有斜線
  • 確認試算表沒有被刪除或移到垃圾桶

「推播失敗，狀態碼：401」

  • Channel Access Token 不正確
  • 去 LINE Developers → 你的 Bot → Messaging API 頁籤
  • 重新點擊 Issue 按鈕產生新的 Token
  • 注意：新 Token 產生後，舊的會立刻失效

「推播失敗，狀態碼：400」

  • User ID 或 Group ID 格式錯誤
  • User ID 應該是 U 開頭（約 33 字元）
  • Group ID 應該是 C 開頭（約 33 字元）
  • 確認沒有多複製空格或換行符號

「全員出席或無記錄，不推播」

  • 這不是錯誤！是正常的條件式觸發
  • 表示今天的記錄中沒有「缺席」或「請假」
  • 確認步驟 5 的 testAddSampleData 有成功執行
  • 確認試算表中的日期是今天的日期

排程沒有觸發

  • 回到「觸發條件」頁面（左側鬧鐘圖示），確認列表中有一筆記錄
  • 如果記錄旁邊有紅色驚嘆號，表示上次排程執行失敗
  • 先手動跑 testSendReport 確認程式碼沒問題
  • 排程設定的時區以你的 Google 帳號設定為準（通常是 GMT+8）

想推播到群組但不知道群組 ID

  • Group ID 無法從 LINE Developers 後台取得
  • 使用「訊息收集器」（實作 2）：Bot 加入群組 → 群組中發送訊息 → 從收集器試算表找到 C 開頭的 groupId
  • 取得後把 PUSH_TARGET 改成：
    var PUSH_TARGET = 'Cab12cd34...';
  • 記得先到 LINE Developers → Messaging API → 開啟「Allow bot to join group chats」
