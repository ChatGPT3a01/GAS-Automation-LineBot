🔍 程式碼解析 1 — 取得天氣資料

這段的目的：組合 API 網址，呼叫中央氣象署的 API，取回指定縣市的天氣預報原始資料。

📁 getWeatherData(cityName)

function getWeatherData(cityName) {
  // 組合 API 網址
  var url = 'https://opendata.cwa.gov.tw/api/v1/rest/datastore/' + DATASET_ID;
  url += '?Authorization=' + CWA_API_KEY;
  url += '&locationName=' + encodeURIComponent(cityName);

  try {
    var response = UrlFetchApp.fetch(url, { muteHttpExceptions: true });
    var code = response.getResponseCode();

    if (code !== 200) {
      Logger.log('API 呼叫失敗，狀態碼：' + code);
      return null;
    }

    var json = JSON.parse(response.getContentText());

    if (json.success !== 'true') {
      Logger.log('API 回應失敗：' + JSON.stringify(json));
      return null;
    }

    var locations = json.records.location;
    if (!locations || locations.length === 0) {
      Logger.log('找不到 ' + cityName + ' 的資料');
      return null;
    }

    return locations[0];
  } catch (error) {
    Logger.log('getWeatherData 錯誤：' + error.toString());
    return null;
  }
}

逐行解釋：

  • DATASET_ID = 'F-C0032-001' — 氣象署的資料集代碼，代表「36小時天氣預報」
  • encodeURIComponent(cityName) — 把中文縣市名稱轉成 URL 安全的編碼格式（例如「臺北市」→ %E8%87%BA%E5%8C%97%E5%B8%82）
  • UrlFetchApp.fetch(url, options) — GAS 內建的 HTTP 請求工具，用來呼叫外部 API
  • muteHttpExceptions: true — 即使 API 回傳錯誤（如 401、404），程式也不會中斷
  • getResponseCode() — 取得回應狀態碼，200 表示成功
  • JSON.parse() — 把 API 回傳的 JSON 字串轉成 JavaScript 物件
  • json.records.location[0] — 層層取值，取出指定縣市的天氣資料
  • try...catch — 錯誤處理，如果過程中發生任何錯誤，會記錄到 Logger 而不會讓程式崩潰

🔍 程式碼解析 2 — 解析天氣資料

這段的目的：從 API 回傳的原始資料中，提取出我們需要的天氣元素（天氣狀況、溫度、降雨機率等）。

📁 parseWeatherData(locationData)

function parseWeatherData(locationData) {
  var elements = locationData.weatherElement;
  var forecasts = [];

  // 找出各個天氣元素
  var wxData = findElement(elements, 'Wx');      // 天氣現象
  var popData = findElement(elements, 'PoP');     // 降雨機率
  var minTData = findElement(elements, 'MinT');   // 最低溫
  var maxTData = findElement(elements, 'MaxT');   // 最高溫
  var ciData = findElement(elements, 'CI');       // 舒適度

  var timeCount = wxData ? wxData.time.length : 0;

  for (var i = 0; i < timeCount; i++) {
    var forecast = {
      startTime: wxData.time[i].startTime,
      endTime: wxData.time[i].endTime,
      weather: wxData ? wxData.time[i].parameter.parameterName : '未知',
      rainProb: popData ? popData.time[i].parameter.parameterName : '未知',
      minTemp: minTData ? minTData.time[i].parameter.parameterName : '未知',
      maxTemp: maxTData ? maxTData.time[i].parameter.parameterName : '未知',
      comfort: ciData ? ciData.time[i].parameter.parameterName : '未知'
    };
    forecasts.push(forecast);
  }

  return forecasts;
}

API 回傳的 5 個天氣元素

代碼名稱說明範例值
Wx天氣現象天氣狀況的文字描述多雲時晴
PoP降雨機率百分比數值30
MinT最低溫度攝氏溫度18
MaxT最高溫度攝氏溫度25
CI舒適度體感描述舒適

📁 findElement() 輔助函式

function findElement(elements, elementName) {
  for (var i = 0; i < elements.length; i++) {
    if (elements[i].elementName === elementName) {
      return elements[i];
    }
  }
  return null;
}
💡 為什麼需要 findElement()？
API 回傳的 weatherElement 是一個陣列，裡面有 5 個元素，但順序不一定固定。所以我們用 findElement() 透過名稱來找到正確的元素，而不是靠索引位置。

🔍 程式碼解析 3 — 格式化天氣報告

這段的目的：把天氣資料組合成一段好看、好讀的文字訊息，準備推送到 Line。

📁 formatWeatherReport(cityName, forecasts)

function formatWeatherReport(cityName, forecasts) {
  var now = new Date().toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' });

  var report = '🌤 天氣預報 — ' + cityName + '\n';
  report += '━━━━━━━━━━━━━━\n';
  report += '📅 ' + now + '\n\n';

  for (var i = 0; i < forecasts.length; i++) {
    var f = forecasts[i];
    var startTime = f.startTime.split(' ')[1].substring(0, 5);
    var endTime = f.endTime.split(' ')[1].substring(0, 5);
    var startDate = f.startTime.split(' ')[0].substring(5);
    var weatherEmoji = getWeatherEmoji(f.weather);

    report += '🕐 ' + startDate + ' ' + startTime + ' ~ ' + endTime + '\n';
    report += '   ' + weatherEmoji + ' ' + f.weather + '\n';
    report += '   🌡 ' + f.minTemp + '°C ~ ' + f.maxTemp + '°C\n';
    report += '   🌧 降雨機率：' + f.rainProb + '%\n';
    report += '   😊 ' + f.comfort + '\n\n';
  }

  // 智慧提醒
  var firstForecast = forecasts[0];
  var rainProb = parseInt(firstForecast.rainProb);
  if (rainProb >= 50) {
    report += '☂️ 提醒：降雨機率較高，記得帶傘！\n';
  }
  var maxTemp = parseInt(firstForecast.maxTemp);
  if (maxTemp >= 35) {
    report += '🥵 提醒：高溫警報，注意防曬補水！\n';
  } else if (parseInt(firstForecast.minTemp) <= 15) {
    report += '🧥 提醒：天氣較涼，注意保暖！\n';
  }

  return report;
}
  • toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' }) — 取得台灣時區的當前時間字串
  • f.startTime.split(' ')[1].substring(0, 5) — 從「2026-03-09 06:00:00」中取出「06:00」
  • parseInt() — 把字串轉成數字，才能用 >= 做比較判斷
  • 智慧提醒邏輯：降雨 >= 50% 提醒帶傘、高溫 >= 35°C 提醒防曬、低溫 <= 15°C 提醒保暖

function getWeatherEmoji(weather) {

  if (weather.indexOf('晴') !== -1 && weather.indexOf('雲') !== -1) return '🌤';

  if (weather.indexOf('雷') !== -1) return '⛈';
  if (weather.indexOf('雪') !== -1) return '🌨';

💡 indexOf() 怎麼用？

📋 程式碼解析 5 — 主函式 sendWeatherReport()

這段的目的：這是排程會自動執行的主函式，把前面所有函式串起來，完成「取得 → 解析 → 格式化 → 推播 → 記錄」的完整流程。

function sendWeatherReport() {
  // 步驟 1：取得天氣資料
  var locationData = getWeatherData(TARGET_CITY);

  if (!locationData) {
    pushText(LINE_USER_ID, '❌ 天氣預報取得失敗\n\n可能原因：\n' +
      '1. API 授權碼不正確\n2. 縣市名稱不正確\n3. API 服務暫時無法使用');
    return;
  }

  // 步驟 2：解析天氣資料
  var forecasts = parseWeatherData(locationData);

  if (forecasts.length === 0) {
    pushText(LINE_USER_ID, '❌ 天氣預報解析失敗：沒有預報資料');
    return;
  }

  // 步驟 3：格式化報告
  var report = formatWeatherReport(TARGET_CITY, forecasts);

  // 步驟 4：推播到 Line
  pushText(LINE_USER_ID, report);

  // 步驟 5：記錄到試算表
  logWeatherToSheet(TARGET_CITY, forecasts);

  Logger.log('天氣預報已推播');
}

🔍 程式碼解析 6 — 記錄到試算表

這段的目的：每次推播天氣預報時，同步將天氣資料寫入試算表，方便日後回顧和分析。

📁 logWeatherToSheet(cityName, forecasts)

function logWeatherToSheet(cityName, forecasts) {
  try {
    var ss = SpreadsheetApp.getActiveSpreadsheet();
    var sheet = ss.getSheetByName('天氣記錄');

    // 如果工作表不存在，自動建立新的
    if (!sheet) {
      sheet = ss.insertSheet('天氣記錄');
      var headers = ['記錄時間', '縣市', '時段', '天氣',
                     '最低溫', '最高溫', '降雨機率', '舒適度'];
      sheet.getRange(1, 1, 1, headers.length).setValues([headers]);
      sheet.getRange(1, 1, 1, headers.length)
        .setFontWeight('bold')
        .setBackground('#0D47A1')
        .setFontColor('#FFFFFF');
      sheet.setFrozenRows(1);
    }

    var now = new Date().toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' });

    for (var i = 0; i < forecasts.length; i++) {
      var f = forecasts[i];
      sheet.appendRow([
        now, cityName, f.startTime + ' ~ ' + f.endTime,
        f.weather, f.minTemp, f.maxTemp, f.rainProb + '%', f.comfort
      ]);
    }
  } catch (error) {
    Logger.log('記錄天氣資料失敗：' + error);
  }
}
  • SpreadsheetApp.getActiveSpreadsheet() — 取得目前綁定的試算表
  • getSheetByName('天氣記錄') — 找名為「天氣記錄」的工作表
  • if (!sheet) — 如果工作表不存在，自動建立一個新的，並設定好表頭樣式
  • setFrozenRows(1) — 凍結第一列（表頭），捲動時表頭不會消失
  • appendRow() — 在工作表最後一行插入一列新資料
💡 自動建立工作表：
你不需要事先手動建立「天氣記錄」工作表！第一次執行時，程式碼會自動幫你建立，並設定好深藍色表頭和白色文字。之後每次推播，天氣資料就會自動追加到這個表的最後一行。

🔍 程式碼解析 7 — Line Bot 推播函式

這段的目的：提供推播工具函式，讓程式碼可以送出文字訊息到 Line。

📁 pushText(to, text) — 推播文字訊息

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}
  • to — 要推播給誰（就是 LINE_USER_ID）
  • text — 要傳送的文字內容（這裡會是格式化好的天氣報告）
  • 它其實是呼叫下面的 pushLine()，把文字包成 Line 要求的格式

📁 pushLine(to, messages) — 推播底層函式

function pushLine(to, messages) {
  UrlFetchApp.fetch('https://api.line.me/v2/bot/message/push', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({ to: to, messages: messages }),
    muteHttpExceptions: true
  });
}
  • UrlFetchApp.fetch() — 發送 HTTP POST 請求到 Line 的推播 API
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 告訴 Line「我是哪一個 Bot」的身份驗證
  • JSON.stringify() — 把 JavaScript 物件轉成 JSON 字串，作為 HTTP 請求的 payload
  • muteHttpExceptions: true — 即使發生錯誤也不要中斷程式
💡 pushText 和 pushLine 的關係：
pushText() 是方便使用的「包裝函式」，它自動把純文字包成 Line 規定的 [{ type: 'text', text: '...' }] 格式，然後呼叫 pushLine() 發送出去。

🚀 進階功能 — 多縣市天氣查詢

程式碼中還提供了一個進階函式，可以同時查詢多個縣市的天氣。（進階選讀，可跳過）

📁 sendMultiCityWeather()

function sendMultiCityWeather() {
  var cities = ['臺北市', '臺中市', '高雄市'];

  for (var i = 0; i < cities.length; i++) {
    var locationData = getWeatherData(cities[i]);
    if (locationData) {
      var forecasts = parseWeatherData(locationData);
      var report = formatWeatherReport(cities[i], forecasts);
      pushText(LINE_USER_ID, report);

      // 避免短時間內送太多訊息
      Utilities.sleep(1000);
    }
  }

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？（第一步！）

  1. 在 Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
  2. 會出現執行歷史記錄列表，每次執行都會留下一筆記錄
  3. 找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
  4. 展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

「API 呼叫失敗，狀態碼：401」

原因：API 授權碼不正確

  • 回到氣象署會員中心，重新複製 API 授權碼
  • 確認沒有多餘的空格或缺漏的字元
  • 確認授權碼格式為 CWA-XXXXXXXX-...

「找不到 XX市 的資料」

原因：縣市名稱不正確

  • 最常見：用了「台」而不是「臺」
  • 確認縣市名稱包含「市」或「縣」
  • 例如：「臺北市」而不是「臺北」或「台北」

Line 完全沒收到訊息

原因：LINE_TOKEN 或 LINE_USER_ID 不正確

  • 回到 LINE Developers 後台重新複製 Token 和 User ID
  • 確認 Line Bot 已加為好友（手機掃 QR Code）
  • 檢查執行記錄中是否有「推播失敗，狀態碼：401」
  • 免費帳號每月推播上限 500 則

排程沒有觸發

原因：觸發條件設定有誤

  • 回到「觸發條件」頁面（左側鬧鐘圖示），確認列表中有一筆記錄
  • 如果記錄旁有紅色驚嘆號，表示上次排程執行失敗
  • 先手動跑 sendWeatherReport 確認程式碼沒問題
  • 排程時區以你的 Google 帳號設定為準（通常是 GMT+8）
💡 除錯技巧：如果不確定是哪裡出問題，先單獨執行 testApiConnection 來確認 API 連線。如果 API 沒問題，再檢查 LINE_TOKEN 和 LINE_USER_ID。分段測試是最有效的除錯方式！
