🔍 程式碼解析 1 — FormApp 自動建表單

這段的目的：用程式碼自動建立一份完整的報名表單，包含欄位設計、確認訊息、連結試算表。

📁 createRegistrationForm()

function createRegistrationForm() {
  var form = FormApp.create(EVENT_NAME + ' — 報名表');

  // 必填欄位
  form.addTextItem().setTitle('姓名').setRequired(true);
  form.addTextItem().setTitle('Email').setRequired(true);
  form.addTextItem().setTitle('聯絡電話').setRequired(true);
  form.addMultipleChoiceItem()
    .setTitle('用餐需求')
    .setChoiceValues(['葷食', '素食', '不需要'])
    .setRequired(true);
  form.addParagraphTextItem().setTitle('備註（選填）');

  // 設定表單
  form.setConfirmationMessage('報名成功！我們會盡快與您聯繫。');
  form.setCollectEmail(false);

  // 建立回覆試算表
  form.setDestination(FormApp.DestinationType.SPREADSHEET,
    SpreadsheetApp.getActiveSpreadsheet().getId());

  Logger.log('表單已建立！');
  Logger.log('表單網址：' + form.getPublishedUrl());
  Logger.log('表單編輯：' + form.getEditUrl());
  Logger.log('表單 ID：' + form.getId());

  // 推播表單連結
  var message = '✅ 報名表單已建立！\n';
  message += '━━━━━━━━━━━━━━━\n';
  message += '📌 活動：' + EVENT_NAME + '\n';
  message += '📝 表單連結：\n' + form.getPublishedUrl() + '\n\n';
  message += '⚠️ 請記得更新程式碼中的 FORM_ID：\n' + form.getId();

  pushText(LINE_USER_ID, message);
}

逐行解釋：

  • FormApp.create(名稱) — 建立一份新的 Google 表單，參數是表單標題
  • form.addTextItem() — 新增一個「簡答」類型的題目
  • form.addMultipleChoiceItem() — 新增一個「單選」類型的題目
  • .setChoiceValues(['葷食', '素食', '不需要']) — 設定單選的選項內容
  • form.addParagraphTextItem() — 新增一個「段落」類型的題目（長文字輸入）
  • .setRequired(true) — 設為必填欄位
  • form.setConfirmationMessage() — 設定提交後的確認訊息
  • form.setDestination() — 將表單回覆連結到指定的試算表
  • form.getPublishedUrl() — 取得表單的填寫網址（給學員用的）
  • form.getId() — 取得表單 ID（要填回 FORM_ID 設定值）

🔍 程式碼解析 2 — onFormSubmit 事件觸發

這段的目的：這是整個系統的核心 — 當有人提交表單時，GAS 自動呼叫這個函式，取得報名資料並推播通知。

function onFormSubmit(e) {
  try {
    Logger.log('===== 收到新報名 =====');

    // 取得提交的資料
    var responses = e.namedValues;
    var timestamp = e.namedValues['時間戳記']
      ? e.namedValues['時間戳記'][0]
      : new Date().toLocaleString('zh-TW');

    // 組合報名資訊
    var info = '';
    var keys = Object.keys(responses);
    for (var i = 0; i < keys.length; i++) {
      var key = keys[i];
      if (key !== '時間戳記') {
        info += '  • ' + key + '：' + responses[key][0] + '\n';
      }
    }

    // 取得目前報名人數
    var currentCount = getCurrentCount();

    // 組合通知訊息
    var message = '🎉 收到新報名！\n';
    message += '━━━━━━━━━━━━━━━\n';
    message += '📌 活動：' + EVENT_NAME + '\n';
    message += '🕐 時間：' + timestamp + '\n\n';
    message += '📝 報名資料：\n' + info + '\n';
    message += '📊 目前報名人數：' + currentCount;

    if (MAX_PARTICIPANTS > 0) {
      message += ' / ' + MAX_PARTICIPANTS;
      var remaining = MAX_PARTICIPANTS - currentCount;
      message += '\n📋 剩餘名額：' + remaining + ' 位';
    }

    // 推播通知管理員
    pushText(LINE_USER_ID, message);

    // 檢查是否額滿
    if (MAX_PARTICIPANTS > 0 && currentCount >= MAX_PARTICIPANTS) {
      closeForm();
    }

    Logger.log('===== 報名處理完成 =====');

  } catch (error) {
    Logger.log('onFormSubmit 錯誤：' + error.message);
    pushText(LINE_USER_ID, '⚠️ 報名系統發生錯誤：' + error.message);
  }
}
💡 e.namedValues 的格式：
e 是表單提交的事件物件，e.namedValues 是一個物件，key 是欄位名稱，value 是一個陣列： 
{ '姓名': ['王小明'], 'Email': ['test@gmail.com'], '聯絡電話': ['0912345678'], ... }

注意：每個值都是陣列，所以要用 [0] 取第一個元素。

🔍 程式碼解析 3 — 報名人數統計 + 額滿關閉

這段的目的：計算目前報名人數，以及達到上限時自動關閉表單。

📁 getCurrentCount() — 取得目前報名人數

function getCurrentCount() {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheets()[0];
  var lastRow = sheet.getLastRow();
  // 扣掉標題列
  var count = lastRow > 0 ? lastRow - 1 : 0;
  Logger.log('目前報名人數：' + count);
  return count;
}
  • SpreadsheetApp.getActiveSpreadsheet() — 取得目前這個試算表
  • .getSheets()[0] — 取得第一個工作表（表單回覆會存在這裡）
  • sheet.getLastRow() — 取得最後一列的列號（例如有 6 列資料就回傳 6）
  • lastRow - 1 — 扣掉標題列，就是實際報名人數

📁 closeForm() — 關閉表單

function closeForm() {

    var form = FormApp.openById(FORM_ID);

🔍 程式碼解析 4 — Line Bot 通知組合

這段的目的：提供推播工具函式，讓前面的程式碼可以輕鬆送出文字到 Line。另外還有重新開啟表單和報名統計報告功能。

📁 pushText(to, text) + pushLine(to, messages)

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}

function pushLine(to, messages) {
  var url = 'https://api.line.me/v2/bot/message/push';
  var options = {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      to: to,
      messages: messages
    }),
    muteHttpExceptions: true
  };

  var response = UrlFetchApp.fetch(url, options);
  var code = response.getResponseCode();

  if (code !== 200) {
    Logger.log('推播失敗，狀態碼：' + code);
    Logger.log('錯誤內容：' + response.getContentText());
  } else {
    Logger.log('推播成功！');
  }
}
  • pushText(to, text) — 推播文字訊息的便利函式，注意有兩個參數：接收者和文字內容
  • pushLine(to, messages) — 推播底層函式，使用 LINE Messaging API 的 push 端點
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 告訴 Line「我是哪一個 Bot」的身份驗證
  • muteHttpExceptions: true — 即使發生錯誤也不要中斷程式
  • getResponseCode() — 取得回應的狀態碼，200 表示成功

📁 openForm() — 重新開啟表單 + sendRegistrationReport() — 報名統計報告

❓ 常見問題排解（強化版）

🔍 怎麼找到錯誤訊息？（第一步！）

📍 位置：Apps Script 編輯器左側選單
👆 動作：找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
👀 結果：會出現執行歷史記錄列表，每次執行都會留下一筆記錄
👆 動作：找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
👀 結果：展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

填完表單但 Line 沒收到通知

  • 最常見原因：GAS 專案建在表單裡而非試算表裡
  • 確認觸發條件是「從試算表 → 提交表單時」
  • 先跑 testPush 函式，確認 Token 和 ID 是否正確
  • 到「執行項目」查看是否有 onFormSubmit 的執行記錄
  • 如果完全沒有執行記錄 → 觸發條件沒有設對

推播失敗，狀態碼 401

  • 表示 Channel Access Token 不正確
  • 回到 LINE Developers 重新複製 Token
  • 確認 Token 前後沒有多餘的空格或換行
  • 確認單引號沒有被刪掉

「FORM_ID」相關錯誤

  • 錯誤訊息含 Form not found 或 Invalid argument
  • 確認你有執行過 createRegistrationForm()
  • 確認 FORM_ID 已經從執行記錄中複製並貼上
  • 確認是表單 ID，不是試算表 ID

額滿沒有自動關閉

  • 確認 FORM_ID 已填入正確的表單 ID
  • 確認 MAX_PARTICIPANTS 設定值正確
  • 到「執行項目」查看 onFormSubmit 是否有執行並出錯
  • 可能是 closeForm() 的權限不足 → 重新授權一次
