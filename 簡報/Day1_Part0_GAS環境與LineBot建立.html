🤖 Vibe Coding — 用 AI 生成程式碼

在開始動手之前，讓我們學習一個重要觀念：Vibe Coding（氛圍編程）。你不需要從零開始寫程式，而是用自然語言告訴 AI 你想要什麼，AI 就能幫你生成程式碼！

💡 什麼是 Vibe Coding？
Vibe Coding 是 2025 年由 Andrej Karpathy（OpenAI 創辦成員）提出的概念：「你只要描述你的需求，AI 來幫你寫程式。」這不是偷懶，而是現代開發者的核心技能！

📝 本單元的 AI 提示詞範例

把以下提示詞貼給 AI（Claude、ChatGPT、Gemini 都可以）：

請幫我寫一個 Google Apps Script 的 Line Bot 共用推播模組，需求如下：

1. 使用 Line Messaging API 的 Push Message 功能
2. 包含以下函式：
   - pushText(to, text)：推播文字訊息
   - pushImage(to, imageUrl)：推播圖片訊息
   - pushLine(to, messages)：底層推播函式
   - replyText(replyToken, text)：回覆文字
   - replyLine(replyToken, messages)：底層回覆函式
   - testPush()：測試推播函式
3. LINE_TOKEN 和 LINE_USER_ID 用變數設定在最上方
4. 要有完整的錯誤處理和 Logger.log 記錄
5. 加上中文註解說明每個函式的用途

🎯 寫好提示詞的 3 個技巧

1. 說清楚「平台」

開頭要說「Google Apps Script」，AI 才知道要用 GAS 的 API（如 UrlFetchApp、SpreadsheetApp）。

2. 列出「功能清單」

用編號列出你需要的每一個功能，越具體越好。不要只說「幫我寫推播」，要說明推播什麼、給誰、格式如何。

3. 指定「細節要求」

例如：要有錯誤處理、要有中文註解、變數要放最上方。這些細節決定了程式碼的品質。

⚠️ 重要提醒：AI 生成的程式碼不一定 100% 正確，你需要理解它在做什麼，才能除錯和修改。這也是為什麼我們要先學會基礎觀念！

💡 什麼是 GAS？

GAS = Google Apps Script = Google 免費自動化工具

📝 用 JavaScript 寫

語法就是 JavaScript，全球最多人使用的程式語言之一，學習資源豐富，容易上手。

☁️ 在 Google 雲端執行

程式碼跑在 Google 的伺服器上，不佔你的電腦資源，關機也能自動執行！

💰 完全免費

只要有 Google 帳號就能用，不需要信用卡、不需要付費，零成本開始。

🖥️ 認識編輯器介面

Apps Script 編輯器有五個重要區域，請先認識它們：

🅰️ 左側檔案列表

顯示專案中所有的 .gs 檔案。預設會有一個 程式碼.gs，之後我們會新增更多檔案。

🅱️ 中間程式碼編輯區

這裡是你寫程式的地方！點選左側的檔案就會在這裡顯示內容，可以直接編輯。

🅲 執行按鈕（▶）

工具列上的 ▶ 執行 按鈕。點下去就會執行目前選定的函式。

🅳 執行記錄

點擊左側的 「執行記錄」，可以查看過去每次程式執行的結果和錯誤訊息。

🅴 觸發條件（鬧鐘）

點擊左側的 「觸發條件」（鬧鐘圖示），可以設定排程，讓程式定時自動執行。

💡 記住：最常用的就是 🅱️ 編輯區 和 🅲 執行按鈕，先熟悉這兩個就好！

📌 步驟 3 — Hello GAS！第一次執行

讓我們來執行人生第一支 GAS 程式！

程式碼

function myFunction() {
  Logger.log("Hello GAS！我的第一支程式！");
}

操作步驟

  1. 1 在編輯區中，刪除預設的所有程式碼。
  2. 2 貼上上方的程式碼。
  3. 3 按 Ctrl + S 儲存。
  4. 4 點擊工具列上的 ▶ 執行 按鈕。
💡 Logger.log() 是什麼？
這是 GAS 的「印出」功能，類似其他語言的 print 或 console.log。
執行後，結果會顯示在下方的「執行記錄」面板中。

🔐 授權流程

⚠️ 第一次執行會跳出授權視窗！
這是正常的，因為 Google 需要確認你允許這個程式存取你的帳號資源。

授權步驟

  1. 1 點擊 「審查權限」。
  2. 2 選擇你的 Google 帳號。
  3. 3 畫面顯示「Google 尚未驗證這個應用程式」，點擊左下角的 「進階」。
  4. 4 點擊 「前往『未命名的專案』（不安全）」。
  5. 5 點擊 「允許」。

⚙️ GAS 的限制

GAS 雖然免費好用，但有一些使用上的限制需要知道：

項目 限制 說明
每次執行時間 6 分鐘 單次函式執行不能超過 6 分鐘
每天觸發時間 90 分鐘 所有自動觸發的總執行時間
UrlFetchApp 呼叫 20,000 次/天 對外部 API 的請求次數
Email 寄送 100 封/天 透過 GmailApp 發送的郵件數量
觸發條件 20 個/專案 每個專案可設定的排程觸發數量
💡 不用擔心！以上限制對我們一般使用來說綽綽有餘。除非你要發大量信件或是做非常頻繁的 API 呼叫，否則幾乎不會碰到上限。

📝 共用推播模組程式碼

將以下程式碼完整貼入 LineBot.gs 檔案中：

// ===== Line Bot 共用推播模組 =====
const LINE_TOKEN  = '在這裡貼上你的 Channel Access Token';   // ← 改這裡
const LINE_USER_ID = '在這裡貼上你的 User ID';                // ← 改這裡
const LINE_PUSH_URL  = 'https://api.line.me/v2/bot/message/push';
const LINE_REPLY_URL = 'https://api.line.me/v2/bot/message/reply';

/** 推播文字訊息 */
function pushText(text, userId) {
  userId = userId || LINE_USER_ID;
  const payload = {
    to: userId,
    messages: [{ type: 'text', text: text }]
  };
  UrlFetchApp.fetch(LINE_PUSH_URL, {
    method: 'post',
    contentType: 'application/json',
    headers: { 'Authorization': 'Bearer ' + LINE_TOKEN },
    payload: JSON.stringify(payload)
  });
}

/** 推播圖片訊息 */
function pushImage(imageUrl, userId) {
  userId = userId || LINE_USER_ID;
  const payload = {
    to: userId,
    messages: [{
      type: 'image',
      originalContentUrl: imageUrl,
      previewImageUrl: imageUrl
    }]
  };
  UrlFetchApp.fetch(LINE_PUSH_URL, {
    method: 'post',
    contentType: 'application/json',
    headers: { 'Authorization': 'Bearer ' + LINE_TOKEN },
    payload: JSON.stringify(payload)
  });
}

/** 推播多則訊息 */
function pushLine(messages, userId) {
  userId = userId || LINE_USER_ID;
  const payload = { to: userId, messages: messages };
  UrlFetchApp.fetch(LINE_PUSH_URL, {
    method: 'post',
    contentType: 'application/json',
    headers: { 'Authorization': 'Bearer ' + LINE_TOKEN },
    payload: JSON.stringify(payload)
  });
}

/** 回覆文字訊息（用於 Webhook） */
function replyText(replyToken, text) {
  const payload = {
    replyToken: replyToken,
    messages: [{ type: 'text', text: text }]
  };
  UrlFetchApp.fetch(LINE_REPLY_URL, {
    method: 'post',
    contentType: 'application/json',
    headers: { 'Authorization': 'Bearer ' + LINE_TOKEN },
    payload: JSON.stringify(payload)
  });
}

/** 回覆多則訊息（用於 Webhook） */
function replyLine(replyToken, messages) {
  const payload = { replyToken: replyToken, messages: messages };
  UrlFetchApp.fetch(LINE_REPLY_URL, {
    method: 'post',
    contentType: 'application/json',
    headers: { 'Authorization': 'Bearer ' + LINE_TOKEN },
    payload: JSON.stringify(payload)
  });
}

/** 測試推播 — 用來驗證設定是否正確 */
function testPush() {
  pushText('🎉 恭喜！Line Bot 推播測試成功！');
  Logger.log('推播完成，請檢查手機 Line！');
}
⚠️ 重要：請將前兩行的 LINE_TOKEN 和 LINE_USER_ID 替換成你自己在記事本上暫存的值。
Token 和 User ID 前後一定要有單引號 ' '！

🔧 測試失敗？排除檢查清單

如果測試推播沒有成功，請依照以下清單逐一檢查：

1. Token 是否正確？

回到 Line Developers → Messaging API 分頁，確認 Channel Access Token 是否完整複製，沒有多餘的空格或遺漏。

2. User ID 是否以 U 開頭？

回到 Basic settings 分頁，確認 Your user ID 開頭是大寫的 U，如果不是，可能複製到了別的值。

3. 是否已加 Bot 好友？

確認手機 Line 有掃描 QR Code 並加入 Bot 好友。沒有加好友就無法收到推播！

4. 執行記錄有無錯誤？

點擊左側的「執行記錄」查看錯誤訊息。常見錯誤：401 表示 Token 錯誤，400 表示 User ID 錯誤。

💡 其他常見問題：
  • 確認 Token 和 User ID 有被單引號包住
  • 確認程式碼沒有多餘的中文標點符號
  • 試著重新 Issue 一組新的 Token
✅ 恭喜！排除問題後成功推播，代表單元 0 完成！
