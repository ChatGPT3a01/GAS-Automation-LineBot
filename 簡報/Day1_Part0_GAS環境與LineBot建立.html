1 登入 LINE Developers Console

第一步，我們要登入 LINE 的開發者後台，這是建立 Line Bot 的入口。

Step 1-1：開啟 LINE Developers 網站

👆 動作：打開 Chrome 瀏覽器，在網址列輸入 https://developers.line.biz/ 然後按 Enter
⏱️ 等待：約 3~5 秒，頁面載入完成
👀 結果：你會看到 LINE Developers 的首頁，畫面上方是黑色導覽列，右上角有一個綠色的 Log in 按鈕

Step 1-2：點擊登入

📍 位置：頁面右上角
👆 動作：點擊綠色的 Log in 按鈕
👀 結果：會跳轉到登入頁面，你會看到兩個選項：「使用 LINE 帳號登入」和「使用商業帳號登入」

Step 1-3：用 LINE 帳號登入

👆 動作：點擊綠色的 使用 LINE 帳號登入 按鈕
👀 結果：會出現 LINE 的登入畫面，你可以選擇：
  • 方法 A：輸入你的 Line 帳號（Email）和密碼
  • 方法 B：用手機 Line 掃描畫面上的 QR Code 登入（推薦，最簡單）

Step 1-4：首次登入需填寫開發者資料

👀 結果：如果你是第一次登入，會出現一個表單要求填寫開發者資料
👆 動作：填寫 Your name（你的名字，中英文都可以）和 Email（你的 Email）
👆 動作：勾選同意條款的核取方塊，然後點擊 Create my account
👀 結果：你會進入 LINE Developers Console 的主畫面
💡 提醒：LINE Developers 是免費的開發者平台，使用你平常的 Line 帳號就能登入，不需要另外註冊。如果你已經登入過，會直接跳到主畫面。

2 建立 Provider

Provider 是一個「開發者群組」的概念，可以把它想成一個資料夾，用來管理你的所有 Bot。

Step 2-1：找到建立按鈕

📍 位置：LINE Developers Console 主畫面
👆 動作：在畫面中找到 Create a new provider 按鈕（綠色或灰色按鈕），點擊它
👀 結果：會跳出一個小視窗（或新頁面），裡面有一個「Provider name」的輸入框

Step 2-2：輸入 Provider 名稱

👆 動作：在 Provider name 輸入框中輸入一個名稱，例如 GAS 練習（或任何你喜歡的名稱，中英文都可以）
👆 動作：點擊 Create 按鈕
👀 結果：畫面會跳轉到這個 Provider 的頁面，中間會顯示「No channels have been created yet.」表示還沒建立任何 Bot
💡 Provider 名稱怎麼取？
取什麼名字都可以，不會影響 Bot 的功能。你可以用自己的名字、專案名稱、或是「GAS 練習」之類的。

3 建立 Messaging API Channel

Channel 就是一個 Line Bot。我們要在剛才建立的 Provider 底下，建立一個 Messaging API 類型的 Channel。

Step 3-1：開始建立 Channel

📍 位置：你剛才建立的 Provider 頁面
👆 動作：點擊 Create a Messaging API channel 按鈕（有一個聊天氣泡圖示）
👀 結果：會出現一個很長的表單，需要填寫好幾個欄位

Step 3-2：填寫表單

依照下表逐一填寫（帶 * 號的是必填欄位）：

欄位名稱要填的內容說明
Channel typeMessaging API已自動選好，不用改
Provider你剛才建立的 Provider 名稱已自動帶入，不用改
Channel icon（可跳過）Bot 的大頭貼，之後可以再改
Channel name *GAS 推播助手你的 Bot 在 Line 上顯示的名稱
Channel description *GAS 自動推播通知簡短描述，隨便寫就好
Category *選 教育隨便選一個分類即可
Subcategory *選 學校（或任何子分類）隨便選一個即可
Email address *你的 Email通常已自動帶入

Step 3-3：同意條款並建立

📍 位置：表單的最下方
👆 動作：勾選兩個同意條款的核取方塊（LINE Official Account Terms of Use 和 LINE Official Account API Terms of Use）
👆 動作：點擊 Create 按鈕
⏱️ 等待：約 2~3 秒
👀 結果：會彈出一個確認視窗，點擊 OK。然後你會被帶到這個 Channel 的設定頁面，上方有好幾個分頁標籤（Basic settings、Messaging API 等）
⚠️ Channel name 注意：Channel name 設定後7 天內只能修改一次，但這不影響功能，所以隨便取個名字就好。

4 取得 Channel Access Token

Channel Access Token 是你的 Bot 的「身份證」，程式碼需要用它來告訴 LINE「我是哪一個 Bot」。

Step 4-1：切換到 Messaging API 分頁

📍 位置：你的 Channel 設定頁面上方，有一排分頁標籤
👆 動作：點擊 Messaging API 這個分頁標籤
👀 結果：頁面會顯示 Messaging API 的相關設定，你可以看到 Bot 的 QR Code、Webhook URL 等區塊

Step 4-2：找到 Channel access token 區塊

📍 位置：Messaging API 分頁
👆 動作：用滑鼠滾輪向下捲到頁面最底部
👀 結果：你會看到一個標題寫著 Channel access token (long-lived) 的區塊

Step 4-3：產生並複製 Token

👆 動作：點擊 Issue 按鈕（綠色按鈕）
⏱️ 等待：約 1~2 秒
👀 結果：會出現一串非常長的英數字文字（大約 170 個字元，看起來像 eyJhbGciOi...）
👆 動作：點擊 Token 旁邊的 Copy 按鈕（複製按鈕）
👆 動作：打開電腦的記事本（Windows 搜尋「記事本」），按 Ctrl+V 貼上，暫時保存
👀 結果：記事本上會出現一串很長的 Token
⚠️ 安全提醒：Channel Access Token 等同於你的 Bot 密碼，千萬不要分享給別人、不要放在公開的程式碼中！如果洩漏了，回到這裡按 Reissue 重新產生一組就好。

5 取得 User ID

User ID 告訴 Bot「要把訊息推播給誰」。每個 LINE 使用者都有一個唯一的 User ID。

Step 5-1：切換到 Basic settings 分頁

📍 位置：同一個 Channel 設定頁面的上方分頁標籤
👆 動作：點擊 Basic settings 這個分頁標籤（在 Messaging API 的左邊）
👀 結果：頁面會顯示 Channel 的基本資訊

Step 5-2：找到 Your user ID

📍 位置：Basic settings 分頁
👆 動作：用滑鼠滾輪向下捲到頁面最底部
👀 結果：你會看到一個標題寫著 Your user ID 的區塊，下方顯示一組以 U 開頭的字串（例如 Uab012cdef3456789...，大約 33 個字元）

Step 5-3：複製 User ID

👆 動作：點擊 User ID 旁邊的 Copy 按鈕
👆 動作：切換到記事本，按 Enter 換一行，再按 Ctrl+V 貼上
👀 結果：記事本上現在有兩筆資料：
✅ 確認你的記事本上有這兩筆資料：
  1. Channel Access Token — 很長一串英數字（約 170 字元，以 eyJ 開頭）
  2. Your user ID — 以大寫 U 開頭的字串（約 33 字元）
這兩筆資料等一下會貼到程式碼中，請不要關掉記事本！

6 關閉自動回覆 + 加 Bot 好友

這一步做兩件事：關閉 LINE 預設的自動回覆（避免跟我們的程式衝突），然後用手機加 Bot 好友。

6-A：關閉自動回覆

📍 位置：Channel 設定頁面 → Messaging API 分頁
👆 動作：在頁面中找到 Auto-reply messages 區塊，點擊右邊的 Edit 連結
👀 結果：會在新分頁開啟 LINE Official Account Manager 頁面
📍 位置：LINE Official Account Manager → 左側選單 → 回應設定
👆 動作：找到以下兩個開關，確認它們都是關閉狀態（灰色）：
  1. 自動回應訊息 → 關閉
  2. 加入好友的歡迎訊息 → 關閉
👀 結果：兩個開關都是灰色，表示已關閉
⚠️ 一定要關！不關的話，Line 會自動回覆制式訊息，和我們後面的程式邏輯衝突。

6-B：用手機加 Bot 好友

📍 位置：回到 LINE Developers → Channel 設定頁面 → Messaging API 分頁
👆 動作：在頁面上方找到 Bot information 區塊，裡面有一個 QR Code
👆 動作：拿出手機，打開 Line App → 點右上角的「加入好友」→「行動條碼」→ 掃描電腦螢幕上的 QR Code
👀 結果：手機上會出現你的 Bot（名稱就是你剛才設定的 Channel name），點擊 加入好友
💡 為什麼要加好友？
Line Bot 只能推播訊息給已加入好友的使用者。如果沒加好友，即使程式碼正確，你也收不到任何訊息！

7 建立 Google 試算表 + 開啟 Apps Script

LINE 端的設定已經完成，現在回到 Google 這邊，建立 GAS 專案。

Step 7-1：建立新的 Google 試算表

👆 動作：在 Chrome 瀏覽器的網址列輸入 sheets.new 然後按 Enter
⏱️ 等待：約 3~5 秒，瀏覽器會自動建立並開啟一個全新的 Google 試算表
👀 結果：你會看到一個空白的試算表，左上角顯示「未命名的試算表」

Step 7-2：替試算表命名

📍 位置：試算表畫面的左上角，有一行小字寫著「未命名的試算表」
👆 動作：點擊「未命名的試算表」那個文字，它會變成可編輯的輸入框
👆 動作：輸入 GAS 練習，然後按 Enter 或點擊其他地方
👀 結果：左上角的名稱會變成「GAS 練習」

Step 7-3：開啟 Apps Script 編輯器

📍 位置：試算表上方的選單列（檔案、編輯、檢視... 那一排），找到最右邊的「擴充功能」
👆 動作：點擊「擴充功能」
👀 結果：會展開一個下拉選單
👆 動作：在下拉選單中找到 Apps Script，點擊它
⏱️ 等待：瀏覽器會開啟一個全新的分頁，等待 3~5 秒直到編輯器完全載入
👀 結果：你會看到 Apps Script 編輯器：左邊是檔案列表（有一個「程式碼.gs」）、右邊是程式碼編輯區（裡面有預設的 function myFunction() { }）、上方有一個 ▶ 執行 按鈕
💡 記住以下 5 個重要區塊：
🅰 左側檔案列表 — 顯示專案中所有 .gs 檔案
🅱 中間程式碼編輯區 — 你寫程式的地方
🅲 ▶ 執行按鈕 — 點一下就能執行程式
🅳 執行記錄（左側選單第 3 個圖示）— 查看程式執行結果
🅴 觸發條件（左側選單第 4 個圖示，鬧鐘）— 設定排程自動執行

8 Hello GAS！第一次執行 + 授權

在貼上共用模組之前，我們先跑一個最簡單的程式，學會「執行」和「授權」的操作流程。

Step 8-1：貼上 Hello GAS 程式碼

📍 位置：Apps Script 編輯器的右半邊程式碼編輯區（你會看到預設的 function myFunction() { }）
👆 動作：在編輯區裡面按一下，按 Ctrl+A（全選），再按 Delete（刪除）
👆 動作：輸入以下程式碼（或複製貼上）： 
function myFunction() {
  Logger.log("Hello GAS！我的第一支程式！");
}
👆 動作：按 Ctrl+S 儲存

Step 8-2：執行程式

📍 位置：編輯器上方工具列
👆 動作：確認 ▶ 執行 按鈕左邊的下拉選單顯示的是 myFunction，然後點擊 ▶ 執行 按鈕
👀 結果：會跳出「需要授權」的對話框（第一次執行都會出現）

Step 8-3：完成授權流程（只有第一次需要）

授權第 1 步：彈出「需要授權」對話框 → 點擊 審查權限 按鈕
授權第 2 步：選擇你的 Google 帳號（點擊你的帳號名稱）
授權第 3 步：出現「Google 尚未驗證這個應用程式」的警告 → 點擊左下角的 進階 小字
授權第 4 步：畫面展開更多文字 → 點擊 前往「GAS 練習」（不安全）
授權第 5 步：會列出權限清單 → 點擊 允許 按鈕
⏱️ 等待：授權完成後，程式會自動開始執行，約 1~2 秒
👀 結果：編輯器下方會彈出「執行記錄」面板，裡面顯示：資訊 Hello GAS！我的第一支程式！
💡 別擔心「不安全」！這是你自己寫的程式，存在你自己的帳號裡，所以是安全的。Google 對所有未驗證的腳本都會顯示這個警告。只有第一次需要授權，之後就不會再出現了。

9 建立 LineBot.gs + 貼上共用模組

現在要建立一個新的 .gs 檔案，貼上課程提供的 Line Bot 共用推播模組程式碼。

Step 9-1：新增 LineBot.gs 檔案

📍 位置：Apps Script 編輯器的左側檔案列表區域
👆 動作：找到「檔案」標題旁邊的 + 按鈕（一個小圓形加號），點擊它
👀 結果：會出現一個小選單，有兩個選項：「指令碼」和「HTML」
👆 動作：點擊 指令碼
👀 結果：檔案列表中會多出一個新檔案，名稱正在等你輸入（閃爍的游標）
👆 動作：輸入 LineBot（注意：不要輸入 .gs，系統會自動加上）
👆 動作：按 Enter 確認
👀 結果：左側檔案列表現在有兩個檔案：程式碼.gs 和 LineBot.gs。右邊的編輯區會自動切換到空白的 LineBot.gs

🔍 程式碼解析 1 — 設定區 + 推播文字

這段的目的：宣告 Token 和 User ID，並提供一個簡單的推播文字函式。

📁 設定區

var LINE_TOKEN = '在此貼上你的 Channel Access Token';
var LINE_USER_ID = '在此貼上你的 User ID';
  • var — JavaScript 的變數宣告關鍵字，表示「建立一個變數」
  • LINE_TOKEN — Bot 的身份證，用來告訴 LINE「我是哪一個 Bot」
  • LINE_USER_ID — 你的使用者 ID，告訴 Bot「要把訊息推給誰」

📁 pushText(to, text) — 推播文字訊息

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}
  • to — 第 1 個參數：要推播給誰（通常傳入 LINE_USER_ID）
  • text — 第 2 個參數：要傳送的文字內容
  • [{ type: 'text', text: text }] — 把文字包成 LINE 要求的 JSON 格式
  • 它其實是呼叫下面的 pushLine() 底層函式來發送

📁 pushImage(to, imageUrl) — 推播圖片訊息

function pushImage(to, imageUrl) {
  pushLine(to, [{
    type: 'image',
    originalContentUrl: imageUrl,
    previewImageUrl: imageUrl
  }]);
}
  • imageUrl — 圖片的公開網址（必須是 https:// 開頭，且可公開存取）
  • originalContentUrl — 使用者點開圖片時顯示的原圖
  • previewImageUrl — 聊天室中顯示的縮圖（這裡跟原圖用同一個網址）

🔍 程式碼解析 2 — pushLine 底層推播

這段的目的：這是真正發送 HTTP 請求到 LINE 伺服器的底層函式，pushText 和 pushImage 都靠它來發送。

function pushLine(to, messages) {
  var url = 'https://api.line.me/v2/bot/message/push';
  var options = {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      to: to,
      messages: messages
    }),
    muteHttpExceptions: true
  };

  var response = UrlFetchApp.fetch(url, options);
  var code = response.getResponseCode();

  if (code !== 200) {
    Logger.log('推播失敗，狀態碼：' + code);
    Logger.log('錯誤內容：' + response.getContentText());
  } else {
    Logger.log('推播成功！');
  }
}

逐行解釋：

  • var url = '...' — LINE Messaging API 的推播端點網址
  • method: 'post' — 用 POST 方法發送請求（推資料給 LINE 伺服器）
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 身份驗證：告訴 LINE「我是哪一個 Bot」
  • JSON.stringify({...}) — 把 JavaScript 物件轉成 JSON 字串
  • muteHttpExceptions: true — 即使發生錯誤也不要中斷程式，讓我們自己處理
  • UrlFetchApp.fetch(url, options) — GAS 內建的 HTTP 請求工具，等同瀏覽器發送網路請求
  • getResponseCode() — 取得回應狀態碼，200 表示成功
  • Logger.log() — 把結果記錄到執行記錄中，方便除錯

🔍 程式碼解析 3 — 回覆函式 + 測試函式

📁 replyText / replyLine — 回覆函式

目的：當使用者傳訊息給 Bot 時，Bot 可以「回覆」（需要 Webhook，後面 Part 會教）

function replyText(replyToken, text) {
  replyLine(replyToken,
    [{ type: 'text', text: text }]);
}

function replyLine(replyToken, messages) {
  var url = 'https://api.line.me/v2/bot/message/reply';
  var options = {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      replyToken: replyToken,
      messages: messages
    }),
    muteHttpExceptions: true
  };
  var response = UrlFetchApp.fetch(url, options);
  // ...
}
  • replyToken — LINE 提供的一次性回覆 token（每次使用者傳訊息時都不同）
  • 回覆 API 的端點是 .../reply，推播是 .../push

📁 testPush() — 測試推播函式

目的：用來驗證 Token 和 User ID 設定是否正確

function testPush() {
  var now = new Date()
    .toLocaleString('zh-TW',
      { timeZone: 'Asia/Taipei' });
  pushText(LINE_USER_ID,
    '🎉 Line Bot 連線測試成功！\n\n'
    + '目前時間：' + now
    + '\n\n如果你看到這則訊息，'
    + '表示 GAS 與 Line Bot '
    + '已成功串接。');
}
  • new Date() — 取得目前的日期時間
  • .toLocaleString('zh-TW', ...) — 轉換成台灣時間格式
  • pushText(LINE_USER_ID, ...) — 推播給你自己
  • \n — 換行符號
💡 推播 vs 回覆的差異：
推播 (push)：Bot 主動發訊息，不需要使用者先傳訊息
回覆 (reply)：使用者傳訊息後，Bot 才能回覆（需要 replyToken）

❓ 常見問題排解

🔍 第一步：怎麼找到錯誤訊息？

📍 位置：Apps Script 編輯器左側選單
👆 動作：找到從上面數下來第 3 個圖示（⏱ 時鐘圖示），把滑鼠移上去會顯示「執行項目」，點擊它
👀 結果：會出現執行歷史記錄列表，每次執行都會留下一筆紀錄
👆 動作：找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的展開箭頭
👀 結果：展開後看到的紅色文字就是錯誤原因

推播失敗，狀態碼：401

原因：Channel Access Token 不正確

  • 回到 LINE Developers → Messaging API 分頁
  • 確認 Token 是否完整複製（沒有頭尾被截斷）
  • 確認前後沒有多餘的空格
  • 試試重新按 Reissue 產生一組新的

推播失敗，狀態碼：400

原因：User ID 不正確

  • 回到 LINE Developers → Basic settings 分頁
  • 確認 User ID 是以大寫 U 開頭
  • 確認是從 Your user ID 欄位複製的（不是 Channel ID）
  • 確認單引號 ' ' 包住整個 ID

執行成功但手機沒收到

原因：可能沒加 Bot 好友

  • 確認手機 Line 有掃 QR Code 並加入 Bot 好友
  • 在 Line 的好友列表中搜尋 Bot 名稱
  • 如果找不到，回到 Messaging API 分頁重新掃 QR Code

TypeError 或語法錯誤

原因：程式碼貼不完整或格式損壞

  • 刪除 LineBot.gs 中所有程式碼，重新從 .gs 檔案複製貼上
  • 確認全部程式碼都有貼進去（約 131 行）
  • 確認沒有多餘的中文標點符號混入

⚙️ GAS 的限制（了解即可）

GAS 雖然免費好用，但有一些使用上的限制。不過對我們的課程來說綽綽有餘，知道就好。

項目限制說明
每次執行時間6 分鐘單次函式執行不能超過 6 分鐘
每天觸發時間90 分鐘所有自動觸發的總執行時間
UrlFetchApp 呼叫20,000 次/天對外部 API 的請求次數
