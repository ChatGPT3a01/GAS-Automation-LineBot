🎯 學習目標

完成這個實作後，你將學會以下四大核心能力：

📄 1. HtmlService 建立網頁介面

使用 GAS 內建的 HtmlService 服務，建立具有完整 HTML/CSS/JS 的互動式網頁介面，不需要額外的伺服器或網域。

🌐 2. doGet() 回傳 HTML 頁面

透過 doGet() 函式攔截 HTTP GET 請求，將自訂的 HTML 檔案回傳給瀏覽器，讓使用者看到漂亮的表單頁面。

🔗 3. google.script.run 前端呼叫後端

學會使用 google.script.run 這個神奇的橋梁，讓前端 HTML 頁面能呼叫後端 GAS 函式，實現前後端溝通。

🚀 4. 完整的 Web App 部署流程

掌握從開發到部署的完整流程：撰寫程式碼 → 新增 HTML 檔案 → 設定權限 → 部署為網頁應用程式 → 取得公開網址。

🔍 後端程式碼解說（一）：doGet()

目的：當使用者開啟網址時，顯示 HTML 表單頁面

function doGet(e) {
  var html = HtmlService.createHtmlOutputFromFile('form');
  html.setTitle('聯絡表單');
  html.setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
  return html;
}

逐行解釋

  • function doGet(e) — GAS 的特殊函式：當有人用瀏覽器開啟部署網址時，自動執行此函式。e 是請求參數（本例未使用）
  • var html = HtmlService.createHtmlOutputFromFile('form') — 去找名為 form 的 HTML 檔案（即 form.html），將其內容轉成可回傳的 HTML 物件
  • html.setTitle('聯絡表單') — 設定瀏覽器分頁標題（顯示在分頁標籤上的文字）
  • html.setXFrameOptionsMode(...ALLOWALL) — 允許此頁面被嵌入 iframe（可選設定，讓頁面更靈活）
  • return html — 將 HTML 頁面回傳給瀏覽器顯示
💡 核心概念：doGet() 和 doPost() 是 GAS Web App 的兩個入口。doGet() 處理「打開網頁」的請求（GET），doPost() 處理「提交資料」的請求（POST）。本實作只用 doGet()，因為表單提交是透過 google.script.run 而非 HTTP POST。

🔍 後端程式碼解說（二）：processForm()

目的：接收表單資料 → 寫入試算表 → 推播 Line 通知 → 回傳結果

function processForm(formData) {
  try {
    var ss = SpreadsheetApp.getActiveSpreadsheet();
    var sheet = ss.getSheetByName(FORM_SHEET_NAME);

    if (!sheet) {
      sheet = createFormSheet(ss);
    }

    var now = new Date().toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' });

    sheet.appendRow([
      now, formData.name, formData.email, formData.message
    ]);

    var rowCount = sheet.getLastRow() - 1;

    var notification = '📬 收到新的表單填寫！\n'
      + '━━━━━━━━━━━━\n'
      + '👤 姓名：' + formData.name + '\n'
      + '📧 Email：' + formData.email + '\n'
      + '💬 訊息：' + formData.message + '\n'
      + '━━━━━━━━━━━━\n'
      + '📅 時間：' + now + '\n'
      + '📊 累計第 ' + rowCount + ' 筆';

    pushText(LINE_USER_ID, notification);

    return { success: true, message: '提交成功！感謝你的填寫。' };
  } catch (error) {
    return { success: false, message: '提交失敗：' + error.toString() };
  }
}

逐行解釋

  • function processForm(formData) — 接收前端傳來的物件 { name, email, message }
  • try { ... } catch (error) { ... } — 錯誤處理：如果中間任何步驟出錯，不會讓程式當掉，而是回傳失敗訊息
  • ss.getSheetByName(FORM_SHEET_NAME) — 尋找名為「表單回覆」的工作表
  • if (!sheet) { sheet = createFormSheet(ss); } — 如果工作表不存在，自動建立一個（含格式化標題列）
  • sheet.appendRow([...]) — 在試算表最後一列新增一筆資料
  • sheet.getLastRow() - 1 — 取得目前總筆數（扣掉標題列）
  • pushText(LINE_USER_ID, notification) — 將通知訊息推播到你的 LINE
  • return { success: true, ... } — 回傳 JSON 物件給前端，前端依此顯示成功或失敗訊息

🔍 前端程式碼解說：form.html

目的：提供漂亮的表單介面，送出時透過 google.script.run 呼叫後端

核心 JavaScript：submitForm()

function submitForm(event) {
  event.preventDefault();

  var formData = {
    name: document.getElementById('name').value.trim(),
    email: document.getElementById('email').value.trim(),
    message: document.getElementById('message').value.trim()
  };

  if (!formData.name || !formData.email || !formData.message) {
    showResult('請填寫所有欄位！', false);
    return;
  }

  document.getElementById('submitBtn').disabled = true;
  document.getElementById('submitBtn').textContent = '提交中...';
  document.getElementById('loading').classList.add('show');

  google.script.run
    .withSuccessHandler(onSuccess)
    .withFailureHandler(onFailure)
    .processForm(formData);
}

逐行解釋

  • event.preventDefault() — 阻止表單預設的提交行為（預設會重新整理頁面），改用 JavaScript 控制
  • var formData = { name: ..., email: ..., message: ... } — 從三個輸入框取值，.trim() 去掉前後空白
  • if (!formData.name || ...) — 前端驗證：如果有任何欄位是空的，顯示錯誤訊息並中止
  • submitBtn.disabled = true — 按鈕灰掉，防止重複點擊送出
  • google.script.run — 前端呼叫後端的橋梁
  • .withSuccessHandler(onSuccess) — 後端成功執行後，呼叫前端的 onSuccess 函式
  • .withFailureHandler(onFailure) — 後端發生錯誤時，呼叫前端的 onFailure 函式
  • .processForm(formData) — 呼叫後端的 processForm 函式，傳入表單資料
💡 重要：google.script.run 只能在 GAS 部署的 HTML 頁面中使用。它不是標準的 JavaScript API，而是 Google 提供的專屬橋梁。

🔍 前端程式碼解說：回呼函式

目的：處理後端回傳的結果，顯示成功或失敗訊息

function onSuccess(result) {
  document.getElementById('loading').classList.remove('show');

  if (result.success) {
    showResult(result.message, true);
    document.getElementById('contactForm').reset();
  } else {
    showResult(result.message, false);
  }

  document.getElementById('submitBtn').disabled = false;
  document.getElementById('submitBtn').textContent = '送出表單';
}

function onFailure(error) {
  document.getElementById('loading').classList.remove('show');
  showResult('提交失敗：' + error.message, false);
  document.getElementById('submitBtn').disabled = false;
  document.getElementById('submitBtn').textContent = '送出表單';
}

function showResult(message, isSuccess) {
  var el = document.getElementById('resultMessage');
  el.textContent = message;
  el.className = 'result-message ' + (isSuccess ? 'success' : 'error');
}

逐行解釋

  • onSuccess(result) — 後端 processForm 成功執行後自動被呼叫，result 就是後端 return 的 { success, message } 物件
  • loading.classList.remove('show') — 隱藏「提交中...」的載入動畫
  • if (result.success) — 判斷後端回傳的結果是成功還是失敗
  • contactForm.reset() — 成功時清空表單，讓使用者可以再填一次
  • onFailure(error) — 後端發生未預期的錯誤（例如網路斷線、權限問題）時被呼叫
  • showResult(message, isSuccess) — 顯示結果訊息，true 顯示綠色成功框，false 顯示紅色失敗框

前端送出 → 後端處理 → return { success, message } → onSuccess / onFailure

🔍 後端程式碼解說（三）：工具函式

createFormSheet()：自動建立工作表

function createFormSheet(ss) {
  var sheet = ss.insertSheet(FORM_SHEET_NAME);

  var headers = ['時間戳記', '姓名', 'Email', '訊息內容'];
  sheet.getRange(1, 1, 1, headers.length).setValues([headers]);
  sheet.getRange(1, 1, 1, headers.length)
    .setFontWeight('bold')
    .setBackground('#3F51B5')
    .setFontColor('#FFFFFF')
    .setHorizontalAlignment('center');

  sheet.setColumnWidth(1, 180);
  sheet.setColumnWidth(2, 120);
  sheet.setColumnWidth(3, 200);
  sheet.setColumnWidth(4, 400);
  sheet.setFrozenRows(1);

  return sheet;
}
  • ss.insertSheet(FORM_SHEET_NAME) — 在試算表中新增名為「表單回覆」的工作表
  • setValues([headers]) — 在第一列寫入標題：時間戳記、姓名、Email、訊息內容
  • setFontWeight / setBackground / setFontColor — 格式化標題列：粗體白字、藍色背景
  • setColumnWidth — 設定各欄寬度，讓資料更易閱讀
  • setFrozenRows(1) — 凍結第一列，向下捲動時標題列不會消失

pushText() 與 pushLine()：Line 推播

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}

function pushLine(to, messages) {
  UrlFetchApp.fetch('https://api.line.me/v2/bot/message/push', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({ to: to, messages: messages }),
    muteHttpExceptions: true
  });
}
  • pushText(to, text) — 簡化版：傳入收件者 ID 和文字，自動包裝成 LINE 訊息格式
  • pushLine(to, messages) — 底層函式：呼叫 LINE Push Message API 送出訊息
  • muteHttpExceptions: true — 即使 API 回傳錯誤，也不要讓程式中斷（靜默處理）

🧪 額外測試：用 testFormSubmit() 驗證

除了從網頁填寫表單，你也可以在 Apps Script 編輯器中直接執行測試函式：

目的：不開網頁，直接從編輯器模擬表單提交

function testFormSubmit() {
  var testData = {
    name: '測試同學',
    email: 'test@example.com',
    message: '這是一筆測試資料，用來確認系統運作正常。'
  };

  var result = processForm(testData);
  Logger.log('測試結果：' + JSON.stringify(result));
}
  • var testData = { ... } — 模擬一筆表單資料（和真正填寫表單傳入的格式一模一樣）
  • processForm(testData) — 直接呼叫後端函式處理這筆資料
  • Logger.log(...) — 將結果印到執行記錄中

📍 Apps Script 編輯器 → 上方函式下拉選單

👆 選擇 testFormSubmit → 點擊「▶ 執行」

⏱️ 等待執行完成（約 3-5 秒）

👀 執行記錄顯示「測試結果：{"success":true,...}」+ LINE 收到通知

💡 什麼時候用 testFormSubmit？
  • 剛貼完程式碼，想快速確認後端邏輯是否正確（不需部署）
  • 修改程式碼後，想測試 LINE 推播是否正常
  • 除錯時，想看 Logger.log 的輸出訊息

🔧 常見問題排解

遇到問題不要慌！以下是最常見的錯誤和解決方法：

如何查看錯誤訊息？

📍 Apps Script 編輯器 → 左側選單

👆 點擊「執行項目」（時鐘圖示）→ 查看最近的執行記錄

👀 看到每次執行的狀態（成功/失敗）和錯誤訊息

錯誤現象 錯誤訊息 解決方法
開啟網址顯示空白 Script function not found: doGet Code.gs 沒有 doGet 函式，確認程式碼已貼上且已儲存
開啟網址報錯 Cannot read file 'form' HTML 檔案名稱不是 form，請刪除後重新建立
表單送出失敗 Script function not found: processForm Code.gs 缺少 processForm 函式，確認程式碼完整
試算表沒資料 Cannot call method "appendRow" of null getActiveSpreadsheet() 為 null，確認 Code.gs 綁定的是試算表
LINE 沒收到通知 無錯誤訊息（靜默失敗） 檢查 LINE_TOKEN 和 LINE_USER_ID 是否正確（注意前後空白）
LINE API 回傳 401 Authentication failed LINE_TOKEN 不正確或已過期，重新從 LINE Developers 複製
修改後沒生效 （顯示舊版表單） 需要建立新版本部署：部署 → 管理部署作業 → 鉛筆圖示 → 版本選「新版本」→ 部署
⚠️ 最常犯的錯誤：修改程式碼後忘記「建立新版本」！GAS 的部署是版本制的，修改程式碼後必須到「管理部署作業」建立新版本，否則使用者看到的仍是舊版。
