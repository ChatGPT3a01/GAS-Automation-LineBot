1 註冊兩個免費 API

這個 Bot 需要兩組免費的 API Key，都只要 Email 註冊就能拿到。

API-1：OpenWeatherMap（天氣資料）

👆 動作：打開瀏覽器，前往 https://openweathermap.org/api
👆 動作：點擊右上角 Sign Up，用 Email 註冊帳號
👆 動作：註冊完成後，到你的信箱點確認信
👆 動作：登入後，點擊上方「API keys」分頁
👀 結果：頁面上顯示一組 API Key，複製它存到記事本

API-2：Stormglass（洋流/海洋資料）

👆 動作：前往 https://stormglass.io/
👆 動作：點擊 Sign Up Free 或 Get API Key
👆 動作：用 Email 註冊並確認
👀 結果：Dashboard 頁面顯示你的 API Key，複製它存到記事本
⚠️ 免費額度說明：
- OpenWeatherMap：每分鐘 60 次請求（非常充裕）
- Stormglass：每天 10 次請求（省著用，測試時注意）
- 兩組 API Key 都不要分享給別人

在程式碼最上方找到設定區，依序貼上你的 4 組設定值：

var LINE_TOKEN = '在此貼上你的 Line Channel Access Token';

設定項目哪裡取得格式範例

9 設定定時推播觸發器

訂閱只是「記錄」使用者想收天氣，還需要設定「觸發器」讓系統每天自動執行推播。

方法一：用快捷函式設定（推薦）

👆 動作：回到 GAS 編輯器
👆 動作：函式下拉選單 → 選 setup7AM（早上 7 點推播）→ 點 ▶ 執行
👀 結果：執行記錄顯示「已建立每日推播觸發器：7:00」
快捷函式推播時間
setup7AM每天早上 7:00
setup730AM每天早上 7:30
setup8AM每天早上 8:00

方法二：自訂時間

👆 動作：函式下拉選單 → 選 setupDailyWeatherTrigger
📝 修改：在程式碼中找到 setupDailyWeatherTrigger(hour, minute) 呼叫，自行改數字
💡 範例：setupDailyWeatherTrigger(6, 30) = 每天早上 6:30 推播

確認觸發器

👆 動作：GAS 左側選單 →「觸發條件」（鬧鐘圖示）
👀 結果：看到一筆 sendScheduledWeatherPush 的時間觸發器，就表示設定成功！
⚠️ 注意：GAS 的時間觸發器有 ±15 分鐘誤差（Google 的限制）。設定 7:00 可能在 6:45~7:15 之間執行。如果要刪除觸發器，可以在「觸發條件」頁面手動刪除，或執行 deleteDailyWeatherTriggers 函式。

❓ 常見問題排解

Bot 完全沒回應

  • 確認 Webhook URL 有貼對（步驟 6）
  • 確認「Use webhook」是開啟狀態
  • 確認部署時「誰可以存取」選了「所有人」
  • 修改程式碼後要重新部署（部署 → 管理部署作業 → 編輯 → 版本選「新版本」→ 部署）

天氣資料取得失敗

  • 先跑 testWeatherAPI 確認 API 連線
  • 新註冊的 OpenWeatherMap Key 要等 10~30 分鐘生效
  • 確認 Key 前後沒有多餘空格

洋流資料取得失敗

  • Stormglass 免費版每天只有 10 次
  • 用完了要等隔天才會重置
  • 測試時建議只查天氣，省洋流額度

修改程式碼後沒生效

  • Webhook 的程式碼修改後必須重新部署
  • 部署 → 管理部署作業 → 右上角✏️ → 版本選「新版本」→ 部署
  • 這是最常忘記的步驟！

🔍 程式碼解析 — 核心架構

這個 Bot 的架構是「狀態機」— 根據使用者目前的狀態，決定怎麼處理訊息。

doPost() — Webhook 入口

function doPost(e) {
  var events = JSON.parse(e.postData.contents).events;
  for (var i = 0; i < events.length; i++) {
    if (events[i].type === 'message') {
      handleMessage(events[i]);  // 處理每則訊息
    }
  }
}

handleMessage() — 狀態判斷

function handleMessage(event) {
  var userMessage = event.message.text.trim();
  var userId = event.source.userId;
  var userState = getUserState(userId);  // 從試算表讀取狀態

  if (userMessage === '低軌衛星') {
    replyMessages = getMainMenu();         // → 主選單
  } else if (userMessage === '訂閱天氣') {
    replyMessages = getLocationMenu('subscribe_weather'); // → 訂閱
  } else if (userState.state === 'MAIN_MENU') {
    replyMessages = handleMainMenuSelection(...);  // → 功能選擇
  } else if (userState.state === 'SELECT_LOCATION') {
    replyMessages = handleLocationSelection(...);  // → 地區選擇
  }

  replyToLine(replyToken, replyMessages);  // 回覆
}
💡 狀態機是什麼？就像「多步驟對話」：Bot 記住你在哪一步（用試算表記錄），根據你目前的步驟決定下一步要做什麼。這樣才能實現「先選功能 → 再選地區」的互動流程。
