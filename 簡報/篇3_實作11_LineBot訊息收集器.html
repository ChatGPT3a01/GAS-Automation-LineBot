🔍 程式碼解析 1 — doPost() 主流程

這段的目的：這是 Webhook 的入口函式。每當使用者傳訊息給 Bot，Line 平台就會呼叫此函式。

function doPost(e) {
  try {
    var userData = JSON.parse(e.postData.contents);
    var sheet = SpreadsheetApp.openById(SPREADSHEET_ID).getActiveSheet();

    // 檢查是否有標題列，沒有就建立
    if (sheet.getLastRow() === 0) {
      createHeaderRow(sheet);
    }

    // 逐一處理每個事件
    for (var i = 0; i < userData.events.length; i++) {
      var event = userData.events[i];

      // 跳過非訊息事件
      if (event.type !== 'message') {
        continue;
      }

      processMessage(event, sheet);
    }
  } catch (error) {
    Logger.log('doPost 錯誤：' + error.toString());
  }
}

逐行解釋：

  • doPost(e) — GAS 的特殊函式名稱，當外部發送 HTTP POST 請求到部署網址時，GAS 會自動執行這個函式，參數 e 包含請求的所有資料
  • JSON.parse(e.postData.contents) — 將 Line 傳來的 JSON 字串解析成 JavaScript 物件，裡面包含所有事件資料
  • SpreadsheetApp.openById(SPREADSHEET_ID) — 用試算表 ID 開啟指定的 Google 試算表
  • .getActiveSheet() — 取得試算表中目前的工作表（通常是第一個分頁）
  • sheet.getLastRow() === 0 — 檢查試算表是否是空的（沒有任何列），如果是就自動建標題列
  • userData.events — Line 一次可能傳來多個事件（例如使用者快速傳了好幾則訊息），所以用 for 迴圈逐一處理
  • event.type !== 'message' — 跳過非訊息事件（例如加入好友、退出群組等），我們只處理「message」類型
  • try...catch — 錯誤防護，即使某一則訊息處理失敗，也不會讓整個函式崩潰
💡 Webhook 運作原理：
你不需要主動去問 Line「有沒有新訊息」。Line 會主動發送 HTTP POST 請求到你的部署網址，GAS 的 doPost(e) 就會被觸發。這就是「被動接收」的概念。

🔍 程式碼解析 2 — processMessage() 訊息處理

這段的目的：處理單一訊息事件 — 取得來源、用戶資料、訊息內容，最後寫入試算表。

function processMessage(event, sheet) {
  var now = new Date();
  var sourceInfo = getSourceInfo(event);
  var userProfile = getUserProfile(
    sourceInfo.sourceType, sourceInfo.groupId, event.source.userId
  );
  var displayName = userProfile ? userProfile.displayName : '未知用戶';
  var messageResult = processMessageType(event, sourceInfo, now);

  // 寫入試算表
  sheet.appendRow([
    now,                          // A 欄：時間
    sourceInfo.groupId,           // B 欄：群組 ID
    sourceInfo.groupName,         // C 欄：群組名稱
    event.source.userId,          // D 欄：用戶 ID
    displayName,                  // E 欄：用戶名稱
    messageResult.text,           // F 欄：訊息內容
    messageResult.fileLocation    // G 欄：檔案位置
  ]);
}

逐行解釋：

  • new Date() — 取得當下的時間，用來記錄訊息收到的時間戳記
  • getSourceInfo(event) — 判斷訊息來自哪裡（個人聊天、群組、還是聊天室），取得群組 ID 和群組名稱
  • getUserProfile(...) — 透過 Line API 取得使用者的顯示名稱
  • userProfile ? ... : '未知用戶' — 三元運算子：如果成功取得用戶資料就用真名，否則顯示「未知用戶」
  • processMessageType(event, sourceInfo, now) — 根據訊息類型（文字/圖片/貼圖/位置等）做不同處理，回傳 { text, fileLocation }
  • sheet.appendRow([...]) — 在試算表最後一列新增一列資料，陣列中每個元素對應一欄

🔀 程式碼解析 3 — processMessageType() 訊息類型分派

這段的目的：根據不同的訊息類型（文字、貼圖、位置、圖片...），做不同的處理。

function processMessageType(event, sourceInfo, now) {
  var text = '';
  var fileLocation = '';
  var message = event.message;

  switch (message.type) {
    case 'text':
      text = message.text;
      break;

    case 'sticker':
      text = '【貼圖】';
      fileLocation = 'Package：' + message.packageId
                   + '、Sticker：' + message.stickerId;
      break;

    case 'location':
      text = '【位置】經度：' + message.longitude
           + '、緯度：' + message.latitude;
      if (message.title) { text += '、' + message.title; }
      if (message.address) { text += '、' + message.address; }
      break;

    case 'image': case 'video': case 'audio': case 'file':
      text = '【' + getMessageTypeName(message.type) + '】';
      fileLocation = saveFileToDrive(event, sourceInfo, now);
      break;

    default:
      text = '【不支援的訊息類型：' + message.type + '】';
  }

  return { text: text, fileLocation: fileLocation };
}

逐行解釋：

  • switch (message.type) — 根據 message.type 的值來決定執行哪一段程式碼
  • case 'text' — 文字訊息：直接把 message.text（使用者傳的文字）記錄下來
  • case 'sticker' — 貼圖訊息：記錄貼圖的 Package ID 和 Sticker ID（可用於辨識是哪張貼圖）
  • case 'location' — 位置訊息：記錄經緯度，如果有地點名稱和地址也一併記錄
  • case 'image': case 'video': ... — 圖片/影片/音檔/檔案：呼叫 saveFileToDrive() 下載並存到 Google Drive
  • getMessageTypeName() — 把英文類型轉成中文（image→圖片、video→影片、audio→音檔、file→檔案）
  • default — 其他未知的訊息類型：記錄一個提示文字，程式不會崩潰
  • break — 結束當前 case，防止「穿透」到下一個 case（這是 switch 的標準用法）
💡 什麼是 switch/case？
switch 就像一個「多路分岔口」，根據變數的值走不同的路。比起寫一堆 if...else if...else if...，用 switch 更簡潔易讀。每個 case 後面記得加 break！

📁 程式碼解析 4 — saveFileToDrive() 檔案儲存

這段的目的：當使用者傳送圖片、影片、音檔或檔案時，從 Line 伺服器下載並存到 Google Drive。

function saveFileToDrive(event, sourceInfo, now) {
  try {
    var rootFolder = DriveApp.getFolderById(ROOT_FOLDER_ID);
    var folderName = sourceInfo.groupId || event.source.userId;
    var destinationFolder = getOrCreateSubfolder(rootFolder, folderName);

    // 從 Line 下載檔案
    var messageId = event.message.id;
    var fileBlob = getLineFileData(messageId).getBlob();

    // 建立檔案
    var file = destinationFolder.createFile(fileBlob);

    // 設定檔案名稱
    var prefix = formatFilePrefix(now);
    if (event.message.type === 'file' && event.message.fileName) {
      file.setName(prefix + event.message.fileName);
    } else {
      var ext = file.getName().split('.').pop();
      file.setName(prefix + messageId + '.' + ext);
    }

    return 'https://drive.google.com/open?id=' + file.getId();
  } catch (error) {
    return '儲存失敗：' + error.toString();
  }
}

逐行解釋：

  • DriveApp.getFolderById(ROOT_FOLDER_ID) — 用資料夾 ID 找到你在步驟 2 建立的根資料夾
  • sourceInfo.groupId || event.source.userId — 如果在群組中就用群組 ID 當子資料夾名稱，否則用使用者 ID（這樣不同來源的檔案會分開存放）
  • getOrCreateSubfolder() — 在根資料夾底下找到（或新建）以 ID 命名的子資料夾
  • getLineFileData(messageId) — 呼叫 Line Content API 下載檔案的二進制資料
  • .getBlob() — 把下載回來的回應轉成 Blob（二進制大物件），這是 GAS 建立檔案需要的格式
  • destinationFolder.createFile(fileBlob) — 在目標資料夾中建立一個新檔案
  • formatFilePrefix(now) — 用時間戳記當檔案名稱的前綴（例如 20260309143000-），避免重名
  • file.getId() — 取得新建檔案的 ID，組合成可分享的 Google Drive 連結
💡 Line Content API：
下載 URL 是 https://api-data.line.me/v2/bot/message/{messageId}/content。注意這是 api-data.line.me（不是 api.line.me），是專門用來下載多媒體內容的端點。

🌐 程式碼解析 5 — Line API 輔助函式

這段的目的：提供取得來源資訊、用戶資料、群組資料的工具函式。

📁 getSourceInfo(event) — 判斷訊息來源

function getSourceInfo(event) {
  var groupId = '';
  var groupName = '';
  var sourceType = event.source.type || '';

  if (sourceType === 'group') {
    groupId = event.source.groupId;
    var groupProfile = getGroupProfile(groupId);
    groupName = groupProfile ? groupProfile.groupName : '未知群組';
  } else if (sourceType === 'room') {
    groupId = event.source.roomId;
    groupName = '聊天室（無名稱）';
  }

  return { groupId: groupId, groupName: groupName, sourceType: sourceType };
}
  • event.source.type — 訊息來源類型：'user'（個人）、'group'（群組）、'room'（聊天室）
  • 如果是群組，會透過 getGroupProfile() 去 Line API 查詢群組名稱
  • 如果是個人聊天，groupId 和 groupName 都是空字串

📁 getUserProfile() — 取得用戶顯示名稱

function getUserProfile(sourceType, groupId, userId) {
  try {
    var url = '';
    switch (sourceType) {
      case 'group':
        url = 'https://api.line.me/v2/bot/group/' + groupId + '/member/' + userId;
        break;
      case 'room':
        url = 'https://api.line.me/v2/bot/room/' + groupId + '/member/' + userId;
        break;
      default:
        url = 'https://api.line.me/v2/bot/profile/' + userId;
    }
    var response = UrlFetchApp.fetch(url, {
      headers: { 'Authorization': 'Bearer ' + LINE_TOKEN },
      method: 'get', muteHttpExceptions: true
    });
    if (response.getResponseCode() === 200) {
      return JSON.parse(response.getContentText());
    }
    return null;
  } catch (error) { return null; }
}
  • 根據來源類型使用不同的 API 端點 — 群組成員和個人的查詢 URL 不同
  • UrlFetchApp.fetch() — GAS 內建的 HTTP 請求工具
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 告訴 Line「我是哪一個 Bot」的身份驗證
  • muteHttpExceptions: true — 即使 API 回傳錯誤也不會中斷程式
  • 成功時回傳包含 displayName 的物件，失敗時回傳 null

🔧 程式碼解析 6 — 工具函式與推播

這段的目的：提供建標題列、時間格式化、推播訊息等輔助工具。

📁 createHeaderRow(sheet) — 自動建立標題列

function createHeaderRow(sheet) {
  var headers = ['時間', '群組ID', '群組名稱', '用戶ID', '用戶名稱', '訊息內容', '檔案位置'];
  sheet.getRange(1, 1, 1, headers.length).setValues([headers]);
  sheet.getRange(1, 1, 1, headers.length)
    .setFontWeight('bold')
    .setBackground('#1B5E20')
    .setFontColor('#FFFFFF')
    .setHorizontalAlignment('center');
  sheet.setFrozenRows(1);
}
  • getRange(1, 1, 1, headers.length) — 選取第 1 列、第 1 欄開始，共 1 列、headers.length 欄的範圍
  • .setValues([headers]) — 把標題陣列寫入（注意要包一層陣列，因為 setValues 接受二維陣列）
  • .setBackground('#1B5E20') — 設定深綠色背景，讓標題列醒目好辨識
  • sheet.setFrozenRows(1) — 凍結第 1 列（捲動時標題不會消失）

📁 replyText() / replyLine() — 回覆訊息函式

function replyText(replyToken, text) {
  replyLine(replyToken, [{ type: 'text', text: text }]);
}

function replyLine(replyToken, messages) {
  UrlFetchApp.fetch('https://api.line.me/v2/bot/message/reply', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({ replyToken: replyToken, messages: messages }),
    muteHttpExceptions: true
  });
}
  • replyToken — Line 每次事件都會附帶一個回覆用的 token，用這個 token 才能回覆訊息（免費，不算推播額度）
  • 程式碼中自動回覆是被註解掉的（// replyText(event.replyToken, '✅ 已記錄你的訊息！');），如需啟用只要把 // 刪除即可
  • pushText vs replyText：push 是主動推播（算額度），reply 是被動回覆（免費）
💡 testSystem() 測試函式：
程式碼最後有一個 testSystem() 函式，它會推播一則測試訊息到你的 Line，用來確認 Token 和 User ID 是否正確。部署前可以先用它來測試。

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？（第一步！）

  1. 在 Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
  2. 會出現執行歷史記錄列表，每次 Webhook 觸發都會留下一筆記錄
  3. 找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
  4. 展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

Verify 失敗

  • 確認部署時「誰可以存取」選了「所有人」
  • 確認部署網址是以 https://script.google.com/macros/s/ 開頭
  • 確認你貼的是「網頁應用程式」的 URL，不是「部署 ID」
  • 如果修改過程式碼，要重新「新增部署作業」（舊的部署不會自動更新！）

試算表沒有新記錄

  • Webhook URL 是否正確設定？→ 回到 Line Developers 確認
  • Use webhook 開關是否為綠色？→ 點開它
  • SPREADSHEET_ID 是否正確？→ 重新從網址列複製
  • 是否已完成授權？→ 執行一次 testSystem 觸發授權流程

圖片沒存到 Drive

  • G 欄顯示「儲存失敗：Folder not found」→ ROOT_FOLDER_ID 不正確
  • G 欄顯示「儲存失敗：Access denied」→ 重新執行授權流程
  • G 欄完全是空的 → 訊息類型可能不是圖片（純文字不會有檔案連結）
  • 確認資料夾沒有被刪除或移到垃圾桶

修改程式碼後沒生效

  • 每次修改程式碼後，必須重新「新增部署作業」
  • 舊的 Webhook URL 會失效 → 新的部署會產生新的 URL
  • 把新的 URL 貼到 Line Developers 的 Webhook URL 欄位
  • 按 Verify 確認新的 URL 可以正常運作
💡 小技巧：如果只是想改設定值（Token、ID），可以用「管理部署作業」→ 編輯（鉛筆圖示）→ 版本選「新版本」→ 部署。這樣 URL 不會改變，就不用重新貼 Webhook URL。
