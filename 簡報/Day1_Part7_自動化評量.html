4 程式碼解析 2 — Webhook 入口

這段的目的：LINE 送來的訊息都會進入 doPost，這是整個系統的大門。doGet 則是用來驗證 Webhook 是否活著。

/** LINE Webhook 接收點 */
function doPost(e) {
  try {
    var events = JSON.parse(e.postData.contents).events;

    for (var i = 0; i < events.length; i++) {
      handleEvent(events[i]);
    }

    return ContentService.createTextOutput(
      JSON.stringify({ status: 'ok' })
    ).setMimeType(ContentService.MimeType.JSON);
  } catch (error) {
    console.error('doPost 錯誤:', error);
    return ContentService.createTextOutput(
      JSON.stringify({ status: 'error' })
    ).setMimeType(ContentService.MimeType.JSON);
  }
}

/** Webhook 驗證用 */
function doGet(e) {
  return ContentService.createTextOutput('自動化評量系統運作中！');
}

逐行解釋：

  • doPost(e) — GAS 規定的 Webhook 入口函式名稱，LINE 每次送訊息都會呼叫它
  • JSON.parse(e.postData.contents).events — 把 LINE 送來的 JSON 文字轉成物件，取出 events 陣列
  • for (var i = 0; ...) — 逐一處理每個事件（通常只有一個，但可能有多個）
  • handleEvent(events[i]) — 呼叫我們自己寫的事件處理函式（下一頁會寫）
  • ContentService.createTextOutput — 回傳「收到了」給 LINE 伺服器
  • try...catch — 任何錯誤都不會讓程式直接崩掉，會記錄到 console
  • doGet(e) — 用瀏覽器直接開 Webhook URL 時，會看到「運作中」的文字
跟 Part 1、2 的結構完全一樣！差別只在 handleEvent 裡面的處理邏輯不同。

5 程式碼解析 3 — 事件處理 & 使用者狀態

這段的目的：根據使用者傳來的不同訊息做出不同回應，並用 PropertiesService 記住每個使用者目前在哪個步驟。

對話流程設計

1. 加入好友

event.type === 'follow'
→ 回覆歡迎訊息

2. 輸入「幫我評語」

setUserState → waiting_for_file
→ 請使用者上傳 PDF

3. 上傳 PDF 檔案

檢查狀態 → 檢查格式
→ 回覆「處理中」→ 開始處理

function handleEvent(event) {
  var userId = event.source.userId;
  var replyToken = event.replyToken;

  // ─── 加入好友歡迎訊息 ───
  if (event.type === 'follow') {
    replyText(replyToken,
      '👋 歡迎使用「自動化評量系統」！\n\n' +
      '📝 使用方式：\n輸入「幫我評語」即可開始\n\n' +
      '我會根據你的學習成果給予專屬評語！');
    return;
  }

  // ─── 處理文字訊息 ───
  if (event.type === 'message' && event.message.type === 'text') {
    var text = event.message.text.trim();
    if (text === '幫我評語') {
      setUserState(userId, 'waiting_for_file');
      replyText(replyToken,
        '📄 請上傳你的自主學習成果 PDF 檔案\n\n' +
        '⚠️ 僅支援 PDF 格式，建議 10MB 以內');
      return;
    }
    replyText(replyToken, '🤖 我是自動化評量機器人\n\n請輸入「幫我評語」開始使用！');
    return;
  }

  // ─── 處理檔案訊息（下一頁） ───
}
為什麼需要狀態管理？LINE Webhook 每次收到訊息都是獨立的請求，GAS 不會記住「這個人剛才說了什麼」。
PropertiesService 就像一本筆記簿，讓我們記住：這個人已經說過「幫我評語」，正在等他上傳檔案。

6 程式碼解析 4 — 檔案處理 & 狀態管理函式

這段的目的：接續上一頁的 handleEvent，處理使用者上傳的檔案訊息，並定義狀態管理的三個工具函式。

handleEvent 的檔案處理段落

  // ─── 處理檔案訊息（接續 handleEvent 內部） ───
  if (event.type === 'message' && event.message.type === 'file') {
    var state = getUserState(userId);

    if (state !== 'waiting_for_file') {
      replyText(replyToken, '請先輸入「幫我評語」再上傳檔案喔！');
      return;
    }

    var fileName = event.message.fileName;
    var messageId = event.message.id;

    // 檢查 PDF 格式
    if (!fileName.toLowerCase().endsWith('.pdf')) {
      replyText(replyToken, '⚠️ 請上傳 PDF 格式的檔案！');
      return;
    }

    // 回覆「處理中」
    replyText(replyToken, '⏳ 正在處理你的檔案，約需 30 秒到 1 分鐘...');

    // 清除狀態 & 非同步處理
    clearUserState(userId);
    processFileAsync(userId, messageId, fileName);
  }
}

使用者狀態管理函式

function setUserState(userId, state) {
  scriptProps.setProperty('state_' + userId, state);
}

function getUserState(userId) {
  return scriptProps.getProperty('state_' + userId) || '';
}

function clearUserState(userId) {
  scriptProps.deleteProperty('state_' + userId);
}

逐行解釋：

  • getUserState(userId) — 檢查這個使用者有沒有在「等待上傳」的狀態
  • fileName.toLowerCase().endsWith('.pdf') — 把檔名轉小寫後檢查結尾是不是 .pdf，防止使用者傳 Word 或圖片
  • replyText(...'⏳ 處理中...') — 先回覆讓使用者知道系統在跑，不是沒反應
  • clearUserState(userId) — 清除狀態，避免使用者重複觸發
  • processFileAsync(...) — 開始核心的檔案處理流程（下一頁說明）
  • setProperty / getProperty / deleteProperty — ScriptProperties 的增/查/刪操作
狀態流程：（無狀態）→ 輸入「幫我評語」→ waiting_for_file → 上傳 PDF →（清除狀態，開始處理）

這段的目的：從 LINE 伺服器下載使用者上傳的 PDF，重新命名後存到 Google Drive 指定資料夾。

📥 downloadFromLine(messageId)

function downloadFromLine(messageId) {
  var url =
    'https://api-data.line.me/v2/bot/message/'
    + messageId + '/content';

  var res = UrlFetchApp.fetch(url, {
    headers: {
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    muteHttpExceptions: true
  });

  if (res.getResponseCode() !== 200) {
    throw new Error('LINE 檔案下載失敗');
  }
  return res.getBlob();

  • api-data.line.me — LINE 檔案下載用的 API 網址（注意不是 api.line.me）
  • messageId — LINE 每則訊息都有唯一 ID，用這個 ID 下載對應的檔案
  • getBlob() — 取得檔案的二進位資料（Blob），可以直接存到 Drive

📁 saveToDrive(blob, fileName, userId)

function saveToDrive(blob, fileName, userId) {
  var folder =
    DriveApp.getFolderById(FOLDER_ID);

  var timestamp = Utilities.formatDate(
    new Date(), 'Asia/Taipei',
    'yyyyMMdd_HHmmss'
  );
  var newName = timestamp + '_'
    + userId.substring(0, 8)
    + '_' + fileName;

  blob.setName(newName);
  return folder.createFile(blob);
}
  • getFolderById(FOLDER_ID) — 用資料夾 ID 找到目標資料夾
  • Utilities.formatDate — 產生台灣時區的時間戳，如 20260309_143025
  • userId.substring(0, 8) — 取使用者 ID 前 8 碼，方便辨識又保護隱私
  • folder.createFile(blob) — 在指定資料夾建立檔案，回傳 File 物件
檔案命名範例：20260309_143025_Uabcdef1_學習成果報告.pdf
日期時間 + 使用者前 8 碼 + 原始檔名，不會重複也方便管理。

8 程式碼解析 6 — PDF 文字提取（關鍵步驟）

這段的目的：利用 Drive API 把 PDF 「轉換」成 Google Docs，轉換後的 Docs 就包含了 PDF 裡的文字，再讀出來送給 AI。

function extractPdfText(file) {
  try {
    var blob = file.getBlob();
    var tempName = 'temp_pdf_' + Date.now();

    // 1. 將 PDF 轉換為 Google Docs
    var docFile = Drive.Files.insert(
      { title: tempName,
        mimeType: 'application/vnd.google-apps.document' },
      blob,
      { convert: true }
    );

    // 2. 讀取轉換後的文字
    var doc = DocumentApp.openById(docFile.id);
    var text = doc.getBody().getText();

    // 3. 清除暫存檔（丟到垃圾桶）
    DriveApp.getFileById(docFile.id).setTrashed(true);

    console.log('PDF 提取成功，文字長度：' + text.length);
    return text;

  } catch (error) {
    console.error('PDF 提取失敗：' + error.message);
    return '';
  }
}

逐行解釋：

  • file.getBlob() — 從 Drive 檔案物件取得二進位資料
  • Drive.Files.insert({...}, blob, { convert: true }) — 這是 Drive API 進階服務（不是 DriveApp）的方法，convert: true 是關鍵參數，告訴 Google 把 PDF 轉成 Docs
  • mimeType: 'application/vnd.google-apps.document' — 指定轉換目標為 Google Docs 格式
  • DocumentApp.openById(docFile.id) — 用 DocumentApp 開啟轉換後的 Google Docs
  • doc.getBody().getText() — 讀取整份文件的純文字內容
  • setTrashed(true) — 用完就丟到垃圾桶，避免 Drive 裡堆滿暫存檔
⚠️ 常見錯誤
ReferenceError: Drive is not defined
→ 代表你忘了啟用 Drive API！回到 Step 2-2 操作。
限制說明
這個方法只能提取「文字型 PDF」。如果是掃描檔（整頁都是圖片），會提取不到文字，程式會回報「無法讀取 PDF 內容」。

generateFeedback — 整理內容並呼叫 AI

function generateFeedback(pdfContent) {
  // 截取內容避免超過 token 限制
  var maxLen = 8000;
  var content = pdfContent.length > maxLen
    ? pdfContent.substring(0, maxLen) + '...(內容已截取)'
    : pdfContent;

  var userPrompt =
    '請根據以下學生的自主學習成果，撰寫 100-150 字的正向評語：\n\n' +
    '===學生作品===\n' + content + '\n===============';

  if (AI_PROVIDER === 'gemini') {
    return callGemini(SYSTEM_PROMPT, userPrompt);
  } else {
    return callOpenAI(SYSTEM_PROMPT, userPrompt);
  }
}

逐行解釋：

  • SYSTEM_PROMPT — 用陣列 .join('\n') 組成多行文字，比用 + 連接更好讀
  • maxLen = 8000 — PDF 文字可能很長，截取前 8000 字避免超過 AI 的 token 上限
  • AI_PROVIDER === 'gemini' — 根據設定區的選擇，呼叫對應的 AI API
Prompt 設計三要素：指定「字數」（100-150 字）、「語氣」（溫暖正向）、「結構」（肯定→優點→建議），AI 生成的品質會大幅提升！

10 程式碼解析 8 — 呼叫 AI API

這段的目的：實際發送 HTTP 請求到 Gemini 或 OpenAI，取得 AI 生成的評語文字。

🟢 callGemini（推薦，免費）

function callGemini(systemPrompt, userPrompt) {
  var url =
    'https://generativelanguage.googleapis.com'
    + '/v1beta/models/' + GEMINI_MODEL
    + ':generateContent?key=' + AI_API_KEY;

  var payload = {
    system_instruction: {
      parts: [{ text: systemPrompt }]
    },
    contents: [{
      parts: [{ text: userPrompt }]
    }],
    generationConfig: {
      temperature: 0.7,
      maxOutputTokens: 500
    }
  };

  var res = UrlFetchApp.fetch(url, {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  });
  var result = JSON.parse(res.getContentText());

  if (result.error) {
    throw new Error(
      'Gemini 錯誤：' + result.error.message
    );
  }
  return result.candidates[0]
    .content.parts[0].text.trim();
}

🔵 callOpenAI（付費方案）

function callOpenAI(systemPrompt, userPrompt) {
  var url =
    'https://api.openai.com/v1/chat/completions';

  var payload = {
    model: OPENAI_MODEL,
    messages: [
      { role: 'system',
        content: systemPrompt },
      { role: 'user',
        content: userPrompt }
    ],
    max_tokens: 500,
    temperature: 0.7
  };

  var res = UrlFetchApp.fetch(url, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + AI_API_KEY
    },
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  });
  var result = JSON.parse(res.getContentText());

  if (result.error) {
    throw new Error(
      'OpenAI 錯誤：' + result.error.message
    );
  }
  return result.choices[0]
    .message.content.trim();
}

關鍵差異：

  • Gemini — API Key 放在 URL 的 ?key= 參數中；回應在 result.candidates[0].content.parts[0].text
  • OpenAI — API Key 放在 HTTP Header 的 Authorization: Bearer；回應在 result.choices[0].message.content
  • temperature: 0.7 — 控制 AI 創意程度（0 = 死板，1 = 天馬行空），0.7 是平衡點

11 程式碼解析 9 — LINE API & 試算表記錄

這段的目的：定義 LINE 的回覆/推播函式，以及把每筆評語自動記錄到 Google Sheets。

💬 LINE 訊息函式

/** 回覆訊息（使用 replyToken） */
function replyText(replyToken, text) {
  UrlFetchApp.fetch(
    'https://api.line.me/v2/bot/message/reply', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      replyToken: replyToken,
      messages: [{ type: 'text', text: text }]
    }),
    muteHttpExceptions: true
  });
}

/** 主動推送訊息（使用 userId） */
function pushText(userId, text) {
  UrlFetchApp.fetch(
    'https://api.line.me/v2/bot/message/push', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      to: userId,
      messages: [{ type: 'text', text: text }]
    }),
    muteHttpExceptions: true
  });
}

📊 logToSheet — 記錄到試算表

function logToSheet(userId, fileName, feedback) {
  try {
    var ss = SpreadsheetApp.openById(
      SPREADSHEET_ID
    );
    var sheet = ss.getSheetByName('評語記錄');

    // 若工作表不存在，自動建立
    if (!sheet) {
      sheet = ss.insertSheet('評語記錄');
      sheet.getRange(1, 1, 1, 5).setValues([
        ['時間', '使用者 ID',
         '檔案名稱', '評語', '狀態']
      ]);
      sheet.getRange(1, 1, 1, 5)
        .setFontWeight('bold');
    }

    var timestamp = Utilities.formatDate(
      new Date(), 'Asia/Taipei',
      'yyyy/MM/dd HH:mm:ss'
    );
    var maskedId =
      userId.substring(0, 10) + '***';

    sheet.appendRow([
      timestamp, maskedId, fileName,
      feedback, '完成'
    ]);
  } catch (error) {
    console.error('試算表記錄錯誤:', error);
  }
}

逐行解釋重點：

  • replyText vs pushText — reply 用 replyToken（30 秒內有效），push 用 userId（隨時可發）
  • ss.getSheetByName('評語記錄') — 找名為「評語記錄」的工作表，找不到就自動建立一個
  • userId.substring(0, 10) + '***' — 只記錄 ID 前 10 碼，保護學生隱私
  • sheet.appendRow([...]) — 在最後一行新增一筆記錄

12 程式碼解析 10 — 核心處理流程

這段的目的：把前面所有步驟串起來！這個函式是整個系統的心臟。

function processFileAsync(userId, messageId, fileName) {
  try {
    // Step 1：從 LINE 下載檔案
    var fileBlob = downloadFromLine(messageId);

    // Step 2：存到 Google Drive
    var driveFile = saveToDrive(fileBlob, fileName, userId);

    // Step 3：提取 PDF 文字
    var pdfText = extractPdfText(driveFile);
    if (!pdfText || pdfText.trim().length < 10) {
      pushText(userId,
        '⚠️ 無法讀取 PDF 內容\n\n' +
        '可能原因：掃描檔或圖片型 PDF\n' +
        '請確保 PDF 內含可選取的文字。');
      return;
    }

    // Step 4：AI 生成評語
    var feedback = generateFeedback(pdfText);

    // Step 5：記錄到試算表
    logToSheet(userId, fileName, feedback);

    // Step 6：回傳評語給學生
    pushText(userId,
      '✨ 你的自主學習評語 ✨\n\n' +
      feedback + '\n\n' +
      '───\n📌 繼續加油！歡迎再次分享學習成果！');

  } catch (error) {
    console.error('processFileAsync 錯誤:', error);
    pushText(userId, '❌ 處理時發生錯誤：' + error.message + '\n\n請稍後再試。');
  }
}

6 步驟流程圖

1️⃣ LINE 下載
→
2️⃣ 存入 Drive
→
3️⃣ 提取文字
→
4️⃣ AI 評語
→
5️⃣ 記錄試算表
→
6️⃣ 推送結果
為什麼用 pushText 不用 replyText？
因為 replyToken 只有 30 秒有效期。PDF 處理（下載 + 轉換 + AI 生成）可能超過 30 秒，
所以先用 replyText 回覆「處理中」，處理完再用 pushText 主動推送結果。

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？（第一步！）

📍 位置：Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個）
👆 動作：點擊它 → 找到最近一筆記錄 → 如果狀態是紅色「失敗」，點擊那一行的 ▶ 展開箭頭
👀 結果：展開後的紅色文字就是錯誤原因

ReferenceError: Drive is not defined

原因：沒有啟用 Drive API 進階服務

解法：GAS 編輯器 → 左側「服務 +」→ 搜尋 Drive API → 新增

Gemini 錯誤：API key not valid

原因：API Key 無效或已過期

解法：到 aistudio.google.com 重新產生 API Key，貼回 AI_API_KEY

LINE 檔案下載失敗（狀態碼 401）

原因：LINE Channel Access Token 不正確

解法：到 LINE Developers 後台重新 Issue Token，貼回 LINE_TOKEN

Folder not found / Invalid argument: id

原因：資料夾 ID 或試算表 ID 貼錯

解法：回到 Google Drive / Sheets，重新從網址列複製 /folders/ 或 /d/ 後面的那段 ID

Gemini 錯誤：429 Resource exhausted

原因：Gemini 免費版有每分鐘請求次數限制

解法：等 1 分鐘再試，或改用 gemini-2.5-flash（配額較高）

Bot 完全沒回覆任何訊息

原因：Webhook URL 未設定或部署類型選錯

解法：確認部署類型是「網路應用程式」，Webhook URL 已正確貼到 LINE Developers，「Use webhook」已開啟
