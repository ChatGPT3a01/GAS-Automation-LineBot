🎯 學習目標

完成這個實作後，你將學會以下四大核心技能：

📥 1. doPost() 接收 Webhook

理解 doPost(e) 函式如何接收來自 Line 平台的 Webhook 事件，解析 JSON 格式的事件資料，取得訊息內容與使用者資訊。

🌐 2. UrlFetchApp 呼叫 API

使用 UrlFetchApp.fetch() 呼叫 Line Messaging API，包含取得使用者資料、下載圖片內容等功能。

📁 3. DriveApp 操作雲端硬碟

使用 DriveApp 將圖片檔案儲存到指定的 Google Drive 資料夾，並取得檔案的分享連結。

💬 4. 處理多種訊息類型

根據訊息類型（文字、圖片、位置、貼圖）做不同處理，建立完整的訊息收集與記錄機制。

請幫我用 Google Apps Script 建立一個 Line Bot 訊息收集器，需求如下：

6. 需要設定 4 個變數：LINE_TOKEN、LINE_USER_ID、ROOT_FOLDER_ID、SPREADSHEET_ID
7. 圖片下載要用 Line 的 Content API：
   https://api-data.line.me/v2/bot/message/{messageId}/content
8. 加上完整中文註解和錯誤處理

完成後，你的 Line Bot 將能自動收集並記錄以下三種訊息：

💬 文字訊息

在 Line 中傳送文字訊息給 Bot，訊息內容會自動記錄到 Google 試算表，包含時間戳記、使用者名稱、訊息內容等欄位。

Line 傳文字 → 試算表新增一列記錄

🖼️ 圖片訊息

傳送圖片給 Bot，圖片會自動下載並存到 Google Drive 指定資料夾，同時在試算表記錄檔案的分享連結。

Line 傳圖片 → Drive 存檔 + 試算表記錄連結

📍 位置訊息

傳送位置資訊給 Bot，經緯度座標會自動記錄到試算表，方便後續地理資訊分析。

Line 傳位置 → 試算表記錄經緯度
💡 實用場景：客服訊息記錄、活動簽到收集、巡檢回報系統、問卷收集等，都能用類似架構實現！

📋 步驟 1 — 建立試算表 + 取得 ID

1-2. 取得試算表 ID

試算表 ID 就在網址中，位於 /d/ 和 /edit 之間的那一段：

🔍 程式碼架構解說

來了解一下這支程式碼的核心架構，每個函式負責什麼工作：

📥 doPost(e)

入口函式 — 接收 Line 平台發來的 Webhook 事件。解析 JSON 資料，判斷訊息類型（text / image / location / sticker），分派給對應的處理函式。

💬 handleTextMessage()

文字訊息處理 — 取得使用者傳來的文字內容，連同時間戳記、使用者名稱等資訊，一併寫入 Google 試算表的新一列。

🖼️ handleImageMessage()

圖片訊息處理 — 透過 Line API 下載圖片的二進制資料，使用 DriveApp 存到指定資料夾，並將檔案連結記錄到試算表。

📍 handleLocationMessage()

位置訊息處理 — 取得使用者分享的地點名稱、地址、經度、緯度等資訊，記錄到試算表中。

😄 handleStickerMessage()

貼圖訊息處理 — 記錄貼圖的 Package ID 和 Sticker ID 到試算表，雖然無法直接顯示貼圖，但可以追蹤使用者的互動。

💡 Webhook 運作原理：當使用者在 Line 傳訊息時，Line 平台會主動發送 HTTP POST 請求到你設定的 Webhook URL（也就是 GAS 的部署網址）。GAS 的 doPost(e) 函式會自動被觸發，參數 e 包含了完整的事件資料。這就是「被動接收」的概念 —— 你不需要主動去問 Line 有沒有新訊息，Line 會主動通知你。

🚀 步驟 5-6 — 部署為網頁應用程式

5. 儲存程式碼

點擊工具列的 💾 儲存按鈕，或按 Ctrl + S。

6. 部署為網頁應用程式

  1. 點選右上角「部署」→「新增部署作業」
  2. 點選齒輪圖示 ⚙️，選擇「網頁應用程式」
  3. 填寫部署設定：
設定欄位 填入值
說明 訊息收集器 v1
執行身分 我（你的 Google 帳號）
誰可以存取 所有人
  1. 點擊「部署」
  2. 第一次部署會要求授權，點「授權存取」→ 選擇你的帳號 → 「進階」→「前往...（不安全）」→「允許」
  3. 複製產生的部署網址（以 https://script.google.com/macros/s/... 開頭）
💡 提醒：「誰可以存取」必須選「所有人」，因為 Line 平台需要從外部存取這個網址來發送 Webhook 事件。
⚠️ 注意：每次修改程式碼後，都需要重新「新增部署作業」才會生效。舊的部署網址不會自動更新。

🔗 步驟 7 — 設定 Webhook URL

⚠️ 重要觀念：一個 Line Bot 同時只能設定一個 Webhook URL！新的 URL 會覆蓋舊的。如果你之前在實作 1 設定過 Webhook URL，這裡設定新的 URL 後，實作 1 的功能就會失效。

設定步驟

  1. 前往 Line Developers Console（developers.line.biz）
  2. 選擇你的 Provider → 選擇你的 Channel
  3. 切換到「Messaging API」分頁
  4. 找到「Webhook URL」區塊
  5. 點擊「Edit」，貼上步驟 6 複製的部署網址
  6. 點擊「Update」儲存
  7. 點擊「Verify」測試連線 → 看到 Success 表示成功
  8. 確認「Use webhook」開關是開啟狀態（綠色）
💡 Verify 的意義：Line 平台會發一個測試請求到你的 Webhook URL，如果 GAS 正確回應（HTTP 200），就會顯示 Success。如果失敗，請檢查部署網址是否正確、是否已完成授權。
✅ 小技巧：如果想同時保留實作 1 的功能，可以在程式碼中將兩個實作的邏輯合併到同一個 doPost(e) 函式中！

⚠️ 常見問題排除：
  • 試算表沒有新記錄？ → 檢查 Webhook URL 是否正確設定、Use webhook 是否開啟
  • 圖片沒有存到 Drive？ → 檢查 ROOT_FOLDER_ID 是否正確、是否已授權 Drive 存取權限
  • Verify 失敗？ → 確認部署時「誰可以存取」是選「所有人」
  • 出現權限錯誤？ → 重新執行一次授權流程（部署 → 授權存取）
  • 試算表 ID 或資料夾 ID 有誤？ → 重新從網址列複製，注意不要多複製空格或斜線
💡 除錯技巧：到 GAS 編輯器 →「執行項目」查看執行記錄，可以看到每次 Webhook 觸發的 log，幫助找出問題所在。
