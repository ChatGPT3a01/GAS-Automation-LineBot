🔍 程式碼解析 1 — createSignInSheet()

這段的目的：建立一個格式漂亮的簽到表，包含標題列、欄寬、背景色、凍結列等完整格式。

function createSignInSheet() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);

  // 如果已存在，先刪除
  if (sheet) {
    ss.deleteSheet(sheet);
  }

  // 建立新的工作表
  sheet = ss.insertSheet(SIGNIN_SHEET_NAME);

  // ===== 標題列 =====
  var headers = ['序號', '時間戳記', '姓名', '狀態', '備註'];
  sheet.getRange(1, 1, 1, headers.length).setValues([headers]);

  // 格式化標題列
  var headerRange = sheet.getRange(1, 1, 1, headers.length);
  headerRange.setFontWeight('bold');
  headerRange.setFontSize(12);
  headerRange.setBackground('#1565C0');
  headerRange.setFontColor('#FFFFFF');
  headerRange.setHorizontalAlignment('center');
  headerRange.setBorder(true, true, true, true, true, true);

  // 設定欄寬 + 凍結標題列
  sheet.setColumnWidth(1, 60);    // 序號
  sheet.setColumnWidth(2, 200);   // 時間戳記
  sheet.setColumnWidth(3, 120);   // 姓名
  sheet.setColumnWidth(4, 80);    // 狀態
  sheet.setColumnWidth(5, 200);   // 備註
  sheet.setFrozenRows(1);
}

逐行解釋：

  • SpreadsheetApp.getActiveSpreadsheet() — 取得目前綁定的試算表
  • ss.getSheetByName('簽到表') — 用名稱找工作表，找不到回傳 null
  • ss.deleteSheet(sheet) — 如果已存在就先刪除（重新建立）
  • ss.insertSheet('簽到表') — 建立一個新的工作表
  • getRange(1, 1, 1, 5).setValues([headers]) — 在第 1 列第 1 欄開始，寫入 1 列 5 欄的標題
  • setFontWeight('bold') — 粗體文字
  • setBackground('#1565C0') — 深藍色背景
  • setFontColor('#FFFFFF') — 白色文字
  • setBorder(true, true, true, true, true, true) — 上、左、下、右、垂直、水平 全框線
  • setFrozenRows(1) — 凍結第 1 列，往下捲動時標題列不會消失

🔍 程式碼解析 2 — signIn(name, note)

這段的目的：單筆簽到 — 自動判斷準時/遲到、寫入試算表、推播 Line 確認。

function signIn(name, note) {
  name = name || '未填姓名';
  note = note || '';

  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);

  if (!sheet) {
    createSignInSheet();
    sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);
  }

  // 計算序號
  var serialNumber = sheet.getLastRow();

  // 取得目前時間
  var now = new Date();
  var timestamp = now.toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' });

  // 判斷簽到狀態
  var hour = now.getHours();
  var minute = now.getMinutes();
  var status = (hour < 9 || (hour === 9 && minute === 0))
    ? '✅ 準時' : '⚠️ 遲到';

  // 新增一列
  sheet.appendRow([serialNumber, timestamp, name, status, note]);
}

逐行解釋：

  • name = name || '未填姓名' — 如果沒傳入姓名，預設為「未填姓名」（防呆設計）
  • sheet.getLastRow() — 取得目前最後一列的列號，用來計算新序號
  • new Date() — 取得「現在這一刻」的時間物件
  • toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' }) — 格式化為台灣時間的字串
  • now.getHours() — 取得現在的「時」（0~23），用來判斷是否遲到
  • (hour < 9 || (hour === 9 && minute === 0)) — 9:00（含）以前算準時
  • sheet.appendRow([...]) — 在試算表最後一列新增一筆資料
💡 appendRow() 的特性：
appendRow() 永遠把資料加在試算表的最後一列。你不需要指定要寫到第幾列，它會自動找到最後面。每呼叫一次就新增一列，不會覆蓋現有資料。

🔍 程式碼解析 4 — batchSignIn()

這段的目的：示範 setValues() 批次寫入 — 一次把 5 筆資料寫入試算表，效率遠高於逐筆 appendRow。

function batchSignIn() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);

  // 準備批次資料
  var now = new Date();
  var timestamp = now.toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' });
  var startSerial = sheet.getLastRow();

  var batchData = [
    [startSerial + 0, timestamp, '陳大文', '✅ 準時', ''],
    [startSerial + 1, timestamp, '林小美', '✅ 準時', ''],
    [startSerial + 2, timestamp, '張志明', '⚠️ 遲到', '塞車'],
    [startSerial + 3, timestamp, '王小華', '✅ 準時', ''],
    [startSerial + 4, timestamp, '李建國', '⚠️ 遲到', '家裡有事']
  ];

  // 使用 setValues() 一次寫入所有資料
  var startRow = sheet.getLastRow() + 1;
  sheet.getRange(startRow, 1, batchData.length, 5).setValues(batchData);
}

逐行解釋：

  • batchData — 二維陣列，每個子陣列 [...] 就是一列資料
  • sheet.getLastRow() + 1 — 從現有資料的下一列開始寫入
  • getRange(startRow, 1, batchData.length, 5) — 選取目標範圍：起始列、起始欄、列數、欄數
  • .setValues(batchData) — 一次把整個二維陣列寫入選取的範圍

getRange() 四個參數圖解

sheet.getRange(startRow, 1, batchData.length, 5)
//           起始列     起始欄  共幾列              共幾欄
//           第6列      第A欄   5列（5筆資料）      5欄（序號~備註）
💡 appendRow vs setValues 效能差異：
假設要寫入 100 筆資料：appendRow 要呼叫 100 次 API，setValues 只要呼叫 1 次。資料量越大，差距越明顯！

🔍 程式碼解析 5 — sendSignInSummary()

這段的目的：讀取簽到表所有資料，統計準時/遲到人數，組合成報表推播到 Line。

function sendSignInSummary() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);

  if (!sheet || sheet.getLastRow() <= 1) {
    pushText(LINE_USER_ID, '📋 簽到統計：尚無簽到紀錄。');
    return;
  }

  var data = sheet.getRange(2, 1, sheet.getLastRow() - 1, 5).getValues();

  var total = data.length;
  var onTime = 0;
  var late = 0;

  for (var i = 0; i < data.length; i++) {
    var status = String(data[i][3]);
    if (status.indexOf('準時') !== -1) {
      onTime++;
    } else if (status.indexOf('遲到') !== -1) {
      late++;
    }
  }

  var summary = '📊 簽到統計報表\n';
  summary += '━━━━━━━━━━━━\n';
  summary += '👥 總簽到人數：' + total + ' 人\n';
  summary += '✅ 準時：' + onTime + ' 人\n';
  summary += '⚠️ 遲到：' + late + ' 人\n';
  summary += '⏱ 準時率：' + Math.round(onTime / total * 100) + '%';

  pushText(LINE_USER_ID, summary);
}

逐行解釋：

  • sheet.getLastRow() <= 1 — 如果只有標題列（或空表），就沒資料可統計
  • getRange(2, 1, sheet.getLastRow() - 1, 5) — 從第 2 列開始（跳過標題），讀取所有資料列
  • .getValues() — 讀取指定範圍的所有值，回傳二維陣列
  • data[i][3] — 第 i 列的第 4 欄（索引從 0 起算），也就是「狀態」欄
  • String(data[i][3]) — 確保轉為字串再做 indexOf 搜尋
  • Math.round(onTime / total * 100) — 計算準時率百分比，四捨五入
💡 getValues() vs getValue()：
getValues() 回傳二維陣列（多列多欄），getValue() 回傳單一儲存格的值。讀取多筆資料時一定用 getValues()，效率高出許多。

🔍 程式碼解析 6 — Line Bot 推播函式

這段的目的：提供 pushText() 和 pushLine() 兩個工具函式，讓程式碼可以送訊息到 Line。

📁 pushText(to, text) — 推播文字訊息

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}
  • to — 要推播給誰（就是 LINE_USER_ID）
  • text — 要傳送的文字內容
  • 它其實是呼叫下面的 pushLine()，把文字包成 Line 要求的格式

📁 pushLine(to, messages) — 推播底層函式

function pushLine(to, messages) {
  UrlFetchApp.fetch('https://api.line.me/v2/bot/message/push', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({ to: to, messages: messages }),
    muteHttpExceptions: true
  });
}
  • UrlFetchApp.fetch() — GAS 內建的 HTTP 請求工具，等同於瀏覽器發送網路請求
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 告訴 Line「我是哪一個 Bot」的身份驗證
  • JSON.stringify() — 把 JavaScript 物件轉成 JSON 格式的文字
  • muteHttpExceptions: true — 即使發生錯誤也不要中斷程式，方便除錯
💡 這段跟前面實作一樣！
如果你已經完成 Part 3、Part 4，會發現推播函式的寫法幾乎相同。這就是「共用模組」的概念 — 同樣的功能不需要重複寫，直接複製過來用。

⚡ appendRow vs setValues 完整比較

這是本實作最核心的技術對比，務必理解兩者的差異與適用場景。

📝 appendRow() — 逐筆新增

  • 一次新增一列資料
  • 自動找到最後一行，簡單直覺
  • 適合即時寫入（如使用者簽到）
// 一次一列
sheet.appendRow([1, '小明', '準時']);
sheet.appendRow([2, '小華', '遲到']);
sheet.appendRow([3, '小美', '準時']);
// 3 筆 = 呼叫 3 次 API

📊 setValues() — 批次寫入

  • 一次寫入多列資料
  • 需指定寫入範圍（getRange）
  • 適合大量匯入、批次更新
// 一次多列（效率高！）
var data = [
  [1, '小明', '準時'],
  [2, '小華', '遲到'],
  [3, '小美', '準時']
];
sheet.getRange(2, 1, 3, 3)
     .setValues(data);
// 3 筆 = 呼叫 1 次 API
appendRow() setValues()
寫入方式逐筆新增到最後一列批次寫入指定範圍
需指定範圍不需要（自動加在最後）需要（getRange 指定）
效能（100 筆）呼叫 100 次 API呼叫 1 次 API
適用場景即時單筆（簽到、表單）大量匯入、批次更新
程式碼sheet.appendRow([...])range.setValues([[...]])
🚀 效能差異實測：寫入 500 筆資料時，appendRow 迴圈需要約 120 秒，setValues 只要約 2 秒。差距高達 60 倍！

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？（第一步！）

  1. 在 Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
  2. 會出現執行歷史記錄列表，每次執行都會留下一筆記錄
  3. 找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
  4. 展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

「Cannot call getActiveSpreadsheet」

  • 表示程式碼找不到綁定的試算表
  • 解決：確認你是從「簽到系統」試算表 → 擴充功能 → Apps Script 進入的
  • 不要從 script.google.com 直接建立新專案

收不到 Line 推播

  • 檢查 LINE_TOKEN 是否正確貼入（沒有多餘空格、單引號完整）
  • 檢查 LINE_USER_ID 是否為 U 開頭的字串
  • 確認 Line Bot 已加為好友（用手機掃 QR Code）
  • 檢查執行記錄是否有「推播失敗，狀態碼：401」→ Token 不正確

簽到表沒有出現

  • 切回試算表分頁，看下方的工作表標籤是否有「簽到表」
  • 如果沒有，表示 createSignInSheet() 沒有成功執行
  • 回到 Apps Script 看執行記錄有無紅色錯誤
  • 嘗試重新執行 createSignInSheet()

「setValues 資料大小不符」

  • 錯誤訊息通常是 The number of rows/columns in the data does not match
  • 表示 getRange() 指定的範圍大小與 setValues() 的陣列大小不一致
  • 解決：確認 getRange 的列數和欄數與資料陣列完全吻合
  • 例如 5 筆 5 欄的資料 → getRange(row, 1, 5, 5)
