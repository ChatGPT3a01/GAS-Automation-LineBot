💡 為什麼需要 User ID？

Line Bot 有兩種傳訊息的方式，差別在於「誰先開口」：

💬 回覆（Reply）

情境：使用者先傳訊息給 Bot，Bot 回覆

需要：replyToken（自動夾帶，不用另外找）

限制：只能在使用者傳訊後的短時間內回覆

不需要知道 userId！

📣 推播（Push）

情境：Bot 主動傳訊息給使用者（使用者沒先說話）

需要：userId 或 groupId（必須事先取得）

限制：免費帳號每月 500 則

必須知道 userId！

⚠️ 常見誤解：
- userId 不是你的 LINE ID（就是加好友那個 @xxx）
- userId 是一串 U 開頭、共 33 個字元的字串，例如 U1234567890abcdef1234567890abcdef
- 同一個人在不同的 Line Bot Channel 會有不同的 userId

1 方法一：從試算表查看

如果你在實作 11 已經做好「訊息收集器」，最簡單的方法就是直接從試算表找！

Step 1-1：打開訊息收集器的試算表

👆 動作：打開你在實作 11 建立的 Google 試算表（訊息收集器會把所有收到的訊息存在這裡）
👀 結果：你會看到試算表裡有好幾列資料，每一列代表一則收到的訊息

Step 1-2：找到 userId 欄位

📍 位置：試算表的欄標題（第一列）
👆 動作：找到名為「userId」或「來源ID」的那一欄
👀 結果：該欄下方會有一串 U 開頭的長字串

Step 1-3：複製 userId

👆 動作：點擊含有 U 開頭字串的儲存格，按 Ctrl+C 複製
👀 結果：你已經拿到你的 userId 了！把它貼到其他 GAS 專案的 LINE_USER_ID 設定值中即可
💡 這是最推薦的方法！因為不需要修改任何程式碼，直接從已經存好的資料中查找即可。如果試算表裡沒有資料，先用手機傳一則訊息給你的 Line Bot。

2 方法二：用 Logger.log() 記錄

如果你沒有訊息收集器，也可以在任何 Line Bot 的 doPost() 裡面加一行 log。

Step 2-1：在 doPost() 開頭加入 Logger

📍 位置：你的 Line Bot GAS 專案，找到 doPost(e) 函式
👆 動作：在函式的第一行加入以下程式碼： 
function doPost(e) {
  // ===== 加入這兩行，把完整的 Webhook 資料印出來 =====
  var data = JSON.parse(e.postData.contents);
  Logger.log('Webhook 資料：' + JSON.stringify(data));

  // 取出 userId
  var userId = data.events[0].source.userId;
  Logger.log('userId = ' + userId);

  // ... 你原本的程式碼繼續寫在下面 ...
}

Step 2-2：重新部署

👆 動作：點擊右上角 部署 → 管理部署 → 鉛筆圖示 → 版本選「新版本」→ 部署
👀 結果：新版本已上線

Step 2-3：傳訊息並查看 Log

👆 動作：用手機傳一則訊息給你的 Line Bot
👆 動作：回到 GAS 編輯器，點擊左側「⏱ 執行項目」
👀 結果：找到最近一筆 doPost 的執行記錄，展開就能看到 userId = U...
⚠️ 注意：Logger.log 的記錄只會保留一段時間。看到 userId 後，趕快複製下來存好！確認拿到後，可以把這兩行 Logger 刪除，避免影響效能。

3 方法三：取得群組 groupId

如果你想讓 Bot 推播訊息到群組（而不是個人），就需要 groupId。

Step 3-1：把 Bot 加入群組

👆 動作：在 LINE 手機 App 中，建立一個群組（或使用現有群組）
👆 動作：在群組中點擊「邀請」，把你的 Line Bot 加入群組
👀 結果：群組中會顯示 Bot 已加入

Step 3-2：在群組中傳一則訊息

👆 動作：在群組中隨便傳一則訊息（任何文字都行）
👀 結果：這則訊息會觸發 Bot 的 doPost()

Step 3-3：從 Logger 或試算表取得 groupId

👆 動作：查看 Logger 或試算表中的 Webhook 資料
👀 結果：你會看到 source 欄位中有 groupId，格式為 C 開頭的長字串 
// 群組訊息的 Webhook 資料結構
{
  "events": [{
    "type": "message",
    "source": {
      "type": "group",              // ← 注意這裡是 "group"
      "groupId": "Cab1234567890...", // ← 這就是 groupId
      "userId": "U1234567890..."    // ← 傳訊息的人的 userId
    },
    "message": {
      "type": "text",
      "text": "你好"
    }
  }]
}
💡 重點：群組訊息的 source.type 是 "group"，而且同時包含 groupId（群組的 ID）和 userId（傳訊息那個人的 ID）。推播到群組時，用 groupId 即可。

🔀 userId vs groupId 差異比較

兩者用途不同，搞清楚什麼時候用哪一個：

比較項目userIdgroupId
格式 U 開頭，33 個字元 C 開頭，33 個字元
代表 一個 LINE 使用者 一個 LINE 群組
推播效果 只有那個人收到訊息 群組裡所有人都看到
取得方式 使用者傳訊息時，從 source.userId 取得 在群組中傳訊息時，從 source.groupId 取得
唯一性 同一人在不同 Channel 有不同 userId 同一群組在不同 Channel 有不同 groupId
常見用途 個人通知、個人查詢結果 班級通知、團隊公告

程式碼中怎麼用？

// 推播給個人
pushText('U1234567890abcdef...', '你好，這是個人訊息');

// 推播到群組
pushText('Cab1234567890abcdef...', '大家好，這是群組公告');

// 兩個函式的用法完全一樣！
// 差別只在於第一個參數放 userId 還是 groupId

⚠️ 注意事項

❌ userId 不等於 LINE ID

你在 LINE 設定中看到的「LINE ID」（例如 @mylineid）是給好友搜尋用的，跟程式用的 userId 完全不同。

LINE ID：你自己設定的，可讀的暱稱

userId：系統自動產生的 U 開頭字串，無法修改

❌ 不同 Channel 的 userId 不同

同一個人對你的 Bot A 和 Bot B，會有兩個不同的 userId。

所以從 Bot A 拿到的 userId，不能拿去 Bot B 用！每個 Bot（Channel）都要各自收集 userId。

💡 userId 不會過期

一旦取得了 userId，只要那個人沒有封鎖你的 Bot，這個 userId 就會一直有效。可以存在程式碼的常數裡面。

💡 Bot 需要開啟「加入群組」

要取得 groupId，Bot 必須能被邀請加入群組。到 LINE Developers Console → Bot 設定 → 確認「Allow bot to join group chats」是開啟的。

⚠️ 安全提醒：userId 和 groupId 是敏感資訊，不要公開分享。雖然光有 userId 不能做什麼壞事（還需要 Channel Access Token），但為了安全起見，建議妥善保管。
