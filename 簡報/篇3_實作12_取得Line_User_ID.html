💡 這個實作在做什麼？
我們要建一個「ID 查詢 Bot」。你傳任何訊息給這個 Bot，它就會直接回覆你的 User ID。
拿到 ID 後，就可以貼到其他實作的 LINE_USER_ID 設定裡面使用。

💡 為什麼需要 User ID？

Line Bot 有兩種傳訊息的方式：

💬 回覆（Reply）

使用者先說話 → Bot 回覆

不需要知道 userId

用 replyToken 就能回

像是「有人問你，你回答」

📣 推播（Push）

Bot 主動傳訊息（使用者沒先說話）

必須知道 userId 或 groupId

用 pushText(userId, '...')

像是「你主動傳訊息給朋友」

⚠️ userId 不是 LINE ID！
userId 是系統產生的 U 開頭長字串（例如 U1234567890abcdef...），跟你設定的 LINE ID（@xxx）完全不同。只能透過程式取得，沒辦法自己查。

Step 2：複製 userId

👆 動作：在 LINE 中長按那個 U 開頭的字串 → 點「複製」

✅ 成功的樣子

  • Bot 秒回你的 userId（U 開頭長字串）

  • 群組聊天：額外顯示 groupId

❌ 失敗怎麼辦

6 額外：取得群組 groupId

如果你想讓 Bot 推播到群組，也只要一個動作：

Step 1：把 Bot 加入群組

👆 動作：在 LINE 群組中 → 點「邀請」→ 找到你的 Bot → 邀請加入
👀 結果：Bot 出現在群組成員中

Step 2：在群組中傳一則訊息

👆 動作：在群組裡隨便打一句話
👀 結果：Bot 會回覆 userId 和 groupId！ 
📋 你的 Line ID 資訊
━━━━━━━━━━━━━━━

👤 你的 userId：
U1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6

👥 群組 groupId：
Cab1234567890abcdef1234567890abcd

💡 推播給個人 → 用 userId
💡 推播到群組 → 用 groupId
💡 何時用哪個？
userId：推播給個人（例如：通知你自己、查詢結果回覆）
groupId：推播到群組（例如：班級通知、團隊公告）

7 取得 ID 後 — 記得改回 Webhook

⚠️ 很重要！取得 userId 後，記得把 Webhook URL 改回原本的網址。
否則你的 Bot 會一直只回覆 ID，其他功能都不會動了！

Step 1：改回 Webhook URL

👆 動作：到 LINE Developers Console → Messaging API → Webhook settings
👆 動作：把 Webhook URL 改回你之前備份的那個網址
👆 動作：按 Save
👀 結果：Bot 恢復原本的功能

你拿到的 ID 要怎麼用？

把 userId 貼到其他實作的程式碼裡面：

// 在其他實作的「設定區」，找到這行：
var LINE_USER_ID = '在此貼上你的 User ID';

// 把你拿到的 userId 貼進去：
var LINE_USER_ID = 'U1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6';
💡 userId 不會過期！只要那個人沒有封鎖你的 Bot，userId 就永遠有效。存好一次就夠了。

🔧 常見問題排解

問題原因解決方法
Bot 完全沒回覆 Webhook URL 沒設定 確認 LINE Developers 的 Webhook URL 已貼上 GAS 部署網址
Bot 回覆罐頭訊息 沒部署或沒選「新版本」 部署 → 管理部署 → 鉛筆圖示 → 版本選「新版本」→ 部署
授權視窗沒出現 已經授權過了 這是正常的，如果之前已授權就不會再跳
userId 顯示 undefined LINE Token 無效 重新從 LINE Developers 複製 Token
在群組傳訊息沒回覆 Bot 沒被加入群組 先邀請 Bot 加入群組，再傳訊息
⚠️ 除錯方式：到 GAS 編輯器左側的「⏱ 執行項目」查看錯誤記錄。如果沒有任何記錄，代表 LINE 根本沒呼叫到你的 GAS → 檢查 Webhook URL。

📖 程式碼解說

整個程式碼只做一件事：收到訊息 → 回覆 ID。超級簡單！

核心邏輯（doPost 函式）

function doPost(e) {
  // 1. 解析 LINE 傳來的資料
  var data = JSON.parse(e.postData.contents);
  var event = data.events[0];
  var source = event.source;

  // 2. 組合回覆訊息
  var reply = '👤 你的 userId：\n' + source.userId;

  // 3. 如果是群組，額外顯示 groupId
  if (source.type === 'group') {
    reply += '\n👥 群組 groupId：\n' + source.groupId;
  }

  // 4. 回覆給使用者
  replyLine(event.replyToken, [{ type: 'text', text: reply }]);
}

source.userId

傳訊息的那個人的 ID。不管是個人聊天還是群組，都會有這個值。

source.groupId

只有在群組中傳訊息時才會有。個人聊天不會有這個值。
