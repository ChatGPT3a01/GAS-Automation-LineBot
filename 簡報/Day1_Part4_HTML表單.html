🎯 學習目標

完成這個實作後，你將學會以下四大核心能力：

📄 1. HtmlService 建立網頁介面

使用 GAS 內建的 HtmlService 服務，建立具有完整 HTML/CSS/JS 的互動式網頁介面，不需要額外的伺服器或網域。

🌐 2. doGet() 回傳 HTML 頁面

透過 doGet() 函式攔截 HTTP GET 請求，將自訂的 HTML 檔案回傳給瀏覽器，讓使用者看到漂亮的表單頁面。

🔗 3. google.script.run 前端呼叫後端

學會使用 google.script.run 這個神奇的橋梁，讓前端 HTML 頁面能呼叫後端 GAS 函式，實現前後端溝通。

掌握從開發到部署的完整流程：撰寫程式碼 → 新增 HTML 檔案 → 設定權限 → 部署為網頁應用程式 → 取得公開網址。

🔗 前端程式碼解說：google.script.run

核心橋梁：前端呼叫後端

當使用者在 HTML 表單中點擊「送出」按鈕時，前端 JavaScript 會透過 google.script.run 來呼叫後端 GAS 函式：

google.script.run
  .withSuccessHandler(onSuccess)    // 成功時執行
  .withFailureHandler(onFailure)    // 失敗時執行
  .processForm(formData);           // 呼叫後端的 processForm 函式
💡 核心概念：google.script.run 是前端（HTML）呼叫後端（GAS）的橋梁。它讓你在瀏覽器中執行的 JavaScript，能夠遠端呼叫伺服器上的 GAS 函式！

完整資料流程

前端表單 → google.script.run → 後端 processForm() → 寫入試算表 + 推播 Line

方法 說明
.withSuccessHandler(fn) 後端函式成功執行後，呼叫前端的 fn 函式處理回傳值
.withFailureHandler(fn) 後端函式發生錯誤時，呼叫前端的 fn 函式處理錯誤
.processForm(formData) 呼叫後端名為 processForm 的函式，並傳入 formData 參數

⚙️ 後端 processForm() 解說

processForm() 函式做了三件事

當前端透過 google.script.run.processForm(formData) 呼叫後端時，這個函式會依序執行：

function processForm(formData) {
  // 1. 取得試算表
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var timestamp = new Date().toLocaleString('zh-TW');

  // 2. 寫入試算表（新增一列資料）
  sheet.appendRow([
    timestamp,
    formData.name,
    formData.email,
    formData.message
  ]);

  // 3. 組合 Line 通知訊息
  var rowCount = sheet.getLastRow() - 1;
  var msg = '📬 收到新的表單填寫！\n'
          + '━━━━━━━━━━━━\n'
          + '👤 姓名：' + formData.name + '\n'
          + '📧 Email：' + formData.email + '\n'
          + '💬 訊息：' + formData.message + '\n'
          + '━━━━━━━━━━━━\n'
          + '📅 時間：' + timestamp + '\n'
          + '📊 累計第 ' + rowCount + ' 筆';

  // 4. 推播 Line 通知
  pushText(LINE_USER_ID, msg);

  // 5. 回傳成功訊息給前端
  return { success: true, message: '表單已成功送出！' };
}

📱 第二步

組合通知訊息，透過 LINE Messaging API 推播給你

✅ 第三步

回傳成功訊息給前端，讓使用者看到「送出成功」提示
