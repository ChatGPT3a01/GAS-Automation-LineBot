什麼是 RSS？

📡 RSS = Really Simple Syndication

RSS 是一種標準化的新聞訂閱格式，網站把最新內容用 XML 格式公開，任何人都可以「訂閱」自動取得更新。

🏠 比喻：新聞宅配服務

  • 傳統方式：你每天打開 10 個新聞網站，一個個看
  • RSS 方式：所有新聞「送到你家門口」，你只要開信箱
  • GAS + RSS：不只送到門口，還幫你整理成摘要推到 Line！

兩種常見 RSS 格式

  • RSS 2.0 — 用 <channel> → <item> 結構（最常見）
  • Atom — 用 <entry> 結構（較新的格式）

我們的程式碼兩種都支援！

📡 程式碼解析 1 — RSS 來源設定

這段的目的：定義我們要抓取哪些新聞來源，以及每個來源要取幾則新聞。

// ========== RSS 來源設定 ==========
// 可以自行新增或修改 RSS 來源
var RSS_FEEDS = [
  { name: '中央社',     url: 'https://feeds.feedburner.com/caboraa' },
  { name: '科技新報',   url: 'https://technews.tw/feed/' },
  { name: '數位時代',   url: 'https://www.bnext.com.tw/rss' }
];

// 每個來源取幾則新聞
var NEWS_COUNT = 3;

逐行解釋：

  • var RSS_FEEDS = [...] — 宣告一個陣列（清單），裡面放了多個 RSS 來源
  • { name: '中央社', url: '...' } — 每個來源是一個物件，包含兩個屬性：name（顯示名稱）和 url（RSS 網址）
  • name — 推播訊息中會顯示的來源名稱，例如「【中央社】」
  • url — RSS Feed 的網址，GAS 會去這個網址抓取 XML 資料
  • var NEWS_COUNT = 3 — 每個來源最多取 3 則新聞。如果覺得訊息太長，可以改成 2；太短可以改成 5
💡 陣列的概念：
RSS_FEEDS 是一個陣列（用 [ ] 包起來），裡面可以放無限多個來源。想加新來源？只要多加一行 { name: '名稱', url: '網址' } 就好。想移除來源？把那一行刪掉即可。
⚠️ 注意 RSS 來源數量：
LINE 推播訊息上限 5000 字。3 個來源、每個 3 則 = 9 則新聞，通常不會超過。如果你加到 6 個以上來源，可能會超過字數限制，程式碼有內建截斷處理（在主函式中）。

🔍 程式碼解析 2 — fetchRSS 函式

這段的目的：抓取一個 RSS Feed 的 XML 資料，解析出新聞標題、連結、發布時間，回傳一個新聞陣列。同時支援 RSS 2.0 和 Atom 兩種格式。

📡 fetchRSS(feedUrl) — 抓取並解析 RSS

function fetchRSS(feedUrl) {
  try {
    // 呼叫 RSS 網址
    var response = UrlFetchApp.fetch(feedUrl, {
      muteHttpExceptions: true,
      followRedirects: true
    });

    if (response.getResponseCode() !== 200) {
      Logger.log('RSS 抓取失敗：' + feedUrl + '（狀態碼：' + response.getResponseCode() + '）');
      return [];
    }

    // 解析 XML
    var xml = XmlService.parse(response.getContentText());
    var root = xml.getRootElement();

    // RSS 2.0 格式：root > channel > item
    // Atom 格式：root > entry
    var items = [];
    var namespace = root.getNamespace();

    // 嘗試 RSS 2.0 格式
    var channel = root.getChild('channel');
    if (channel) {
      var rssItems = channel.getChildren('item');
      for (var i = 0; i < Math.min(rssItems.length, NEWS_COUNT); i++) {
        var item = rssItems[i];
        items.push({
          title: item.getChildText('title') || '無標題',
          link: item.getChildText('link') || '',
          pubDate: item.getChildText('pubDate') || ''
        });
      }
    }

    // 嘗試 Atom 格式（省略部分程式碼，完整版在 .gs 檔案中）
    // ...

    Logger.log('從 ' + feedUrl + ' 取得 ' + items.length + ' 則新聞');
    return items;

  } catch (error) {
    Logger.log('解析 RSS 發生錯誤：' + error.message);
    return [];
  }
}

逐行解釋（核心部分）：

  • UrlFetchApp.fetch(feedUrl, {...}) — 就像在瀏覽器輸入網址一樣，從網路抓取 RSS 的 XML 文字
  • muteHttpExceptions: true — 即使 HTTP 請求失敗，也不要中斷程式（讓我們自己檢查錯誤）
  • followRedirects: true — 如果網址有跳轉（301/302），自動跟著跳轉
  • response.getResponseCode() — 取得 HTTP 狀態碼，200 = 成功
  • XmlService.parse(response.getContentText()) — 把抓到的 XML 文字轉成可操作的 Document 物件
  • xml.getRootElement() — 取得 XML 的根元素（通常是 <rss> 或 <feed>）
  • root.getChild('channel') — 取得 <channel> 子元素（RSS 2.0 格式）
  • channel.getChildren('item') — 取得所有 <item> 子元素（每個 item = 一則新聞）
  • item.getChildText('title') — 直接取得 <title> 元素的文字內容
  • catch (error) — 如果任何步驟出錯，印出錯誤訊息並回傳空陣列，不影響其他來源

📋 程式碼解析 3 — buildNewsMessage 訊息組合

這段的目的：從所有 RSS 來源取得新聞，把它們組合成一則格式化的推播訊息。

📰 buildNewsMessage() — 組合新聞訊息

function buildNewsMessage() {
  var now = new Date();
  var dateStr = Utilities.formatDate(now, 'Asia/Taipei', 'yyyy/MM/dd (EEE)');

  var message = '📰 每日新聞摘要\n';
  message += '📅 ' + dateStr + '\n';
  message += '━━━━━━━━━━━━━━━\n';

  var hasNews = false;

  for (var i = 0; i < RSS_FEEDS.length; i++) {
    var feed = RSS_FEEDS[i];
    Logger.log('正在抓取：' + feed.name + '（' + feed.url + '）');

    var items = fetchRSS(feed.url);

    if (items.length > 0) {
      hasNews = true;
      message += '\n【' + feed.name + '】\n';

      for (var j = 0; j < items.length; j++) {
        var item = items[j];
        var time = formatDate(item.pubDate);
        message += '\n' + (j + 1) + '. ' + item.title;
        if (time) message += '\n   🕐 ' + time;
        if (item.link) message += '\n   🔗 ' + item.link;
        message += '\n';
      }
    }
  }

  if (!hasNews) {
    message += '\n目前沒有取得到新聞，請檢查 RSS 來源設定。';
  }

  message += '\n━━━━━━━━━━━━━━━';
  message += '\n💡 由 GAS 自動推播';

  return message;
}

逐行解釋：

  • Utilities.formatDate(now, 'Asia/Taipei', 'yyyy/MM/dd (EEE)') — 格式化今天的日期，例如 2026/03/09 (Sun)
  • for (var i = 0; i < RSS_FEEDS.length; i++) — 用 for 迴圈逐一處理每個 RSS 來源
  • var items = fetchRSS(feed.url) — 呼叫前一頁學到的 fetchRSS 函式，取得該來源的新聞陣列
  • message += '【' + feed.name + '】\n' — 在訊息中加入來源名稱標題，例如「【中央社】」
  • var time = formatDate(item.pubDate) — 呼叫 formatDate 函式，把日期轉成簡短格式（例如 03/09 08:30）
  • if (!hasNews) — 如果所有來源都抓不到新聞，顯示錯誤提醒
  • \n — 換行符號，讓訊息有段落分隔
💡 formatDate 輔助函式：
程式碼中還有一個 formatDate(dateStr) 函式（完整版在 .gs 檔案第 172-180 行），負責把 RSS 提供的各種日期格式轉成統一的 MM/dd HH:mm 格式。

📋 程式碼解析 4 — 主函式 sendDailyNews()

這段的目的：這是排程會自動執行的主函式，把前面所有函式串起來，完成「抓取 RSS → 組合訊息 → 推播到 Line」的完整流程。

📰 sendDailyNews() — 每日新聞推播主函式

function sendDailyNews() {
  Logger.log('===== 開始推播每日新聞 =====');

  // 組合新聞訊息
  var newsMessage = buildNewsMessage();

  // Line Bot 單則訊息上限 5000 字
  // 如果超過，只取前 4900 字
  if (newsMessage.length > 4900) {
    newsMessage = newsMessage.substring(0, 4900) + '\n\n...（訊息過長，已截斷）';
  }

  // 推播到 Line
  pushText(LINE_USER_ID, newsMessage);

  Logger.log('===== 每日新聞推播完成 =====');
}

逐行解釋：

  • Logger.log('===== 開始推播每日新聞 =====') — 在執行記錄中印出開始標記，方便後續查看
  • var newsMessage = buildNewsMessage() — 呼叫上一頁學到的 buildNewsMessage()，取得組合好的新聞訊息字串
  • if (newsMessage.length > 4900) — 檢查訊息是否超過 4900 字（保留 100 字給截斷提示）
  • newsMessage.substring(0, 4900) — 如果超過，只取前 4900 個字元
  • pushText(LINE_USER_ID, newsMessage) — 呼叫推播函式，把訊息發送到你的 Line

完整流程圖

1️⃣ 呼叫 buildNewsMessage()
→
2️⃣ 逐一抓取 RSS
→
3️⃣ 格式化組合訊息
→
4️⃣ 檢查 5000 字上限
→
5️⃣ pushText() 推播
⚠️ 5000 字上限：
LINE 推播的單則文字訊息最多 5000 字元。程式碼用 4900 做門檻（預留 100 字給截斷提示文字），這是一種防禦性程式設計的做法。如果你的 RSS 來源太多，建議控制在 3~5 個來源。

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？（第一步！）

  1. 在 Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
  2. 會出現執行歷史記錄列表，每次執行都會留下一筆記錄
  3. 找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
  4. 展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

❌ 收不到 Line 推播

常見錯誤訊息：推播失敗，狀態碼：401

  • 確認 LINE_TOKEN 沒有多餘空格（前後都不能有）
  • 確認 LINE_USER_ID 以 U 開頭（大寫 U）
  • 確認 Line Bot 已加為好友（用手機掃 QR Code）
  • 檢查 LINE Bot 的免費推播額度是否用完（每月 500 則）

解法：先用 testPush 函式測試純文字推播，確認 Token 和 ID 無誤後再測新聞推播。

❌ XmlService.parse() 失敗

常見錯誤訊息：解析 RSS 發生錯誤：The markup in the document preceding the root element must be well-formed.

  • 有些 RSS Feed 的 XML 格式不標準，或包含非法字元
  • 先在瀏覽器直接打開 RSS 網址，確認有 XML 內容
  • 如果瀏覽器顯示的是 HTML 頁面而非 XML，代表網址不是 RSS Feed

解法：把有問題的 RSS 來源從 RSS_FEEDS 陣列中移除即可。程式有 try...catch 保護，不會影響其他來源。

❌ 抓不到新聞（空結果）

常見錯誤訊息：RSS 抓取失敗：... （狀態碼：403）

  • RSS 網址可能已變更或失效（403 = 被拒絕存取）
  • Google 搜尋「該網站名稱 + RSS」找最新網址
  • 用 testFetchSingleRSS 函式單獨測試該來源

解法：更換 RSS 網址，或將該來源從陣列中移除。

❌ 排程沒有觸發

確認步驟：

  • 回到「觸發條件」頁面（左側鬧鐘圖示），確認列表中有一筆記錄
  • 如果記錄旁邊有紅色驚嘆號，表示上次排程執行失敗
  • 先手動跑 testSendDailyNews 確認程式碼沒有問題
  • 確認觸發的函式是 sendDailyNews（不是 test 開頭的）
  • 排程時區以你的 Google 帳號設定為準
