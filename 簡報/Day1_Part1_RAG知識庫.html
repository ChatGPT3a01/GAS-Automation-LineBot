實作 1：RAG 知識庫 API

用 Line Bot 查詢知識庫，即問即答

doGet / doPost Web App 部署 Line Webhook

⏱ 預計時間：50 分鐘 ｜ 📗 Day 1 實作 1

🎯 學習目標

完成這個實作後，你將學會以下四大核心技能：

🌐 doGet() / doPost()

Web App 的核心入口函式。doGet() 處理瀏覽器的 GET 請求，doPost() 處理 POST 請求（包含 Line Webhook）。這是所有 GAS Web 應用的起點。

📊 SpreadsheetApp 讀寫試算表

使用 SpreadsheetApp 讀取與寫入 Google 試算表。學會 getDataRange()、getValues()、appendRow() 等方法，把試算表當作資料庫來使用。

📤 ContentService 回傳 JSON

使用 ContentService.createTextOutput() 搭配 setMimeType(JSON)，讓你的 GAS 程式可以回傳標準 JSON 格式的資料，成為真正的 REST API。

💬 Line Bot Webhook 設定

將 GAS 部署網址設定為 Line Bot 的 Webhook URL，讓使用者在 Line 傳送訊息時，自動觸發 doPost() 來搜尋知識庫並回覆結果。

📝 提示詞範例（直接貼給 AI）

請幫我用 Google Apps Script 建立一個 RAG 知識庫系統，需求如下：

1. 使用 Google 試算表作為知識庫儲存
2. 實作 doGet() 提供 REST API，支援以下 action：
   - action=list：列出所有知識庫資料
   - action=search&query=關鍵字：搜尋知識庫
   - action=stats：回傳統計資料
3. 實作 doPost() 同時處理：
   - REST API 的 CRUD 操作（新增、修改、刪除知識）
   - Line Bot Webhook 事件（接收使用者訊息，搜尋知識庫後回覆）
4. 判斷 doPost 是 Line Webhook 還是 API 呼叫的方法：
   檢查 JSON 資料中是否有 events 陣列
5. 知識庫工作表不存在時自動建立，並填入 5 筆範例資料
6. 使用 ContentService 回傳 JSON 格式
7. 包含 Line Bot 的 pushText 和 replyText 函式
8. LINE_TOKEN 和 LINE_USER_ID 設定在最上方
9. 加上完整的中文註解

REST API 查詢
