🔍 程式碼解析 1 — doGet() 處理 GET 請求

這段的目的：當使用者用瀏覽器輸入網址時觸發，根據 action 參數決定要做什麼（列出全部、搜尋、統計）。

function doGet(e) {
  try {
    var params = e.parameter;
    var action = params.action || 'list';

    switch (action) {
      case 'list':
        return getKnowledgeBase(params);
      case 'search':
        return searchKnowledgeBase(params);
      case 'stats':
        return getStatistics();
      default:
        return createResponse(false, '不支援的操作：' + action, null);
    }
  } catch (error) {
    return createResponse(false, '發生錯誤：' + error.toString(), null);
  }
}

逐行解釋：

  • doGet(e) — GAS 內建的特殊函式，當有人用瀏覽器存取你的 Web App 網址時自動執行。e 是事件物件，包含網址中的參數
  • e.parameter — 取出網址中 ? 後面的參數（例如 ?action=search&query=GAS）
  • params.action || 'list' — 取得 action 參數，如果沒有指定就預設為 'list'
  • switch (action) — 根據 action 的值分流處理，就像一個路由器
  • try...catch — 錯誤處理機制，如果程式出錯不會直接崩潰，而是回傳錯誤訊息
💡 三種 API 操作：
?action=list → 列出所有知識 ｜ ?action=search&query=GAS → 搜尋 ｜ ?action=stats → 統計

🔍 程式碼解析 2 — doPost() 處理 POST 請求

這段的目的：同時處理兩種 POST 請求 — Line Bot Webhook 和 REST API 寫入操作。關鍵在於自動判斷來源。

function doPost(e) {
  try {
    var data = JSON.parse(e.postData.contents);

    // 判斷是否為 Line Bot Webhook 事件
    if (data.events) {
      return handleLineWebhook(data);
    }

    // 否則當作 REST API 處理
    var action = data.action;
    switch (action) {
      case 'add':
        return addKnowledge(data);
      case 'update':
        return updateKnowledge(data);
      case 'delete':
        return deleteKnowledge(data);
      case 'batch_add':
        return batchAddKnowledge(data);
      default:
        return createResponse(false, '不支援的操作：' + action, null);
    }
  } catch (error) {
    return createResponse(false, '發生錯誤：' + error.toString(), null);
  }
}

逐行解釋：

  • e.postData.contents — 取得 POST 請求的原始內容（字串格式）
  • JSON.parse() — 把 JSON 字串解析成 JavaScript 物件，才能讀取裡面的資料
  • if (data.events) — 核心判斷邏輯：Line Webhook 的 JSON 裡面一定有 events 陣列，REST API 不會有
  • handleLineWebhook(data) — 如果是 Line Webhook，交給專門的處理函式
  • switch (action) — 如果是 REST API，根據 action 執行 CRUD 操作
💡 一個 doPost 同時服務兩個來源：
Line 平台傳來的 JSON 固定有 events 欄位，所以只要檢查 data.events 存不存在，就能分辨是 Line Webhook 還是一般 API 呼叫。

🔍 程式碼解析 3 — handleLineWebhook() Line Bot 回覆

這段的目的：使用者在 Line 傳訊息 → 搜尋知識庫 → 組合回覆文字 → 回覆給使用者。

function handleLineWebhook(data) {
  for (var i = 0; i < data.events.length; i++) {
    var event = data.events[i];

    // 只處理文字訊息
    if (event.type !== 'message' || event.message.type !== 'text') {
      continue;
    }

    var userMessage = event.message.text.trim();
    var replyToken = event.replyToken;

    // 搜尋知識庫
    var results = searchKnowledgeData(userMessage);

    // 組合回覆訊息
    var replyMessage = '';
    if (results.length === 0) {
      replyMessage = '🔍 找不到與「' + userMessage + '」相關的知識。\n\n請試試其他關鍵字！';
    } else {
      replyMessage = '🔍 查詢「' + userMessage + '」的結果：\n\n';
      var showCount = Math.min(results.length, 3);
      for (var j = 0; j < showCount; j++) {
        replyMessage += '📌 ' + results[j].question + '\n';
        replyMessage += '💡 ' + results[j].answer + '\n\n';
      }
      if (results.length > 3) {
        replyMessage += '（還有 ' + (results.length - 3) + ' 筆結果未顯示）';
      }
    }

    replyText(replyToken, replyMessage);
    logQuery(userMessage, results.length);
  }

  return ContentService
    .createTextOutput(JSON.stringify({ status: 'ok' }))
    .setMimeType(ContentService.MimeType.JSON);
}

逐行解釋：

  • data.events — Line 一次可能傳來多個事件，所以用 for 迴圈逐一處理
  • event.type !== 'message' — 只處理「訊息」事件（忽略加好友、取消追蹤等）
  • event.message.text.trim() — 取得使用者傳的文字，trim() 去除前後空白
  • event.replyToken — Line 給的回覆令牌，必須用它才能回覆使用者（有效期限很短）
  • searchKnowledgeData(userMessage) — 用使用者的文字去搜尋知識庫
  • Math.min(results.length, 3) — 最多顯示 3 筆結果，避免回覆訊息太長
  • replyText(replyToken, replyMessage) — 把組合好的文字回覆給使用者
  • logQuery() — 把這次查詢記錄到「查詢記錄」工作表

🔍 程式碼解析 4 — searchKnowledgeData() 搜尋核心

這段的目的：從試算表中讀取所有資料，逐列比對關鍵字，找出匹配的知識。

function searchKnowledgeData(query) {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SHEET_NAME);

  if (!sheet) { return []; }

  var data = sheet.getDataRange().getValues();
  var headers = data[0];
  var rows = data.slice(1);
  var searchQuery = query.toLowerCase();

  var results = [];
  for (var i = 0; i < rows.length; i++) {
    var question = String(rows[i][0] || '').toLowerCase();
    var answer = String(rows[i][1] || '').toLowerCase();
    var category = String(rows[i][2] || '').toLowerCase();

    if (question.indexOf(searchQuery) !== -1 ||
        answer.indexOf(searchQuery) !== -1 ||
        category.indexOf(searchQuery) !== -1) {
      results.push({
        rowIndex: i + 2,
        question: rows[i][0],
        answer: rows[i][1],
        category: rows[i][2],
        updateTime: rows[i][3],
        source: rows[i][4]
      });
    }
  }
  return results;
}

逐行解釋：

  • SpreadsheetApp.getActiveSpreadsheet() — 取得目前這個 GAS 所繫結的試算表
  • ss.getSheetByName(SHEET_NAME) — 從試算表中找到名為「知識庫」的工作表
  • sheet.getDataRange().getValues() — 取得工作表中所有有資料的儲存格，回傳一個二維陣列
  • data[0] — 第一列是標題列（問題、答案、分類、更新時間、來源）
  • data.slice(1) — 從第二列開始才是真正的資料
  • query.toLowerCase() — 把搜尋關鍵字轉小寫，實現不分大小寫搜尋
  • indexOf(searchQuery) !== -1 — 檢查字串中是否包含搜尋關鍵字（-1 表示找不到）
  • results.push({...}) — 找到匹配的知識，把它加進結果陣列
💡 搜尋範圍：程式會同時搜尋「問題」、「答案」、「分類」三個欄位。只要任何一個欄位包含關鍵字，該筆知識就會被列入結果。

🔍 程式碼解析 5 — createKnowledgeBaseSheet() 自動建立

這段的目的：首次使用時自動建立「知識庫」工作表，設定標題列格式，並填入 5 筆範例資料。

function createKnowledgeBaseSheet(ss) {
  var sheet = ss.insertSheet(SHEET_NAME);

  // 設定標題列
  var headers = ['問題', '答案', '分類', '更新時間', '來源'];
  sheet.getRange(1, 1, 1, headers.length).setValues([headers]);

  // 格式化標題列
  var headerRange = sheet.getRange(1, 1, 1, headers.length);
  headerRange.setFontWeight('bold');
  headerRange.setBackground('#4CAF50');
  headerRange.setFontColor('#FFFFFF');
  headerRange.setHorizontalAlignment('center');

  // 設定欄寬 + 凍結標題列
  sheet.setColumnWidth(1, 300);
  sheet.setColumnWidth(2, 500);
  sheet.setFrozenRows(1);

  // 新增 5 筆範例資料
  var examples = [
    ['什麼是 Google Apps Script？', 'GAS 是 Google 提供的雲端腳本語言...', '程式設計', now, '範例資料'],
    // ... 共 5 筆
  ];
  sheet.getRange(2, 1, examples.length, 5).setValues(examples);
  return sheet;
}

逐行解釋：

  • ss.insertSheet(SHEET_NAME) — 在試算表中新增一個名為「知識庫」的工作表
  • sheet.getRange(1, 1, 1, 5) — 取得第 1 列、第 1 欄開始、1 列 5 欄的範圍（就是標題列）
  • .setValues([headers]) — 把標題陣列寫入儲存格（注意外面要再包一層陣列）
  • .setBackground('#4CAF50') — 設定背景色為綠色
  • .setFrozenRows(1) — 凍結第一列，捲動時標題列不會消失
  • sheet.getRange(2, 1, 5, 5).setValues(examples) — 一次寫入 5 筆範例資料（批次寫入比逐行 appendRow 快）
💡 5 筆範例資料的主題：

🔍 程式碼解析 6 — Line Bot 推播與回覆函式

這段的目的：提供推播（主動發送）和回覆（回應使用者訊息）兩種 Line Bot 通訊方式。

📤 推播函式（主動發送）

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}

function pushLine(to, messages) {
  var url = 'https://api.line.me/v2/bot/message/push';
  UrlFetchApp.fetch(url, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      to: to, messages: messages
    }),
    muteHttpExceptions: true
  });
}
  • pushText(to, text) — 主動推播文字給指定使用者
  • to — 接收者的 User ID
  • 'Bearer ' + LINE_TOKEN — 身份驗證

💬 回覆函式（回應訊息）

function replyText(replyToken, text) {
  replyLine(replyToken, [{ type: 'text', text: text }]);
}

function replyLine(replyToken, messages) {
  var url = 'https://api.line.me/v2/bot/message/reply';
  UrlFetchApp.fetch(url, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({
      replyToken: replyToken,
      messages: messages
    }),
    muteHttpExceptions: true
  });
}
  • replyText(replyToken, text) — 回覆使用者的訊息
  • replyToken — Line 給的一次性令牌（只能用一次，有效期短）
💡 推播 vs 回覆的差別：
推播（push）：主動發送，需要知道使用者的 User ID，免費帳號每月 500 則。
回覆（reply）：回應使用者的訊息，需要 replyToken，不限次數、不計費。知識庫 Bot 主要用回覆。

🔍 怎麼找到錯誤訊息？（第一步！）

  • 確認 Webhook URL 已正確貼入且點了 Update
  • 確認「Use webhook」開關是綠色開啟狀態
