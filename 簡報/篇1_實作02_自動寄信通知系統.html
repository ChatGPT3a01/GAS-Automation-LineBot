🔍 程式碼解析：createSampleRecipients

逐行看懂這個函式在做什麼：

  • var sheet = SpreadsheetApp.getActiveSheet(); — 取得目前正在看的那個工作表
  • sheet.clear(); — 把工作表上所有資料清除乾淨，從頭開始
  • var headers = ['姓名', 'Email', '主題', '寄送狀態', '寄送時間']; — 用陣列定義 5 個欄位的表頭文字
  • sheet.getRange(1, 1, 1, headers.length).setValues([headers]); — 把表頭寫到第 1 列的 A-E 欄。getRange(列, 欄, 高, 寬)
  • .setFontWeight('bold').setBackground('#1565C0').setFontColor('#FFFFFF') — 表頭設定粗體、藍色背景、白色文字（連續呼叫，叫做 method chaining）
  • var recipients = [['王小明', 'your-email@gmail.com', '活動通知'], ...]; — 用二維陣列準備 3 筆範例收件人（姓名、Email、主題）
  • sheet.getRange(2, 1, recipients.length, 3).setValues(recipients); — 從第 2 列開始，把 3 筆資料一次寫入（批次寫入比逐格快！）
  • sheet.setColumnWidth(1, 100); — 設定 A 欄寬度為 100px，讓表格更好看
  • sheet.setFrozenRows(1); — 凍結第 1 列，往下捲動時表頭會固定在最上面
  • Logger.log('範例名單建立完成！'); — 在「執行記錄」面板印出完成訊息
💡 學到的新技巧：setFrozenRows(1) 凍結表頭列 — 這在資料很多的試算表中非常實用，捲動時表頭始終可見。

🔍 程式碼解析：sendNotifications

這是最核心的函式，逐段看懂它的邏輯：

第一段：讀取資料

  • var sheet = SpreadsheetApp.getActiveSheet(); — 取得目前的工作表
  • var lastRow = sheet.getLastRow(); — 取得最後一列的列號（有 3 筆資料就是 4）
  • if (lastRow <= 1) { ... return; } — 如果只有表頭沒有資料，就印出提示然後直接結束
  • var data = sheet.getRange(2, 1, lastRow - 1, 3).getValues(); — 從第 2 列開始，讀取所有收件人資料（A~C 欄）成為二維陣列

第二段：迴圈逐筆寄信

  • for (var i = 0; i < data.length; i++) — 對每一筆收件人執行以下動作
  • var name = data[i][0], email = data[i][1], subject = data[i][2] || '通知信件'; — 取出姓名、Email、主題。如果主題是空的，用預設值「通知信件」
  • if (!email || email.indexOf('@') === -1) — 檢查 Email 是否為空或不包含 @ 符號。不合格就標記「略過」

第三段：try-catch 錯誤處理

  • try { ... } — 把可能出錯的寄信動作包在 try 裡面
  • MailApp.sendEmail(email, subject, body); — 用三個參數呼叫寄信：收件人、主題、內容
  • sheet.getRange(row, 4).setValue('已寄出').setBackground('#E8F5E9'); — 成功後在 D 欄寫入「已寄出」並上綠底色
  • catch (e) { ... } — 如果寄信失敗（例如額度用完），在 D 欄寫入「失敗」並上紅底色，E 欄記錄錯誤訊息
💡 關鍵觀念：try-catch 讓程式碼遇到錯誤時不會整個中斷，而是跳到 catch 區塊處理錯誤，然後繼續處理下一筆。這在批次操作中非常重要！

🔍 程式碼解析：sendHtmlEmail

HTML 格式信件的關鍵程式碼解析：

function sendHtmlEmail() {
  var sheet = SpreadsheetApp.getActiveSheet();
  var lastRow = sheet.getLastRow();
  var data = sheet.getRange(2, 1, lastRow - 1, 3).getValues();

  for (var i = 0; i < data.length; i++) {
    var name = data[i][0], email = data[i][1],
        subject = data[i][2] || 'HTML 通知';
    if (!email || email.indexOf('@') === -1) continue;

    var htmlBody = '<h2>自動通知</h2>'
      + '<p><strong>' + name + '</strong> 您好</p>';

    MailApp.sendEmail({
      to: email,
      subject: subject,
      htmlBody: htmlBody,
      body: '純文字備援'
    });
  }
}

逐行解釋

  • var htmlBody = '<h2>自動通知</h2>...'; — 用 HTML 標籤組合信件內容。<h2> 是大標題、<strong> 是粗體、<p> 是段落
  • MailApp.sendEmail({ to, subject, htmlBody, body }) — 這裡用物件格式呼叫 sendEmail，不是三個參數的格式
  • htmlBody: htmlBody — 關鍵！用 htmlBody（注意大小寫）就會寄出 HTML 格式的信件
  • body: '純文字備援' — 如果收件人的信箱不支援 HTML，就會顯示這段純文字
  • if (!email || email.indexOf('@') === -1) continue; — Email 不合格就跳過這筆，用 continue 直接進入下一輪迴圈
⚠️ 常見錯誤：寫成 body: htmlBody 而不是 htmlBody: htmlBody，會導致收件人看到 HTML 原始碼（一堆 <tag>），而不是漂亮的格式化信件。

各版本帳號的每日上限

帳號類型每日上限說明
免費 Gmail（@gmail.com）100 封 / 天一般個人帳號
Google Workspace（學校/公司信箱）1,500 封 / 天例如 @school.edu.tw

程式碼（只有兩行就夠用）

function checkQuota() {
  var remaining = MailApp.getRemainingDailyQuota();
  Logger.log('今日剩餘寄信額度：' + remaining + ' 封');
}
⚠️ 額度限制注意事項：
  • 額度每天凌晨（太平洋時間）自動重置
  • 超過上限會收到錯誤：Service invoked too many times for one day
  • MailApp 和 GmailApp 共用同一個額度
  • 大量寄信前，先跑 checkQuota() 確認再決定要不要寄

❓ 常見問題排解

問題原因解決方法
授權時找不到「進階」 Google 更新了介面 往下捲一下，「進階」可能在畫面左下角小字處
錯誤：Service invoked too many times 今日寄信額度已用完 執行 checkQuota() 確認。等隔天凌晨額度自動重置
收件人沒收到信 信件可能被歸類到垃圾郵件 請收件人檢查「垃圾郵件」資料夾，並把你的 Gmail 加入通訊錄
錯誤：Invalid email Email 地址格式不正確 檢查有沒有打錯字、多空格、缺少 @ 符號
HTML 信件顯示原始碼 用了 body 而不是 htmlBody 改用 MailApp.sendEmail({to, subject, htmlBody})，注意是 htmlBody
錯誤：You do not have permission 沒有授權 Gmail 寄信權限 重新執行任一函式，跳出授權彈窗後點「審查權限」→「允許」
寄出的信沒有寄件人名稱 MailApp 不支援設定寄件人名稱 改用 GmailApp.sendEmail(to, subj, body, {name: '系統'})
D 欄顯示「略過」 B 欄的 Email 是空白或沒有 @ 在 B 欄填入正確的 Email 地址，重新執行 sendNotifications
⚠️ 避免被當成垃圾信的技巧：
  • 不要在短時間內寄太多封（建議每批次間隔幾秒）
  • 信件主旨不要全部大寫或有太多驚嘆號
  • 不要放太多連結，尤其是縮網址
