🔧 什麼是自訂函式？

你知道試算表有 =SUM()、=AVERAGE() 這些內建函式嗎？
GAS 讓你可以自己發明新函式，直接在儲存格裡使用！

📘 內建函式

Google 試算表自帶的函式：

  • =SUM(A1:A10) 加總
  • =AVERAGE(A1:A10) 平均
  • =IF(A1>60,"及格","不及格")

🔧 自訂函式（你寫的！）

你在 GAS 裡寫的函式：

  • =calcSubtotal(B2,C2) 算小計
  • =getLevel(D2) 判斷等級
  • 任何你能想到的計算邏輯
💡 關鍵觀念：自訂函式只能用來「回傳計算結果」到儲存格，不能用來修改其他儲存格、寄信、或執行動作。如果需要執行動作，要用一般函式搭配「執行」按鈕。
⚠️ 注意：自訂函式的名稱不能跟 Google 試算表的內建函式同名（例如不能叫 SUM、IF），否則會衝突。

🔍 程式碼解析：calcSubtotal 和 getLevel

calcSubtotal — 計算小計

function calcSubtotal(quantity, price) {
  if (!quantity || !price) return 0;
  return quantity * price;
}
  • function calcSubtotal(quantity, price) — 定義函式，接受「數量」和「單價」兩個參數
  • if (!quantity || !price) return 0; — 防呆機制：如果數量或單價是空的、0、或沒填，就回傳 0，避免算出 NaN
  • return quantity * price; — 把數量乘以單價，回傳小計金額到儲存格

getLevel — 判斷等級

function getLevel(amount) {
  if (amount >= 1000) return '高額';
  if (amount >= 500) return '中等';
  if (amount > 0) return '一般';
  return '-';
}
  • function getLevel(amount) — 接受「金額」一個參數
  • if (amount >= 1000) return '高額'; — 金額 ≥ 1000 回傳「高額」
  • if (amount >= 500) return '中等'; — 金額 ≥ 500（但 < 1000）回傳「中等」
  • if (amount > 0) return '一般'; — 金額 > 0（但 < 500）回傳「一般」
  • return '-'; — 金額為 0 或負數，回傳「-」
💡 if 判斷的順序很重要：因為 JavaScript 會從上往下逐行判斷，碰到第一個成立的條件就回傳。所以要把大的數字（1000）放前面，小的（500、0）放後面。

🔍 程式碼解析：generateReport

這個函式的邏輯分成三個階段：讀取資料 → 計算統計 → 寫入報表

第一階段：讀取資料

  • var ss = SpreadsheetApp.getActiveSpreadsheet(); — 取得整份試算表檔案
  • var sheet = ss.getActiveSheet(); — 取得目前的工作表
  • var lastRow = sheet.getLastRow(); — 取得最後一列的列號（例如有 5 筆資料 + 1 列表頭 = 6）
  • if (lastRow <= 1) { ... return; } — 如果只有表頭（或根本沒資料），就印出訊息並結束
  • var data = sheet.getRange(2, 1, lastRow - 1, 4).getValues(); — 從第 2 列開始讀取所有資料（跳過表頭），getValues() 回傳二維陣列

第二階段：用 for 迴圈計算統計

  • var totalAmount = 0, totalItems = 0; — 初始化累計變數
  • for (var i = 0; i < data.length; i++) — 用迴圈跑過每一筆資料
  • var name = data[i][0] — data[i] 是第 i 列，[0] 是第一欄（品項名稱）
  • var subtotal = qty * price; — 計算小計
  • totalAmount += subtotal; — 累加到總金額
  • if (subtotal > maxItem.amount) — 追蹤最高金額的品項

第三階段：寫入新工作表

  • ss.insertSheet('報表') — 在試算表中插入一個名為「報表」的新工作表
  • reportSheet.getRange('A3').setValue(...) — 逐格寫入統計資料

6 建立簽到表 — createSignInSheet

接下來我們來做一個簽到系統。第一步是建立簽到表的表頭和格式：

簽到表程式碼

var SIGNIN_SHEET_NAME = '簽到表';

function createSignInSheet() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);
  if (!sheet) {
    sheet = ss.insertSheet(SIGNIN_SHEET_NAME);
  }
  sheet.clear();
  var headers = ['編號', '姓名', '簽到時間', '備註'];
  sheet.getRange(1, 1, 1, 4).setValues([headers]);
  var headerRange = sheet.getRange(1, 1, 1, 4);
  headerRange.setFontWeight('bold');
  headerRange.setBackground('#1b5e20');
  headerRange.setFontColor('#FFFFFF');
  headerRange.setHorizontalAlignment('center');
  sheet.setFrozenRows(1);
  sheet.setColumnWidths(1, 4, 120);
  Logger.log('簽到表建立完成！');
}

🔍 程式碼解析：appendRow vs setValues

簽到系統使用兩種方式寫入資料，它們的差別非常重要：

signIn — 單筆簽到（appendRow）

function signIn(name, note) {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);
  var nextNum = sheet.getLastRow(); // 表頭佔 1 列
  note = note || '已簽到';
  sheet.appendRow([nextNum, name, new Date(), note]);
  Logger.log(name + ' 簽到成功！');
}
  • sheet.getLastRow() — 取得最後一列的列號，用來算編號
  • note = note || '已簽到'; — 如果沒傳備註參數，就預設「已簽到」
  • sheet.appendRow([...]) — 在最下面新增一列，參數是一維陣列（一列資料）
  • new Date() — 取得當下的時間

appendRow vs setValues 速度比較

特性appendRow()setValues()
寫入方式一次加一列一次寫入整批
參數格式一維陣列 [a, b, c]二維陣列 [[a,b],[c,d]]
寫入位置自動加在最後面必須指定 getRange
速度（100 列）約 30 秒約 1 秒
適合場景即時紀錄、簽到批次匯入、報表

8 批次寫入 — batchSignIn

如果有 10 個人要一次簽到，用 appendRow 要呼叫 10 次，太慢了！
用 setValues 可以一次寫入所有資料：

batchSignIn 程式碼

function batchSignIn() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SIGNIN_SHEET_NAME);
  if (!sheet) { createSignInSheet(); sheet = ss.getSheetByName(SIGNIN_SHEET_NAME); }

  // 準備批次資料（二維陣列）
  var names = ['Alice', 'Bob', 'Charlie', 'David', 'Eva',
               'Frank', 'Grace', 'Henry', 'Ivy', 'Jack'];
  var batchData = [];
  for (var i = 0; i < names.length; i++) {
    batchData.push([i + 1, names[i], new Date(), '批次簽到']);
  }

  // 一次寫入所有資料
  sheet.getRange(2, 1, batchData.length, 4).setValues(batchData);
  Logger.log('批次簽到完成！共 ' + names.length + ' 人');
}

執行步驟

📍 位置：Apps Script 編輯器上方的函式下拉選單

👆 動作：選擇「batchSignIn」→ 點擊「▶ 執行」

⏱️ 等待 2~3 秒（注意：10 筆資料只要 1 次寫入，非常快！）

👀 結果：執行紀錄顯示「批次簽到完成！共 10 人」

📍 位置：切回試算表 → 點擊「簽到表」分頁

👆 動作：檢查簽到表內容

👀 結果：看到 10 筆 Alice~Jack 的簽到紀錄（注意：會覆蓋之前的 3 筆資料）

⚠️ setValues 的重要觀念：它是「覆寫」指定範圍的資料，不是「附加」。如果想保留舊資料再新增，需要先用 getLastRow() 找到最後一列，再從下一列開始寫入。

⚡ appendRow vs setValues 完整比較

這是 GAS 中最常用的兩種寫入方式，理解差別對效能非常重要：

➕ appendRow() — 逐筆新增

// 每次呼叫 = 1 次網路請求
for (var i = 0; i < 100; i++) {
  sheet.appendRow([i, '資料', new Date()]);
}
// ❌ 100 次請求 ≈ 30 秒
  • 自動加在最後一列
  • 不需要指定位置
  • 每次呼叫都是一次網路請求
  • 適合：即時簽到、單筆記錄

⚡ setValues() — 批次寫入

// 先準備好全部資料
var data = [];
for (var i = 0; i < 100; i++) {
  data.push([i, '資料', new Date()]);
}
sheet.getRange(2,1,100,3).setValues(data);
// ✅ 1 次請求 ≈ 1 秒
  • 一次寫入整批資料
  • 需要指定 getRange 位置
  • 只有 1 次網路請求
  • 適合：批次匯入、報表產生
💡 效能原則：GAS 的每次讀寫（getValues / setValues / appendRow）都是一次網路請求。減少請求次數 = 程式跑更快。所以能用 setValues 一次寫完，就不要用 appendRow 逐筆寫。
⚠️ setValues 常見錯誤 — size mismatch：
getRange(2, 1, 10, 3) 指定了 10 列 3 欄，但你的陣列只有 8 列或 4 欄 → 大小不一致就會報錯。
解決方法：用 data.length 和 data[0].length 動態取得陣列大小。

❓ 常見問題排解

問題原因解決方法
自訂函式在儲存格不會出現提示 函式的 JSDoc 註解格式不對 確認有 /** */ 格式的註解，且包含 @customfunction（注意：教學版的 calcSubtotal 沒有加 JSDoc，也能使用，只是不會有自動提示）
自訂函式顯示 #ERROR! 函式內部有語法錯誤 回到 Apps Script 編輯器，按 Ctrl + S 存檔，如果有紅底線就修正語法錯誤
setValues 報錯 size mismatch 陣列的行列數跟 getRange 不一致 用 getRange(row, col, data.length, data[0].length) 動態對應
找不到工作表（null） getSheetByName() 名稱跟實際分頁不一致 確認工作表分頁名稱完全一致（包含空格、大小寫）
getLastRow() 回傳 0 工作表是空的 先確認有資料，或先執行 createSampleData()
格式化沒有效果 程式碼修改後沒儲存就執行 按 Ctrl + S 儲存後再重新執行
「超過最大執行時間」 程式碼有無限迴圈，或 appendRow 寫太多筆 改用 setValues 批次寫入，避免在迴圈中逐筆呼叫 API
appendRow 加在錯誤的列 工作表中間有空白列 getLastRow 會找到最後一列有資料的位置，確保資料連續沒有空白列
