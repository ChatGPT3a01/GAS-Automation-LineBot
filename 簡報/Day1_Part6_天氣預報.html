📋 複製這段提示詞給 AI Prompt Template 
請幫我用 Google Apps Script 製作一個「天氣預報 + Line Bot 每日自動推播」系統。

功能需求：
1. 串接中央氣象署開放資料 API（資料集 F-C0032-001，36小時天氣預報）
2. 使用 UrlFetchApp.fetch() 呼叫 API，中文縣市名稱要用 encodeURIComponent 編碼
3. 取得指定縣市（預設臺北市）的天氣預報資料
4. 解析 JSON 回傳結果，提取以下欄位：
   - Wx（天氣現象：晴、多雲、陰天等）
   - PoP（降雨機率 %）
   - MinT（最低溫 °C）
   - MaxT（最高溫 °C）
   - CI（舒適度）
5. 格式化成好看的天氣報告，包含 emoji 圖示：
   - 依天氣狀況顯示不同 emoji（晴天 ☀️、多雲 ⛅、雨天 🌧 等）
   - 顯示溫度範圍和降雨機率
   - 降雨機率 >= 50% 時加上「記得帶傘」提醒
   - 高溫 >= 35°C 提醒防曬，低溫 <= 15°C 提醒保暖
6. 透過 Line Bot Push API 推播天氣預報
7. 同時將天氣資料記錄到試算表的「天氣記錄」工作表

API 設定用變數放最上面：
- LINE_TOKEN（Line Bot Channel Access Token）
- LINE_USER_ID（Line User ID）
- CWA_API_KEY（中央氣象署 API 授權碼）
- TARGET_CITY（查詢的縣市名稱）

另外提供一個 testApiConnection() 測試函式，
確認 API 連線是否正常。

🌐 UrlFetchApp 解說

GAS 內建的 HTTP 請求工具，用來呼叫外部 API

基本用法

// 1. 呼叫 API
var response = UrlFetchApp.fetch(url, {
  muteHttpExceptions: true   // 錯誤時不中斷程式
});

// 2. 檢查回應狀態碼
var code = response.getResponseCode();
// 200 = 成功, 401 = 授權失敗, 404 = 找不到

// 3. 解析 JSON 回應
var json = JSON.parse(response.getContentText());

// 4. 取出天氣資料
var locations = json.records.location;
var weatherElements = locations[0].weatherElement;
💡 muteHttpExceptions: true 的作用：
預設情況下，如果 API 回傳錯誤（例如 401、404），GAS 會直接報錯中斷。加上這個設定後，程式不會中斷，你可以自己用 getResponseCode() 來判斷和處理錯誤。

📡 fetch()

發送 HTTP 請求到指定 URL

📊 getResponseCode()

取得回應狀態碼（200 = 成功）

📝 getContentText()

取得回應的文字內容

🎨 天氣 Emoji 對照表

getWeatherEmoji() 函式邏輯

天氣關鍵字Emoji說明

晴時多雲🌤天氣描述包含「晴時多雲」或「晴」
多雲⛅天氣描述包含「多雲」
陰天☁️天氣描述包含「陰」
雨天🌧天氣描述包含「雨」
雷雨⛈天氣描述包含「雷」

霧🌫天氣描述包含「霧」
function getWeatherEmoji(description) {
  if (description.indexOf('雷') !== -1) return '⛈';
  if (description.indexOf('雪') !== -1) return '🌨';
  if (description.indexOf('雨') !== -1) return '🌧';
  if (description.indexOf('陰') !== -1) return '☁️';
  if (description.indexOf('多雲') !== -1) return '⛅';
  if (description.indexOf('晴') !== -1) return '🌤';
  if (description.indexOf('霧') !== -1) return '🌫';
  return '🌤';  // 預設
}
💡 判斷順序很重要：「雷雨」要先判斷「雷」，因為它也包含「雨」字；「晴時多雲」要先判斷「多雲」再判斷「晴」。
