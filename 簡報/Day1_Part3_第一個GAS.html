請幫我用 Google Apps Script 寫一個試算表操作程式，搭配 Line Bot 推播，需求如下：

1. 基本讀寫函式：
   - readCell()：讀取 A1 儲存格的值，結果推播到 Line
   - writeCell()：寫入值到指定儲存格（文字、日期、數字）
   - readRange()：讀取 A1:C5 範圍的所有資料，格式化後推播到 Line
2. 自訂函式（可在試算表中用 =函式名() 呼叫）：
   - calcSubtotal(quantity, price)：計算小計（數量 × 單價）
   - getLevel(amount)：根據金額判斷等級（>=1000 高額, >=500 中等, >0 一般）
   記得加上 @customfunction JSDoc 標記
3. 報表推播函式 sendDailyReport()：
   - 讀取試算表中的品項、數量、單價資料
   - 計算每個品項的小計和總金額
   - 找出最高金額品項
   - 格式化成漂亮的報表，用 emoji 裝飾
   - 推播到 Line Bot
4. createSampleData()：自動建立範例資料（5 個飲料品項），D 欄使用自訂函式
5. LINE_TOKEN 和 LINE_USER_ID 設定在最上方
6. 加上中文註解

1 建立新的 Google 試算表

首先，我們要建立一個全新的 Google 試算表，作為銷售報表的資料來源。

Step 1-1：快速建立試算表

👆 動作：在瀏覽器網址列輸入 sheets.new 然後按 Enter
⏱️ 等待：約 3~5 秒，瀏覽器會自動建立並開啟一個全新的 Google 試算表
👀 結果：你會看到一個空白的試算表，左上角顯示「未命名的試算表」

Step 1-2：替試算表命名

📍 位置：試算表畫面的左上角，有一行小字寫著「未命名的試算表」
👆 動作：點擊「未命名的試算表」那個文字，它會變成可編輯的輸入框
👆 動作：輸入 銷售報表，然後按 Enter 或點擊其他地方
👀 結果：左上角的名稱會變成「銷售報表」
💡 為什麼要用 sheets.new？
這是 Google 提供的快速建立捷徑。你也可以到 sheets.google.com 點「+」建立，效果一樣。但 sheets.new 更快，一步到位。

2 開啟 Apps Script 編輯器

從試算表中開啟 Apps Script 編輯器，這是我們寫程式碼的地方。

Step 2-1：找到擴充功能選單

📍 位置：在試算表上方的選單列（檔案、編輯、檢視... 那一排），找到最右邊的「擴充功能」
👆 動作：點擊「擴充功能」
👀 結果：會展開一個下拉選單

Step 2-2：點擊 Apps Script

👆 動作：在下拉選單中找到 Apps Script，點擊它
⏱️ 等待：瀏覽器會開啟一個全新的分頁，等待 3~5 秒直到編輯器完全載入
👀 結果：你會看到 Apps Script 編輯器：左邊是檔案列表（有一個「程式碼.gs」）、右邊是程式碼編輯區、上方有一個 ▶ 執行 按鈕
💡 確認你看到的畫面：
編輯區裡面應該有一段預設程式碼：function myFunction() { }。這段等一下會全部刪除，替換成我們的銷售報表程式碼。
⚠️ 找不到「擴充功能」？
確認你正在使用 Google 試算表（不是 Excel）。如果你用的是公司帳號，管理員可能停用了 Apps Script，請改用個人 Gmail 帳號操作。

4 修改設定值

程式碼最上方有 2 個設定值需要替換為你自己的。這是唯一需要手動修改的地方。

找到設定區（程式碼第 24~26 行附近）

// ========== Line Bot 設定 ==========
var LINE_TOKEN = '在此貼上你的 Channel Access Token';
var LINE_USER_ID = '在此貼上你的 User ID';

逐一替換 2 個設定值

設定項目 要替換的中文提示 替換成什麼 去哪裡找
LINE_TOKEN 在此貼上你的 Channel Access Token 你的 Channel Access Token（很長一串） LINE Developers → 你的 Bot → Messaging API 頁籤 → 最下方 Channel access token
LINE_USER_ID 在此貼上你的 User ID 你的 User ID（U 開頭的字串） LINE Developers → 你的 Bot → Basic settings 頁籤 → Your user ID

替換方式（以 LINE_TOKEN 為例）

👆 動作：在程式碼中找到 '在此貼上你的 Channel Access Token'，用滑鼠只選取單引號中間的中文字（不要選到單引號本身！）
👆 動作：按 Ctrl+V 貼上你的 Channel Access Token
👀 結果：看起來會像 var LINE_TOKEN = 'eyJhbGciOiJ...(略)...';
⚠️ 三個常見錯誤，請務必檢查：
  • 多餘的空格 — Token 和 ID 的前後不要有空格
  • 單引號被刪掉 — 兩個值都要保持用單引號 ' ' 包起來
  • 複製到多餘字元 — 從 LINE Developers 複製時，確認沒有多複製到換行符號
👆 動作：2 個值都替換完後，按 Ctrl+S 儲存

Step 5-1：選擇要執行的函式

📍 位置：Apps Script 編輯器上方工具列，在 ▶ 執行 按鈕的左邊，有一個下拉選單（預設可能顯示「readCell」或其他函式名稱）

👆 動作：找到並選擇 createSampleData

👀 結果：如果是第一次執行，會跳出一個「需要授權」的對話框

Step 5-3：完成授權流程（只有第一次需要）

授權第 1 步：彈出「需要授權」的對話框 → 點擊 審查權限 按鈕
授權第 2 步：選擇你的 Google 帳號（點擊你的帳號名稱）
授權第 3 步：會出現「Google 尚未驗證這個應用程式」的警告畫面 → 點擊左下角的 進階 小字
授權第 4 步：畫面會展開更多文字 → 點擊 前往「銷售報表」（不安全）
授權第 5 步：會列出程式需要的權限清單 → 點擊 允許 按鈕
⏱️ 等待：授權完成後，程式會自動開始執行，等待 5~10 秒

Step 5-4：確認結果

✅ 成功的樣子

  • 切回試算表分頁，你會看到 A~D 欄已填入 5 筆飲料資料
  • 你的 Line 收到一則通知：「✅ 範例資料已建立完成！」

❌ 失敗的樣子

  • 執行記錄顯示紅色的錯誤文字

  • → 最常見原因：授權流程沒有完成

🔍 程式碼解析 1 — 基本讀寫操作

這段的目的：學會 GAS 讀取和寫入試算表的最基礎方法 — 所有試算表操作都從這裡開始。

📁 readCell() — 讀取單一儲存格

function readCell() {
  var sheet = SpreadsheetApp.getActiveSheet();
  var value = sheet.getRange('A1').getValue();
  Logger.log('A1 的值是：' + value);

  // 推播到 Line
  pushText(LINE_USER_ID, '📖 讀取結果\nA1 儲存格的值是：' + value);
}
  • SpreadsheetApp.getActiveSheet() — 取得目前開啟的工作表物件（就是你看到的那個分頁）
  • sheet.getRange('A1') — 指定要操作的儲存格位置，這裡是 A 欄第 1 列
  • .getValue() — 取出該儲存格裡的值（文字、數字或日期都可以）
  • Logger.log() — 將訊息記錄到「執行記錄」中，方便除錯
  • pushText() — 呼叫推播函式，將結果傳到 Line

📁 writeCell() — 寫入儲存格

function writeCell() {
  var sheet = SpreadsheetApp.getActiveSheet();
  sheet.getRange('E1').setValue('這是 GAS 寫入的文字');
  sheet.getRange('E2').setValue(new Date());   // 寫入目前時間
  sheet.getRange('E3').setValue(12345);         // 寫入數字

  Logger.log('已成功寫入 E1~E3');
  pushText(LINE_USER_ID, '✏️ 寫入完成！\n已在 E1~E3 儲存格寫入資料。');
}
  • .setValue('文字') — 把指定的值寫入儲存格
  • new Date() — JavaScript 內建的日期物件，會寫入目前的時間戳記
  • 可以寫入三種類型：文字（字串）、日期（Date 物件）、數字
💡 記憶口訣：取表 → 指格 → 讀寫
getActiveSheet() → getRange() → getValue() 或 setValue()。所有試算表操作都遵循這三步驟。

🔍 程式碼解析 2 — 讀取範圍 readRange()

這段的目的：一次讀取多個儲存格的資料，回傳一個二維陣列，適合處理整張表格。

📁 readRange() — 讀取 A1:C5 範圍

function readRange() {
  var sheet = SpreadsheetApp.getActiveSheet();
  var data = sheet.getRange('A1:C5').getValues();

  // data 是一個二維陣列
  Logger.log('共讀取 ' + data.length + ' 列資料');

  var message = '📊 試算表資料：\n\n';
  for (var i = 0; i < data.length; i++) {
    // data[i] 是第 i 列的陣列
    if (data[i][0] !== '') {
      message += data[i][0] + ' | ' + data[i][1] + ' | ' + data[i][2] + '\n';
    }
  }

  Logger.log(message);
  pushText(LINE_USER_ID, message);
}

逐行解釋：

  • getRange('A1:C5') — 指定一個範圍（A 欄到 C 欄，第 1 列到第 5 列）
  • .getValues() — 注意有 s！一次讀取整個範圍，回傳二維陣列
  • data.length — 二維陣列的列數（這裡是 5 列）
  • data[i][0] — 第 i 列的第 1 欄（A 欄），data[i][1] 是 B 欄，data[i][2] 是 C 欄
  • if (data[i][0] !== '') — 跳過空白列，只處理有資料的列

二維陣列長什麼樣？

試算表中的資料

ABC
品項名稱數量單價
美式咖啡1060
拿鐵咖啡880

getValues() 回傳的陣列

[
  ['品項名稱', '數量', '單價'],  // data[0]
  ['美式咖啡', 10, 60],          // data[1]
  ['拿鐵咖啡', 8, 80],           // data[2]
  ...
]

data[1][0] = '美式咖啡'
data[1][2] = 60

💡 getValue() vs getValues()：
getValue()（沒有 s）— 讀取單一儲存格，回傳一個值
getValues()（有 s）— 讀取範圍內所有儲存格，回傳二維陣列

🔍 程式碼解析 3 — 自訂函式 @customfunction

這段的目的：建立可在試算表中直接使用的自訂函式，像 =SUM() 一樣方便。

📁 calcSubtotal(quantity, price) — 計算小計

/**
 * 自訂函式：計算小計（數量 × 單價）
 *
 * 在試算表中使用方式：
 *   在 D2 儲存格輸入 =calcSubtotal(B2, C2)
 *
 * @param {number} quantity 數量
 * @param {number} price 單價
 * @return {number} 小計金額
 * @customfunction
 */
function calcSubtotal(quantity, price) {
  if (!quantity || !price) return 0;
  return quantity * price;
}
  • /** ... */ — JSDoc 註解區塊，Google 會讀取這裡的資訊來生成自動完成提示
  • @param {number} quantity — 告訴 Google「第一個參數是數字，名稱叫 quantity」
  • @return {number} — 告訴 Google「這個函式會回傳一個數字」
  • @customfunction — 最關鍵的標記！有了這個，函式才能在試算表中用 = 呼叫
  • if (!quantity || !price) return 0 — 防呆：如果數量或單價是空的，回傳 0 避免錯誤
  • return quantity * price — 計算結果會顯示在儲存格中

📁 getLevel(amount) — 等級判斷

/**
 * @param {number} amount 金額
 * @return {string} 等級文字
 * @customfunction
 */
function getLevel(amount) {
  if (amount >= 1000) return '🔴 高額';
  if (amount >= 500) return '🟡 中等';
  if (amount > 0) return '🟢 一般';
  return '-';
}
  • 使用 if 條件判斷，從最高金額開始往下判斷
  • ≥ 1000 → 高額、≥ 500 → 中等、> 0 → 一般、其他 → 顯示 -
  • return 會直接跳出函式，所以不需要 else
💡 自訂函式小技巧：
在試算表中輸入 =calc 時，自動完成清單會出現你的 calcSubtotal，連同你寫的說明文字！這就是 JSDoc 註解的功用。

🔍 程式碼解析 4 — sendDailyReport() 報表推播

這段的目的：讀取所有品項資料，計算統計數據，組合成格式化報表，透過 Line 推播。

function sendDailyReport() {
  var sheet = SpreadsheetApp.getActiveSheet();
  var lastRow = sheet.getLastRow();

  // 如果沒有資料（只有標題列）
  if (lastRow <= 1) {
    pushText(LINE_USER_ID, '📊 報表通知：試算表中尚無資料。');
    return;
  }

  // 讀取所有資料（從第 2 列開始，跳過標題）
  var data = sheet.getRange(2, 1, lastRow - 1, 4).getValues();

  var totalAmount = 0;
  var totalItems = 0;
  var maxItem = { name: '', amount: 0 };
  var itemList = [];

  for (var i = 0; i < data.length; i++) {
    var name = data[i][0];     // A 欄：品項名稱
    var qty = data[i][1];      // B 欄：數量
    var price = data[i][2];    // C 欄：單價
    var subtotal = qty * price; // 計算小計

    if (name === '') continue;

    totalAmount += subtotal;
    totalItems += qty;
    itemList.push(name + ' ×' + qty + ' = $' + subtotal);

    if (subtotal > maxItem.amount) {
      maxItem = { name: name, amount: subtotal };
    }
  }
  // ... 組合報表訊息並推播
}

逐行解釋：

  • sheet.getLastRow() — 取得最後一列有資料的列號（例如有 6 列資料就回傳 6）
  • if (lastRow <= 1) — 防呆：如果只有標題列（或完全空白），就推播提醒並結束
  • sheet.getRange(2, 1, lastRow - 1, 4) — 從第 2 列第 1 欄開始，讀取 (lastRow-1) 列 × 4 欄
  • totalAmount += subtotal — 累加每個品項的小計，得到總金額
  • if (subtotal > maxItem.amount) — 追蹤金額最高的品項
  • continue — 跳過品項名稱為空的列（防止計算空白列）
💡 getRange 的兩種寫法：
getRange('A1:C5') — 用字串表示（適合固定範圍）
getRange(2, 1, lastRow-1, 4) — 用數字表示（列, 欄, 列數, 欄數），適合動態範圍

🔍 程式碼解析 5 — 報表訊息組合

這段的目的：把統計出來的數據組合成一段漂亮的文字訊息，透過 Line 推播。

  // 組合報表訊息
  var now = new Date().toLocaleString('zh-TW', { timeZone: 'Asia/Taipei' });
  var report = '📊 每日銷售報表\n';
  report += '━━━━━━━━━━━━\n';
  report += '📅 ' + now + '\n\n';

  // 各品項明細
  for (var j = 0; j < itemList.length; j++) {

  report += '\n━━━━━━━━━━━━\n';
  report += '💰 總金額：$' + totalAmount.toLocaleString() + '\n';
  report += '📦 總數量：' + totalItems + ' 個\n';

逐行解釋：

  • report += — 用字串串接的方式，一段一段組合完整報表

🔍 程式碼解析 6 — Line Bot 推播函式

這段的目的：提供推播工具函式，讓前面所有函式都能輕鬆把訊息傳到 Line。

📁 pushText(to, text) — 推播文字訊息

function pushText(to, text) {
  pushLine(to, [{ type: 'text', text: text }]);
}
  • to — 要推播給誰（就是 LINE_USER_ID）
  • text — 要傳送的文字內容
  • 它其實是呼叫下面的 pushLine()，把文字包成 Line API 要求的格式

📁 pushLine(to, messages) — 推播底層函式

function pushLine(to, messages) {
  UrlFetchApp.fetch('https://api.line.me/v2/bot/message/push', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + LINE_TOKEN
    },
    payload: JSON.stringify({ to: to, messages: messages }),
    muteHttpExceptions: true
  });
}
  • UrlFetchApp.fetch() — GAS 內建的 HTTP 請求工具，等同於瀏覽器發送網路請求
  • 'https://api.line.me/v2/bot/message/push' — Line Messaging API 的推播端點
  • 'Authorization': 'Bearer ' + LINE_TOKEN — 告訴 Line「我是哪一個 Bot」的身份驗證
  • JSON.stringify() — 把 JavaScript 物件轉換成 JSON 字串（Line API 要求的格式）
  • muteHttpExceptions: true — 即使發生錯誤也不要中斷程式，讓我們自己處理
💡 為什麼要分成兩個函式？
pushText() 是「簡化版」— 只需傳入文字就好。pushLine() 是「底層版」— 負責實際的 HTTP 請求。這種設計叫做「封裝」，讓程式碼更好讀、更好用。

🔍 程式碼解析 7 — createSampleData() 範例資料

這段的目的：一鍵建立 5 筆完整的範例資料，包含格式化標題列和自訂函式公式。

function createSampleData() {
  var sheet = SpreadsheetApp.getActiveSheet();

  // 清除現有資料
  sheet.clear();

  // 寫入標題
  var headers = ['品項名稱', '數量', '單價', '小計'];
  sheet.getRange(1, 1, 1, 4).setValues([headers]);
  sheet.getRange(1, 1, 1, 4)
    .setFontWeight('bold')
    .setBackground('#FF9800')
    .setFontColor('#FFFFFF')
    .setHorizontalAlignment('center');

  // 寫入範例品項
  var items = [
    ['美式咖啡', 10, 60],
    ['拿鐵咖啡', 8, 80],
    ['珍珠奶茶', 15, 55],
    ['綠茶', 12, 30],
    ['紅茶', 6, 25]
  ];

  for (var i = 0; i < items.length; i++) {
    var row = i + 2;
    sheet.getRange(row, 1).setValue(items[i][0]);
    sheet.getRange(row, 2).setValue(items[i][1]);
    sheet.getRange(row, 3).setValue(items[i][2]);
    // D 欄使用自訂函式
    sheet.getRange(row, 4).setFormula('=calcSubtotal(B' + row + ',C' + row + ')');
  }

  // ... 設定欄寬 + 凍結標題列
}

重點解釋：

  • sheet.clear() — 清除工作表上的所有資料和格式（重新開始）
  • setValues([headers]) — 注意要用二維陣列（陣列中的陣列），所以要加 [ ] 包起來
  • .setFontWeight('bold').setBackground('#FF9800') — 鏈式呼叫，一口氣設定粗體 + 背景色
  • setFormula('=calcSubtotal(B2,C2)') — 在儲存格中寫入公式（而不是固定值）
  • var row = i + 2 — 因為第 1 列是標題，資料從第 2 列開始
  • setFrozenRows(1) — 凍結第 1 列，捲動時標題不會跑掉
💡 setValue() vs setValues() vs setFormula()：
setValue('Hello') — 寫入一個固定值
setValues([['A','B'],['C','D']]) — 一次寫入整個範圍（二維陣列）
setFormula('=SUM(A1:A5)') — 寫入公式，儲存格會自動計算結果

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？（第一步！）

  1. 在 Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示，從上面數第 3 個），點擊它
  2. 會出現執行歷史記錄列表，每次執行都會留下一筆記錄
  3. 找到最近一筆（最上面的），如果狀態是紅色「失敗」，點擊那一行的「▶」展開箭頭
  4. 展開後看詳細資訊中的紅色文字 — 那就是錯誤原因

Line 收不到訊息

  • 確認 LINE_TOKEN 有正確貼上（很長的英數字串）
  • 確認 LINE_USER_ID 以 U 開頭
  • 確認 Line Bot 已加為好友（用手機掃 QR Code）
  • 檢查執行記錄：401 = Token 不正確，400 = User ID 格式錯誤
  • 免費帳號每月推播上限 500 則

試算表沒有資料

  • 確認已先執行 createSampleData（步驟 5）
  • 切回試算表分頁確認資料是否存在
  • 如果執行時報錯「需要授權」，重新跑一次授權流程
  • 確認程式碼完整貼上（不是只貼了一部分）

自訂函式不工作

  • 確認 JSDoc 中有 @customfunction 標記
  • 在儲存格中輸入 =calcSubtotal(10,50) 測試
  • 如果顯示「Unknown function」→ 程式碼沒有儲存，按 Ctrl+S
  • 如果顯示「Loading...」很久 → 重新整理試算表頁面

報表金額不對

  • 確認試算表 B 欄（數量）和 C 欄（單價）的值正確
  • 確認沒有空白列混在資料中間（程式碼會跳過空白列，但不會跳過有空格的列）
  • sendDailyReport() 是自己計算 qty * price，不是讀取 D 欄
  • 確認資料從第 2 列開始（第 1 列是標題，會被跳過）
