1 取得 Gemini API Key

要讓 GAS 呼叫 AI，需要一組 API Key。Google 提供免費的 Gemini API 額度。

Step 1-1：前往 Google AI Studio

👆 動作：打開瀏覽器，前往 https://aistudio.google.com/apikey
👀 結果：會看到 Google AI Studio 的 API Key 管理頁面

Step 1-2：建立 API Key

👆 動作：點擊 Create API Key（建立 API 金鑰）按鈕
👆 動作：選擇你的 Google Cloud 專案（如果沒有，系統會自動建立一個）
👀 結果：畫面上會出現一串類似 AIzaSy... 開頭的長字串，這就是你的 API Key

Step 1-3：複製並保存

👆 動作：點擊 API Key 旁邊的複製圖示，把這串金鑰存到安全的地方
👀 結果：等一下要貼到 GAS 程式碼裡面
⚠️ API Key 安全須知：
- API Key 就像你家的鑰匙，不要分享給別人
- 不要把 API Key 貼在公開的網頁或社群媒體
- 免費版有使用量限制（每分鐘 15 次請求），足夠上課練習用

2 呼叫 Gemini API

用 UrlFetchApp.fetch() 向 Gemini API 發送請求，就像在跟 AI 聊天一樣。

API 呼叫的基本結構

// ===== 呼叫 Gemini AI =====
function callGemini(prompt) {
  var url = 'https://generativelanguage.googleapis.com/v1beta/models/'
          + 'gemini-2.5-flash:generateContent?key=' + GEMINI_API_KEY;

  var payload = {
    contents: [{
      parts: [{ text: prompt }]
    }]
  };

  var options = {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  };

  var response = UrlFetchApp.fetch(url, options);
  var json = JSON.parse(response.getContentText());

  // 取出 AI 回覆的文字
  return json.candidates[0].content.parts[0].text;
}

逐行解釋

  • url — Gemini API 的網址，末尾要附上你的 API Key
  • payload — 要傳送給 AI 的內容，包含你的提示詞（prompt）
  • options — HTTP 請求的設定：用 POST 方法、JSON 格式
  • UrlFetchApp.fetch() — 發送請求，拿到 AI 的回覆
  • json.candidates[0].content.parts[0].text — 從回覆中取出 AI 的文字

3 設計 Prompt — 讓 AI 回傳 JSON

Prompt 設計的關鍵：要求 AI 用 JSON 格式回覆，這樣程式才能解析。

我們的 Prompt 範本

// ===== 產生簡報大綱的 Prompt =====
function buildPrompt(topic) {
  return '請幫我製作一份關於「' + topic + '」的簡報大綱。\n'
       + '回覆格式必須是純 JSON（不要加 markdown 標記），結構如下：\n'
       + '{\n'
       + '  "title": "簡報標題",\n'
       + '  "subtitle": "副標題",\n'
       + '  "slides": [\n'
       + '    {\n'
       + '      "heading": "這一頁的標題",\n'
       + '      "bullets": ["重點1", "重點2", "重點3"]\n'
       + '    }\n'
       + '  ]\n'
       + '}\n'
       + '請產生 5~8 頁內容頁，每頁 3~4 個重點。';
}

為什麼用 JSON？

❌ 純文字回覆

AI 回覆自由格式文字，程式很難拆解每一頁的標題和內容在哪裡。

✅ JSON 結構化回覆

AI 回覆固定格式的 JSON，程式用 JSON.parse() 就能精準取得每筆資料。

💡 Prompt 技巧：明確告訴 AI「回覆格式必須是純 JSON，不要加 markdown 標記」，這樣 AI 就不會在 JSON 前後加上 ```json 等標記，程式解析時就不會出錯。

- 預設投影片大小：720 x 405 點（10 x 5.625 英吋，16:9 比例）

5 建立封面頁

封面頁需要：深色背景 + 大標題 + 副標題，讓簡報看起來專業。

// ===== 建立封面頁 =====
function createCoverSlide(presentation, title, subtitle) {
  // 取得第一頁（建立簡報時會自動產生一頁）
  var slide = presentation.getSlides()[0];

  // 設定深綠色背景
  slide.getBackground().setSolidFill('#1b5e20');

  // 插入標題文字方塊（置中偏上）
  var titleBox = slide.insertTextBox(title, 60, 100, 600, 80);
  var titleStyle = titleBox.getText().getTextStyle();
  titleStyle.setFontSize(40)
            .setBold(true)
            .setForegroundColor('#ffffff');
  titleBox.getText().getParagraphStyle().setParagraphAlignment(
    SlidesApp.ParagraphAlignment.CENTER
  );

  // 插入副標題文字方塊（置中偏下）
  var subBox = slide.insertTextBox(subtitle, 100, 220, 520, 60);
  var subStyle = subBox.getText().getTextStyle();
  subStyle.setFontSize(20)
          .setForegroundColor('#c8e6c9');
  subBox.getText().getParagraphStyle().setParagraphAlignment(
    SlidesApp.ParagraphAlignment.CENTER
  );
}

逐行解釋

  • getSlides()[0] — 取得第一頁投影片（索引 0）
  • setSolidFill('#1b5e20') — 設定背景為深綠色
  • insertTextBox(文字, 左, 上, 寬, 高) — 在指定位置插入文字方塊
  • setFontSize(40) — 設定字體大小 40pt
  • setBold(true) — 設定粗體
  • setForegroundColor('#ffffff') — 設定文字顏色為白色

6 建立內容頁 — 迴圈產生多頁

用 for 迴圈，把 AI 生成的每一頁大綱都變成真正的投影片。

// ===== 建立內容頁（迴圈） =====
function createContentSlides(presentation, slides) {
  for (var i = 0; i < slides.length; i++) {
    var slideData = slides[i];

    // 新增一頁空白投影片
    var slide = presentation.appendSlide(
      SlidesApp.PredefinedLayout.BLANK
    );

    // 插入頁面標題（上方橫條）
    var headerBg = slide.insertShape(
      SlidesApp.ShapeType.RECTANGLE, 0, 0, 720, 60
    );
    headerBg.getFill().setSolidFill('#1b5e20');
    headerBg.getBorder().setTransparent();

    var headerText = slide.insertTextBox(
      slideData.heading, 20, 10, 680, 45
    );
    headerText.getText().getTextStyle()
      .setFontSize(24).setBold(true)
      .setForegroundColor('#ffffff');

    // 插入重點內容（逐條列出）
    var bulletText = '';
    for (var j = 0; j < slideData.bullets.length; j++) {
      bulletText += '● ' + slideData.bullets[j] + '\n\n';
    }

    var contentBox = slide.insertTextBox(
      bulletText, 40, 80, 640, 300
    );
    contentBox.getText().getTextStyle()
      .setFontSize(18)
      .setForegroundColor('#333333');
  }
}
💡 重點：外層 for 迴圈跑每一頁，內層 for 迴圈跑每一頁的重點條目。這就是為什麼要讓 AI 回傳 JSON 格式 — 程式用 slides[i].heading 和 slides[i].bullets[j] 就能精準取得每一筆資料。

📋 AI 回傳的 JSON 長什麼樣？

當你呼叫 Gemini API 後，AI 會回傳一段 JSON 文字。以下是一個實際回傳的範例：

AI 實際回傳的 JSON 範例： 
{
  "title": "人工智慧入門",
  "subtitle": "從基礎觀念到實際應用",
  "slides": [
    {
      "heading": "什麼是人工智慧？",
      "bullets": [
        "AI 是讓電腦模擬人類智慧的技術",
        "包含機器學習、深度學習、自然語言處理",
        "已廣泛應用於日常生活"
      ]
    },
    {
      "heading": "AI 的三大類型",
      "bullets": [...]
    }
  ]
}

每個欄位代表什麼意思？

  • "title" — 簡報的大標題，會放在封面頁
  • "subtitle" — 副標題，放在封面頁標題下方
  • "slides" — 一個陣列，每個元素是一頁投影片
  • "heading" — 這一頁的標題，會顯示在頁面上方的綠色橫條
  • "bullets" — 這一頁的重點列表，每個字串是一個重點條目
💡 程式怎麼使用這段 JSON？
程式就是用 JSON.parse() 把這段文字變成物件，然後用 outline.title 取標題、outline.slides[0].heading 取第一頁標題、outline.slides[0].bullets[0] 取第一頁的第一個重點。

7 完整流程測試

把所有功能串起來，一個函式搞定全部！

// ===== 主函式：一鍵生成簡報 =====
function generatePresentation() {
  var topic = '人工智慧入門';  // ← 改成你想要的主題

  // Step 1：用 AI 生成大綱
  Logger.log('正在請 AI 生成大綱...');
  var prompt = buildPrompt(topic);
  var aiReply = callGemini(prompt);

  // Step 2：清理 AI 回覆（移除可能的 markdown 標記）
  aiReply = aiReply.replace(/```json\n?/g, '')
                   .replace(/```\n?/g, '')
                   .trim();

  // Step 3：解析 JSON
  var outline = JSON.parse(aiReply);
  Logger.log('大綱標題：' + outline.title);
  Logger.log('共 ' + outline.slides.length + ' 頁內容');

  // Step 4：建立簡報
  var pres = SlidesApp.create(outline.title);
  Logger.log('簡報已建立：' + pres.getUrl());

  // Step 5：建立封面頁
  createCoverSlide(pres, outline.title, outline.subtitle);

  // Step 6：建立內容頁
  createContentSlides(pres, outline.slides);

  // 完成！
  Logger.log('✅ 簡報生成完成！共 ' + (outline.slides.length + 1) + ' 頁');
  Logger.log('📎 開啟連結：' + pres.getUrl());
}

成功 / 失敗對照

✅ 成功的樣子

  • Google Drive 根目錄出現新簡報
  • 封面頁有標題和副標題，深綠色背景
  • 5~8 頁內容頁，每頁有標題和重點條目
  • 執行記錄顯示綠色「已完成」

❌ 失敗怎麼辦

  • API Key 錯誤（403）→ 到 Google AI Studio 重新複製 API Key
  • JSON 解析失敗→ 再執行一次（AI 每次回覆可能不同）
  • 簡報排版不好看→ 調整 Prompt，限制每頁字數和重點數量

❓ 常見問題排解

🔍 怎麼找到錯誤訊息？

  1. 在 Apps Script 編輯器左側選單，找到「⏱ 執行項目」圖示（時鐘圖示），點擊它
  2. 找到最近一筆執行記錄，如果狀態是紅色「失敗」，點擊展開看詳細資訊

❌ SyntaxError: Unexpected token

原因：AI 回覆的 JSON 格式不正確，無法用 JSON.parse() 解析

  • AI 可能在 JSON 前後加了 ```json 標記
  • AI 可能回覆了多餘的文字說明

解法：程式碼中的 .replace() 已經處理了大部分情況。如果還是失敗，再執行一次（AI 每次回覆可能不同）。

❌ 403 或 429 錯誤

原因：API Key 無效或超過免費額度

  • 確認 API Key 沒有多餘空格
  • 免費版每分鐘限 15 次請求
  • 等一分鐘再試

解法：到 Google AI Studio 確認 API Key 狀態。

❌ 簡報排版很醜

原因：文字太多超出文字方塊範圍

  • 在 Prompt 中限制每頁重點數量（3~4 個）
  • 限制每個重點的字數（20 字以內）

解法：調整 Prompt，或修改文字方塊的大小參數。

❌ 找不到簡報在哪

原因：簡報建立在 Google Drive 根目錄

  • 查看執行記錄中的簡報網址
  • 到 Google Drive 首頁 → 最近存取

解法：點擊 Logger 印出的網址即可開啟。
