1 建立範例試卷

先用程式自動建立一份 Google Docs 試卷，做為審題的測試資料。

// ===== 建立範例試卷（測試用） =====
function createSampleExam() {
  var doc = DocumentApp.create('段考範例試卷_數學科');
  var body = doc.getBody();

  body.appendParagraph('第一學期段考 — 數學科試題')
      .setHeading(DocumentApp.ParagraphHeading.HEADING1);

  body.appendParagraph('');

  // 第 1 題
  body.appendParagraph('1. 請問 3 + 5 = ?');
  body.appendParagraph('(A) 7  (B) 8  (C) 9  (D) 10');
  body.appendParagraph('');

  // 第 2 題
  body.appendParagraph('2. 下列哪一個是質數？');
  body.appendParagraph('(A) 4  (B) 6  (C) 7  (D) 9');
  body.appendParagraph('');

  // 第 3 題
  body.appendParagraph('3. 一個長方形的長是 8 公分，寬是 5 公分，'
    + '請問面積是多少平方公分？');
  body.appendParagraph('(A) 13  (B) 26  (C) 40  (D) 45');
  body.appendParagraph('');

  // 第 4 題（故意有問題的題目）
  body.appendParagraph('4. 以下哪個答案是對的？');
  body.appendParagraph('(A) 是  (B) 不是  (C) 也許  (D) 以上皆是');

  Logger.log('✅ 範例試卷已建立！');
  Logger.log('📄 文件 ID：' + doc.getId());
  Logger.log('📎 開啟連結：' + doc.getUrl());
}
👆 動作：先執行 createSampleExam 建立測試試卷
👀 結果：Logger 會印出文件 ID，把它複製下來，等一下要用
⚠️ 注意：第 4 題是故意設計的「有問題的題目」（題意不清、選項不合理），用來測試 AI 能不能抓到這種問題。

2 讀取文件內容

用 DocumentApp 打開 Google Docs，把整份試卷的文字取出來。

// ===== 設定區 =====
var GEMINI_API_KEY = '你的_API_Key_貼這裡';
var EXAM_DOC_ID   = '你的_文件ID_貼這裡';

// ===== 讀取文件內容 =====
function readExamContent() {
  var doc = DocumentApp.openById(EXAM_DOC_ID);
  var text = doc.getBody().getText();

  Logger.log('文件標題：' + doc.getName());
  Logger.log('文件內容（前 200 字）：');
  Logger.log(text.substring(0, 200));

  return text;
}

逐行解釋

  • DocumentApp.openById(ID) — 用文件 ID 打開一份 Google Docs
  • doc.getBody() — 取得文件的主體（body）
  • .getText() — 把文件內容轉成純文字字串
  • text.substring(0, 200) — 只印前 200 個字，避免 Logger 太長
💡 DocumentApp vs SpreadsheetApp：
- SpreadsheetApp 操作的是 Google Sheets（試算表）
- DocumentApp 操作的是 Google Docs（文件）
- 兩者都用 openById() 開啟，但之後的操作方法不同

3 設計審題 Prompt

Prompt 的品質決定了 AI 審題的效果。我們要求 AI 回傳結構化 JSON。

// ===== 建立審題 Prompt =====
function buildReviewPrompt(examText) {
  return '你是一位資深教師，請審查以下段考試題的品質。\n\n'
    + '【試題內容】\n' + examText + '\n\n'
    + '請針對每一題進行審查，回覆純 JSON 格式（不要加 markdown 標記）：\n'
    + '{\n'
    + '  "totalQuestions": 題目總數,\n'
    + '  "overallScore": 整體品質分數(1-10),\n'
    + '  "summary": "整體評語",\n'
    + '  "questions": [\n'
    + '    {\n'
    + '      "number": 題號,\n'
    + '      "clarity": 清晰度分數(1-5),\n'
    + '      "difficulty": "簡單/中等/困難",\n'
    + '      "optionQuality": 選項品質分數(1-5),\n'
    + '      "issues": "發現的問題（無則填「無」）",\n'
    + '      "suggestion": "改善建議（無則填「無」）"\n'
    + '    }\n'
    + '  ]\n'
    + '}';
}

Prompt 設計的 3 個關鍵

1. 角色設定

「你是一位資深教師」— 讓 AI 用專業的角度審查試題

2. 明確任務

「針對每一題審查」— 告訴 AI 逐題分析，不是只看整體

3. 格式要求

「回覆純 JSON 格式」— 程式才能用 JSON.parse() 解析

4 呼叫 Gemini AI 審查

把試題內容和 Prompt 一起送給 Gemini，讓 AI 幫你審查。

// ===== 呼叫 Gemini AI =====
function callGemini(prompt) {
  var url = 'https://generativelanguage.googleapis.com/v1beta/models/'
          + 'gemini-2.5-flash:generateContent?key=' + GEMINI_API_KEY;

  var payload = {
    contents: [{ parts: [{ text: prompt }] }]
  };

  var options = {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  };

  var response = UrlFetchApp.fetch(url, options);
  var json = JSON.parse(response.getContentText());

  // 檢查是否有錯誤
  if (json.error) {
    Logger.log('API 錯誤：' + json.error.message);
    return null;
  }

  return json.candidates[0].content.parts[0].text;
}
💡 跟實作 9 的 callGemini 一樣！如果你做過實作 9（AI 自動生成簡報），這個函式你已經很熟了。唯一的差別是這裡多了錯誤檢查（json.error）。

5 解析 AI 回傳結果

AI 回傳 JSON 字串後，用 JSON.parse() 轉成物件，就能取出每一題的評分。

// ===== 解析 AI 回覆 =====
function parseReviewResult(aiReply) {
  // 清理可能的 markdown 標記
  var cleaned = aiReply
    .replace(/```json\n?/g, '')
    .replace(/```\n?/g, '')
    .trim();

  var result = JSON.parse(cleaned);

  // 印出摘要
  Logger.log('===== 審查結果摘要 =====');
  Logger.log('題目數量：' + result.totalQuestions);
  Logger.log('整體評分：' + result.overallScore + ' / 10');
  Logger.log('整體評語：' + result.summary);
  Logger.log('');

  // 印出每題的審查結果
  for (var i = 0; i < result.questions.length; i++) {
    var q = result.questions[i];
    Logger.log('--- 第 ' + q.number + ' 題 ---');
    Logger.log('  清晰度：' + q.clarity + '/5');
    Logger.log('  難度：' + q.difficulty);
    Logger.log('  選項品質：' + q.optionQuality + '/5');
    Logger.log('  問題：' + q.issues);
    Logger.log('  建議：' + q.suggestion);
  }

  return result;
}
⚠️ JSON.parse() 可能失敗：如果 AI 回覆的格式不正確，JSON.parse() 會拋出錯誤。程式碼中的 .replace() 已處理常見的 markdown 標記問題。如果還是失敗，再執行一次即可（AI 每次回覆可能略有不同）。

6 寫入審查報告到試算表

把 AI 的審查結果整理成表格，寫入 Google Sheets，方便後續查閱和分享。

// ===== 寫入審查報告 =====
function writeReportToSheet(result) {
  var ss = SpreadsheetApp.create('段考審查報告_' + new Date().toLocaleDateString());
  var sheet = ss.getActiveSheet();
  sheet.setName('審查結果');

  // 寫入標題列
  var headers = ['題號', '清晰度(1-5)', '難度', '選項品質(1-5)',
                 '發現問題', '改善建議'];
  sheet.getRange(1, 1, 1, headers.length).setValues([headers]);

  // 設定標題列樣式
  var headerRange = sheet.getRange(1, 1, 1, headers.length);
  headerRange.setBackground('#1b5e20')
             .setFontColor('#ffffff')
             .setFontWeight('bold');

  // 寫入每題的審查結果
  for (var i = 0; i < result.questions.length; i++) {
    var q = result.questions[i];
    var row = [q.number, q.clarity, q.difficulty,
               q.optionQuality, q.issues, q.suggestion];
    sheet.getRange(i + 2, 1, 1, row.length).setValues([row]);
  }

  // 寫入整體評語
  var summaryRow = result.questions.length + 3;
  sheet.getRange(summaryRow, 1).setValue('整體評分');
  sheet.getRange(summaryRow, 2).setValue(result.overallScore + ' / 10');
  sheet.getRange(summaryRow + 1, 1).setValue('整體評語');
  sheet.getRange(summaryRow + 1, 2).setValue(result.summary);

  // 自動調整欄寬
  sheet.autoResizeColumns(1, headers.length);

  Logger.log('📊 審查報告已寫入：' + ss.getUrl());
  return ss;
}
💡 重點：用 SpreadsheetApp.create() 建立全新的試算表，而不是覆蓋原有的。報告名稱包含日期，方便辨識是哪次審查的結果。

7 完整流程測試

把所有步驟串起來，一個函式搞定！

// ===== 主函式：一鍵審題 =====
function reviewExamQuestions() {
  Logger.log('📝 開始審查段考試題...');

  // Step 1：讀取試卷內容
  var examText = readExamContent();
  Logger.log('✅ 已讀取試卷，共 ' + examText.length + ' 個字');

  // Step 2：建立審題 Prompt
  var prompt = buildReviewPrompt(examText);

  // Step 3：呼叫 AI 審查
  Logger.log('🤖 正在請 AI 審查...');
  var aiReply = callGemini(prompt);
  if (!aiReply) {
    Logger.log('❌ AI 回覆失敗，請檢查 API Key');
    return;
  }

  // Step 4：解析結果
  var result = parseReviewResult(aiReply);

  // Step 5：寫入報告
  var report = writeReportToSheet(result);

  // 完成！
  Logger.log('');
  Logger.log('🎉 審查完成！');
  Logger.log('📊 報告連結：' + report.getUrl());
}

測試方式

👆 動作：確認 GEMINI_API_KEY 和 EXAM_DOC_ID 已正確設定
👆 動作：在函式下拉選單中選擇 reviewExamQuestions，然後點擊 ▶ 執行
⏱️ 等待：約 15~30 秒（讀文件 + AI 分析 + 寫報告）
👀 結果：Logger 會印出審查摘要和試算表連結

✅ 成功的樣子

  • Logger 印出「審查完成！」
  • 每一題都有清晰度、難度、選項品質的評分
  • 第 4 題（故意有問題的）應該被 AI 標註「題意不清」

❌ 失敗的樣子

  • 出現紅色錯誤文字

  • 「SyntaxError」→ AI 回覆格式有誤，再執行一次
  • 「文件找不到」→ 檢查 EXAM_DOC_ID 是否正確

❓ 常見問題排解

❌ DocumentApp 權限錯誤

錯誤訊息：You do not have permission to call DocumentApp.openById

  • 確認文件 ID 正確（從文件網址中擷取）
  • 確認你有該文件的存取權限
  • 第一次執行需要授權，按照授權流程操作

解法：先執行 createSampleExam 建立自己的測試文件。

❌ JSON.parse 失敗

錯誤訊息：SyntaxError: Unexpected token

  • AI 回覆可能包含 markdown 標記或多餘文字
  • 程式的 .replace() 已處理常見狀況

解法：再執行一次。如果多次失敗，在 Prompt 中加強「只回傳純 JSON」的要求。

❌ API Key 錯誤（403）

錯誤訊息：API 錯誤：API key not valid

  • API Key 前後不能有空格
  • 確認是在 aistudio.google.com 取得的
  • 確認沒有超過免費額度

解法：重新複製 API Key，注意不要複製到空格。

❌ AI 審查結果不準確

現象：AI 的評分或建議與預期不符

  • AI 不是完美的，每次結果可能略有不同
  • 審查結果應做為「參考」，不是最終判定

解法：調整 Prompt 中的評分標準，或多次執行取平均。
